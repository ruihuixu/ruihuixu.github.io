# **1. 每⽇温度**


[⼒扣题⽬链接](https://leetcode.cn/problems/daily-temperatures/)


请根据每⽇ ⽓温 列表，重新⽣成⼀个列表。对应位置的输出为：要想观测到更⾼的⽓温，⾄少需要等待的天数。

如果⽓温在这之后都不会升⾼，请在该位置⽤ 0 来代替。


例如，给定⼀个列表 temperatures = [73, 74, 75, 71, 69, 72, 76, 73]，你的输出应该是 [1, 1, 4, 2, 1, 1, 0, 0]。


提示：⽓温 列表⻓度的范围是 [1, 30000]。每个⽓温的值的均为华⽒度，都是在 [30, 100] 范围内的整数。

## **思路**


⾸先想到的当然是暴⼒解法，两层for循环，把⾄少需要等待的天数就搜出来了。时间复杂度是O(n^2)


那么接下来在来看看使⽤单调栈的解法。


那有同学就问了，我怎么能想到⽤单调栈呢？ 什么时候⽤单调栈呢？


**通常是⼀维数组，要寻找任⼀个元素的右边或者左边第⼀个⽐⾃⼰⼤或者⼩的元素的位置，此时我们就要想到可以**

**⽤单调栈了** 。时间复杂度为O(n)。


例如本题其实就是找找到⼀个元素右边第⼀个⽐⾃⼰⼤的元素，此时就应该想到⽤单调栈了。


那么单调栈的原理是什么呢？为什么时间复杂度是O(n)就可以找到每⼀个元素的右边第⼀个⽐它⼤的元素位置呢？


**单调栈的本质是空间换时间** ，因为在遍历的过程中需要⽤⼀个栈来记录右边第⼀个⽐当前元素⾼的元素，优点是整

个数组只需要遍历⼀次。


**更直⽩来说，就是⽤⼀个栈来记录我们遍历过的元素** ，因为我们遍历数组的时候，我们不知道之前都遍历了哪些元

素，以⾄于遍历⼀个元素找不到是不是之前遍历过⼀个更⼩的，所以我们需要⽤⼀个容器（这⾥⽤单调栈）来记录

我们遍历过的元素。


在使⽤单调栈的时候⾸先要明确如下⼏点：


1. 单调栈⾥存放的元素是什么？


单调栈⾥只需要存放元素的下标i就可以了，如果需要使⽤对应的元素，直接T[i]就可以获取。


2. 单调栈⾥元素是递增呢？ 还是递减呢？


**注意以下讲解中，顺序的描述为** **从栈头到栈底的顺序** ，因为单纯的说从左到右或者从前到后，不说栈头朝哪个⽅向

的话，⼤家⼀定⽐较懵。


这⾥我们要使⽤递增循序（再强调⼀下是指从栈头到栈底的顺序），因为只有递增的时候，栈⾥要加⼊⼀个元素i的

时候，才知道栈顶元素在数组中右⾯第⼀个⽐栈顶元素⼤的元素是i。


即：如果求⼀个元素右边第⼀个更⼤元素，单调栈就是递增的，如果求⼀个元素右边第⼀个更⼩元素，单调栈就是

递减的。


⽂字描述理解起来有点费劲，接下来我画了⼀系列的图，来讲解单调栈的⼯作过程，⼤家再去思考，本题为什么是

递增栈。


使⽤单调栈主要有三个判断条件。


当前遍历的元素T[i]⼩于栈顶元素T[st.top()]的情况

当前遍历的元素T[i]等于栈顶元素T[st.top()]的情况

当前遍历的元素T[i]⼤于栈顶元素T[st.top()]的情况


**把这三种情况分析清楚了，也就理解透彻了** 。


接下来我们⽤temperatures = [73, 74, 75, 71, 71, 72, 76, 73]为例来逐步分析，输出应该是 [1, 1, 4, 2, 1, 1, 0, 0]。


⾸先先将第⼀个遍历元素加⼊单调栈


加⼊T[1] = 74，因为T[1] > T[0]（当前遍历的元素T[i]⼤于栈顶元素T[st.top()]的情况）。


我们要保持⼀个递增单调栈（从栈头到栈底），所以将T[0]弹出，T[1]加⼊，此时result数组可以记录了，result[0]

= 1，即T[0]右⾯第⼀个⽐T[0]⼤的元素是T[1]。


加⼊T[2]，同理，T[1]弹出


加⼊T[3]，T[3] < T[2] （当前遍历的元素T[i]⼩于栈顶元素T[st.top()]的情况），加T[3]加⼊单调栈。


加⼊T[4]，T[4] == T[3] （当前遍历的元素T[i]等于栈顶元素T[st.top()]的情况），此时依然要加⼊栈，不⽤计算距

离，因为我们要求的是右⾯第⼀个⼤于本元素的位置，⽽不是⼤于等于！


加⼊T[5]，T[5] > T[4] （当前遍历的元素T[i]⼤于栈顶元素T[st.top()]的情况），将T[4]弹出，同时计算距离，更新

result


T[4]弹出之后， T[5] > T[3] （当前遍历的元素T[i]⼤于栈顶元素T[st.top()]的情况），将T[3]继续弹出，同时计算距

离，更新result


直到发现T[5]⼩于T[st.top()]，终⽌弹出，将T[5]加⼊单调栈


加⼊T[6]，同理，需要将栈⾥的T[5]，T[2]弹出


同理，继续弹出


此时栈⾥只剩下了T[6]


加⼊T[7]， T[7] < T[6] 直接⼊栈，这就是最后的情况，result数组也更新完了。


此时有同学可能就疑惑了，那result[6], result[7]怎么没更新啊，元素也⼀直在栈⾥。


其实定义result数组的时候，就应该直接初始化为0，如果result没有更新，说明这个元素右⾯没有更⼤的了，也就

是为0。


以上在图解的时候，已经把，这三种情况都做了详细的分析。


情况⼀：当前遍历的元素T[i]⼩于栈顶元素T[st.top()]的情况

情况⼆：当前遍历的元素T[i]等于栈顶元素T[st.top()]的情况

情况三：当前遍历的元素T[i]⼤于栈顶元素T[st.top()]的情况


通过以上过程，⼤家可以⾃⼰再模拟⼀遍，就会发现：只有单调栈递增（从栈⼝到栈底顺序），就是求右边第⼀个

⽐⾃⼰⼤的，单调栈递减的话，就是求右边第⼀个⽐⾃⼰⼩的。


C++代码如下：


// 版本⼀


class Solution {


public:


vector<int> dailyTemperatures(vector<int>& T) {

// 递增栈


stack<int> st;


vector<int> result(T.size(), 0);


st.push(0);


for (int i = 1; i < T.size(); i++) {

if (T[i] < T[st.top()]) {            // 情况⼀


st.push(i);


} else if (T[i] == T[st.top()]) {        // 情况⼆


st.push(i);


} else {

while (!st.empty() && T[i] > T[st.top()]) { // 情况三


result[st.top()] = i            - st.top();


st.pop();


}


st.push(i);


}


}


return result;


}


};

// 版本⼆


class Solution {


public:


vector<int> nextGreaterElement(vector<int>& nums1, vector<int>& nums2) {


stack<int> st;


vector<int> result(nums1.size(), -1);


if (nums1.size() == 0) return result;


unordered_map<int, int> umap; // key:下标元素，value：下标


for (int i = 0; i < nums1.size(); i++) {


umap[nums1[i]] = i;


}


st.push(0);


for (int i = 1; i < nums2.size(); i++) {


while (!st.empty() && nums2[i] > nums2[st.top()]) {

if (umap.count(nums2[st.top()]) > 0) { // 看map⾥是否存在这个元素

int index = umap[nums2[st.top()]]; // 根据map找到nums2[st.top()] 在

nums1中的下标


result[index] = nums2[i];


}


st.pop();


}


st.push(i);


}


return result;


}


};

### **双指针解法**


[本题双指针的写法整体思路和42. 接⾬⽔是⼀致的，但要⽐42. 接⾬⽔难⼀些。](https://programmercarl.com/0042.%E6%8E%A5%E9%9B%A8%E6%B0%B4.html)


难就难在本题要记录记录每个柱⼦ 左边第⼀个⼩于该柱⼦的下标，⽽不是左边第⼀个⼩于该柱⼦的⾼度。


[所以需要循环查找，也就是下⾯在寻找的过程中使⽤了while，详细请看下⾯注释，整理思路在题解：42. 接⾬⽔中](https://programmercarl.com/0042.%E6%8E%A5%E9%9B%A8%E6%B0%B4.html)

class Solution {


public:


int largestRectangleArea(vector<int>& heights) {


vector<int> minLeftIndex(heights.size());


vector<int> minRightIndex(heights.size());


int size = heights.size();


// 记录每个柱⼦ 左边第⼀个⼩于该柱⼦的下标

minLeftIndex[0] = -1; // 注意这⾥初始化，防⽌下⾯while死循环


for (int i = 1; i < size; i++) {


int t = i        - 1;

// 这⾥不是⽤if，⽽是不断向左寻找的过程


while (t >= 0 && heights[t] >= heights[i]) t = minLeftIndex[t];


minLeftIndex[i] = t;


}


// 记录每个柱⼦ 右边第⼀个⼩于该柱⼦的下标

minRightIndex[size      - 1] = size; // 注意这⾥初始化，防⽌下⾯while死循环


for (int i = size      - 2; i >= 0; i--) {


int t = i + 1;

// 这⾥不是⽤if，⽽是不断向右寻找的过程


while (t < size && heights[t] >= heights[i]) t = minRightIndex[t];


minRightIndex[i] = t;


}

// 求和


int result = 0;


for (int i = 0; i < size; i++) {


int sum = heights[i] * (minRightIndex[i] - minLeftIndex[i] - 1);


result = max(sum, result);


}


return result;


}


};

class Solution {


public:


int largestRectangleArea(vector<int>& heights) {


int result = 0;


stack<int> st;

heights.insert(heights.begin(), 0); // 数组头部加⼊元素0

heights.push_back(0); // 数组尾部加⼊元素0


st.push(0);


// 第⼀个元素已经⼊栈，从下标1开始


for (int i = 1; i < heights.size(); i++) {

if (heights[i] > heights[st.top()]) { // 情况⼀


st.push(i);

} else if (heights[i] == heights[st.top()]) { // 情况⼆

st.pop(); // 这个可以加，可以不加，效果⼀样，思路不同


st.push(i);

} else { // 情况三

st.pop();


if (!st.empty()) {

int right = i;

}


}

}

⾸先来说末尾为什么要加元素0？


如果数组本身就是升序的，例如[2,4,6,8]，那么⼊栈之后 都是单调递减，⼀直都没有⾛ 情况三 计算结果的哪⼀

步，所以最后输出的就是0了。 如图：


那么结尾加⼀个0，就会让栈⾥的所有元素，⾛到情况三的逻辑。


开头为什么要加元素0？


如果数组本身是降序的，例如 [8,6,4,2]，在 8 ⼊栈后，6 开始与8 进⾏⽐较，此时我们得到 mid（8），rigt

（6），但是得不到 left。

class Solution {


public:


int largestRectangleArea(vector<int>& heights) {

st.pop();

}


st.push(i);


}
