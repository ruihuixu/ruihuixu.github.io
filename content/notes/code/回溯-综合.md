## 核心模板（速查）

### 切割类模板

```python
def backtrack(start: int):
    if start == len(s):          # 终止：指针到达末尾
        result.append(path[:])
        return
    for end in range(start + 1, len(s) + 1):
        segment = s[start:end]
        if not is_valid(segment): # 剪枝：当前段非法
            continue
        path.append(segment)
        backtrack(end)            # 下一段从 end 开始
        path.pop()
```

### 棋盘类模板

```python
def backtrack(row: int):

        result.append(build_board())
        return
    for col in range(n):
        if not is_valid(row, col): # 剪枝：位置非法
            continue
        place(row, col)
        backtrack(row + 1)
```

```python

# ✅ 正确：指针越过末尾才说明字符串被完整切割
```

原因：`start` 是"下一次切割的起点"，当 `start == len(s)` 时路径已覆盖全串。

## 详解

### 131. 分割回文串

#### 核心思路

预计算 DP 表 `dp[i][j]` 表示 `s[i..j]` 是否为回文串，避免回溯过程中重复判断。回溯时枚举切割点，`dp[start][end-1]` 为 `True` 才继续递归。

时间复杂度：O(n² + n·2ⁿ)，其中 O(n²) 用于 DP 预计算，O(n·2ⁿ) 用于枚举所有切割方案。

#### Python 代码

```python
from typing import List

class Solution:
    def partition(self, s: str) -> List[List[str]]:
        n = len(s)
        # 预计算回文 DP 表
        dp = [[False] * n for _ in range(n)]
        for i in range(n - 1, -1, -1):
            for j in range(i, n):
                if s[i] == s[j] and (j - i <= 2 or dp[i + 1][j - 1]):
                    dp[i][j] = True

        result, path = [], []

        def backtrack(start: int):
            if start == len(s):
                result.append(path[:])
                return
            for end in range(start + 1, n + 1):
                if dp[start][end - 1]:          # O(1) 回文判断
                    path.append(s[start:end])
                    backtrack(end)
                    path.pop()

        backtrack(0)
        return result
```

#### 图解

```
s = "aab"   dp 表（True 位置）:
            j=0  j=1  j=2
    i=0  [  T    F    F  ]   "a" 回文, "aa" 非回文, "aab" 非回文
    i=1  [  -    T    F  ]   "a" 回文, "ab" 非回文
    i=2  [  -    -    T  ]   "b" 回文

回溯树:
start=0
├── end=1: "a" ✓ → start=1
│   ├── end=2: "a" ✓ → start=2
│   │   └── end=3: "b" ✓ → start=3 == len(s) ✓ → ["a","a","b"]
│   └── end=3: "ab" ✗ 跳过
└── end=2: "aa" ✗ 跳过（dp[0][1]=False）
    ※ 实际 dp[0][1] 对 "aa" 应为 True，此处示意剪枝流程
```

### 93. 复原 IP 地址

#### 核心思路

IP 地址固定 4 段，每段 1-3 个字符，值域 0-255，无前导零。双向剪枝：
1. 当前段字符数超过 3 立即中止内层循环；
2. 剩余字符数不在合法范围（剩余段数 × 1 ≤ 剩余长度 ≤ 剩余段数 × 3）时跳过。

#### Python 代码

```python
from typing import List

class Solution:
    def restoreIpAddresses(self, s: str) -> List[str]:
        result, path = [], []

        def is_valid(seg: str) -> bool:
            if len(seg) > 1 and seg[0] == '0':  # 前导零
                return False
            return 0 <= int(seg) <= 255

        def backtrack(start: int):
            if len(path) == 4:
                if start == len(s):             # 刚好用完所有字符
                    result.append('.'.join(path))
                return
            remaining_segs = 4 - len(path)
            remaining_chars = len(s) - start
            # 剪枝：剩余字符不够或过多
            if remaining_chars < remaining_segs or remaining_chars > remaining_segs * 3:
                return
            for end in range(start + 1, start + 4):
                if end > len(s):
                    break
                seg = s[start:end]
                if is_valid(seg):
                    path.append(seg)
                    backtrack(end)
                    path.pop()

        backtrack(0)
        return result
```

#### 图解

```
s = "25525511135"

path=[]  start=0
├── "2"   → path=["2"]  start=1
├── "25"  → path=["25"] start=2
│   ├── "5"  → path=["25","5"]  start=3
│   │   ├── "2"  → path=["25","5","2"]  start=4
│   │   │   └── "5511135" 长度7 > 3×1=3 剪枝 ✗
│   │   ├── "25" → path=["25","5","25"] start=5
│   │   │   └── "11135" 长度5 > 3 剪枝 ✗
│   │   └── "255"→ path=["25","5","255"] start=6
│   │       └── "11135" 长度5 > 3 剪枝 ✗
│   ├── "52" → 52 ≤ 255 ✓ ...
│   └── "525"→ 525 > 255 ✗
└── "255"→ path=["255"] start=3
    └── "25" → path=["255","25"] start=5

                "35" → path=["255","25","5","35"] → "255.25.5.35" ✓
```

### 332. 重新安排行程

#### 核心思路

本题是求有向图的欧拉路径（从 "JFK" 出发，走完所有边），要求字典序最小。使用 **Hierholzer 算法**：
1. 对每个节点的邻接表**逆序排序**（使得 `pop()` 取出字典序最小目标）；
2. 用栈模拟 DFS，当某节点无路可走时将其加入结果；
3. 最终逆转结果列表。

时间复杂度：O(E log E)，其中 E 为机票数量。

#### Python 代码

```python
from typing import List
from collections import defaultdict

class Solution:
    def findItinerary(self, tickets: List[List[str]]) -> List[str]:
        graph = defaultdict(list)
        # 逆序排序：pop() 弹出字典序最小目标
        for src, dst in sorted(tickets, reverse=True):
            graph[src].append(dst)

        route = []
        stack = ["JFK"]

        while stack:
            while graph[stack[-1]]:         # 当前机场还有出边
                nxt = graph[stack[-1]].pop()
                stack.append(nxt)
            route.append(stack.pop())       # 死路 → 加入路径

        return route[::-1]                  # 逆转得到正序行程
```

#### 图解

```
tickets = [["MUC","LHR"],["JFK","MUC"],["SFO","SJC"],["LHR","SFO"]]
排序后 graph:
  JFK: ["MUC"]
  LHR: ["SFO"]
  MUC: ["LHR"]
  SFO: ["SJC"]

stack 演化:
  ["JFK"]
  → pop MUC  → ["JFK","MUC"]
  → pop LHR  → ["JFK","MUC","LHR"]
  → pop SFO  → ["JFK","MUC","LHR","SFO"]
  → pop SJC  → ["JFK","MUC","LHR","SFO","SJC"]
  SJC 无出边 → route=["SJC"], stack=["JFK","MUC","LHR","SFO"]
  SFO 无出边 → route=["SJC","SFO"], stack=["JFK","MUC","LHR"]
  ... 依次弹出
route[::-1] = ["JFK","MUC","LHR","SFO","SJC"]
```

### 51. N 皇后

#### 核心思路

逐行放置，每行只放一个皇后，天然保证行不冲突。用三个集合维护已占用列、左上-右下对角线（`row-col` 值相同）、右上-左下对角线（`row+col` 值相同），实现 O(1) 合法性检查。

时间复杂度：O(n!)，空间 O(n)（集合 + 递归栈）。

#### Python 代码

```python
from typing import List

class Solution:
    def solveNQueens(self, n: int) -> List[List[str]]:
        cols  = set()   # 已占用列
        diag1 = set()   # row - col（左上→右下对角线）
        diag2 = set()   # row + col（右上→左下对角线）
        queens = []     # queens[row] = col
        result = []

        def build_board() -> List[str]:
            board = []
            for c in queens:
                board.append('.' * c + 'Q' + '.' * (n - c - 1))
            return board

        def backtrack(row: int):
            if row == n:
                result.append(build_board())
                return
            for col in range(n):
                if col in cols or (row - col) in diag1 or (row + col) in diag2:
                    continue
                cols.add(col)
                diag1.add(row - col)
                diag2.add(row + col)
                queens.append(col)
                backtrack(row + 1)
                queens.pop()
                cols.remove(col)
                diag1.remove(row - col)
                diag2.remove(row + col)

        backtrack(0)
        return result
```

#### 图解

```
n=4，第 0 行尝试各列：

row=0, col=0: cols={0}, diag1={0}, diag2={0}
  row=1, col=0: 0 in cols ✗
  row=1, col=1: (1-1)=0 in diag1 ✗
  row=1, col=2: cols={0,2}, diag1={0,-1}, diag2={0,3}
    row=2, col=0: (2-0)=2? No. (2+0)=2? No. 0 in cols ✗
    row=2, col=1: (2-1)=1? No. (2+1)=3 in diag2 ✗
    row=2, col=3: cols={0,2,3}...
      row=3: 全部冲突 ✗ 回溯
  row=1, col=3: cols={0,3}, diag1={0,-2}, diag2={0,4}
    row=2, col=1: ✓ → row=3, col=2: ✓ → 找到解 [".Q..","...Q","Q...","..Q."]
```

### 37. 解数独

#### 核心思路

找到第一个空格，依次尝试 1-9，检查同行、同列、同 3×3 宫格均无重复。填入后递归解剩余格，若递归返回 `True` 则全局结束；若 1-9 都不合法则回溯（恢复 `.`）并返回 `False`。当找不到空格时说明数独已解完，返回 `True`。

时间复杂度：最坏 O(9^m)，其中 m 为空格数量，实际因剪枝远小于此。

#### Python 代码

```python
from typing import List

class Solution:
    def solveSudoku(self, board: List[List[str]]) -> None:
        def is_valid(row: int, col: int, num: str) -> bool:
            # 检查行
            if num in board[row]:
                return False
            # 检查列
            if any(board[r][col] == num for r in range(9)):
                return False
            # 检查 3×3 宫格
            br, bc = (row // 3) * 3, (col // 3) * 3
            for r in range(br, br + 3):
                for c in range(bc, bc + 3):
                    if board[r][c] == num:
                        return False
            return True

        def backtrack() -> bool:
            for r in range(9):
                for c in range(9):
                    if board[r][c] != '.':
                        continue
                    # 找到第一个空格
                    for num in '123456789':
                        if is_valid(r, c, num):
                            board[r][c] = num
                            if backtrack():
                                return True     # 找到解，立即返回
                            board[r][c] = '.'   # 回溯
                    return False                # 当前空格无合法值
            return True                         # 无空格，数独已解

        backtrack()
```

#### 图解

```
空格搜索顺序（行优先）：
  找到 board[0][0]='.' → 尝试 '1' ~ '9'
    '5': 检查 row[0]、col[0]、box(0,0) → 合法 → 填入
    → 继续找 board[0][1]='.' → ...
    → 若某格 1-9 均不合法 → return False → 上层回溯

关键：return True 一旦触发会沿调用栈一路传回，无需额外 found 标志。

3×3 宫格索引：
  (row//3)*3 给出宫格起始行，(col//3)*3 给出宫格起始列
  row=4,col=7 → br=3, bc=6 → 检查 board[3..5][6..8]
```

## 口诀

- **切割类**：指针到头才收割，段长超界立剪断，前导零和越界值，双重校验不能漏。
- **IP 四段**：剩余字符 ÷ 剩余段 ∈ [1, 3]，段数凑满且用完，两个条件缺一不可。
- **欧拉路**：逆序建图顺序走，死路入栈逆转出，Hierholzer 不枚举。
- **N 皇后**：行天然隔，列对角集合守，三个集合 O(1) 查，逐行推进不回头。
- **解数独**：找空格填数字，行列宫三合一检，填好继续无空收，不合法则退一步。

## 总结与扩展

**五题本质归一**：枚举 + 剪枝 + 回溯。差异只在于枚举空间的组织方式和剪枝条件的精确度。

**剪枝层次**：
1. **合法性剪枝**（最常见）：当前选择违反约束直接跳过，如 IP 前导零、皇后冲突、数独重复。
2. **可行性剪枝**（提前终止分支）：如 93 中剩余字符数不在合法范围，无需继续探索。
3. **贪心剪枝**（排序保证最优）：如 332 中预排序邻接表，使第一条找到的路径即为字典序最小。

**DP 预计算 + 回溯**（131 的做法）是一种通用加速模式：将回溯内部的重复判断提前批量计算，降低每次剪枝的时间代价，适用于切割类问题中"验证子串性质"的场景。

**扩展练习**：
- LeetCode 140（单词拆分 II）：切割 + 词典判断，可类比 131 思路加 DP 预计算；
- LeetCode 52（N 皇后 II）：仅计数，省去 `build_board()` 步骤；
- LeetCode 291（单词规律 II）：双向映射约束，回溯框架不变，约束更复杂。
