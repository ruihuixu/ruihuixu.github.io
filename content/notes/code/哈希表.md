## 识别信号

> 看到什么特征，想到这篇笔记？

- 需要「快速判断某值是否出现过」，不想 $O(n)$ 线性扫描
- 题目问「两数/多数之和是否等于 target」，需要边扫边查补数
- 需要「统计字符或数字的频次」并做比较（异位词、覆盖关系）
- 需要检测「循环/重复状态」，如快乐数反复求平方和

## 核心模板（速查）

### 三种结构一览

```python
# 结构 1：列表计数（Key 范围已知且小，如 a-z）
cnt = [0] * 26
cnt[ord(c) - ord('a')] += 1      # 字符 → 下标映射

# 结构 2：set（只关心存在性/去重）
seen = set()
seen.add(x)
if x in seen: ...

# 结构 3：dict / Counter（需要 Key→Value 映射）
freq = {}
freq[key] = freq.get(key, 0) + 1    # 手动计数

from collections import Counter
freq = Counter(s)                    # 字符串/列表直接计数，缺失键返回 0
```

## ⚠️ 高频错误

**1. 字符统计题无脑用 `dict`，忽略 `list[26]`**

```python
# ❌ 字符范围只有 a-z，用 dict 多余且有哈希开销
from collections import defaultdict
cnt = defaultdict(int)
for c in s:
    cnt[c] += 1

# ✅ Key 范围固定 26 个，直接列表——速度更快，空间 O(1)
cnt = [0] * 26
for c in s:
    cnt[ord(c) - ord('a')] += 1
```

**2. 两数之和用 `set`——无法返回下标**

```python
# ❌ set 只记录"出现过没有"，拿不到下标
seen = set(nums)
# 即使找到 target - x，也不知道它在哪个位置

# ✅ dict 存 value→index，一步到位
table: dict[int, int] = {}   # {数值: 下标}
```

**3. 快乐数：先推进 `n` 再判 `seen`——漏检初始重复**

```python
# ❌ 顺序错误：先 next 再判断，第一次重复值会被跳过
while n != 1:
    n = get_next(n)
    if n in seen:
        return False
    seen.add(n)

# ✅ 先把当前值记录进 seen，再推进到下一个状态
while n != 1 and n not in seen:
    seen.add(n)
    n = get_next(n)
return n == 1
```

**4. 454 四数相加 II 直接四重循环——$O(n^4)$ 超时**

```python
# ❌ 暴力四重循环，n=500 时约 6×10^10 次操作
for a in nums1:
    for b in nums2:
        for c in nums3:
            for d in nums4:
                if a + b + c + d == 0:
                    ans += 1

# ✅ 分治：先存 A+B 的 Counter，再用 -(C+D) 去查，降到 O(n²)
```

## 详解

### 242 有效的字母异位词

#### 核心思路

两个字符串是异位词 ⟺ 每个字符出现的次数完全相同。用长度为 26 的列表统计 `s` 的频次，再逐字母对 `t` 扣减，最后判断是否全零。

#### Python 代码

```python
def isAnagram(s: str, t: str) -> bool:
    if len(s) != len(t):
        return False

    cnt = [0] * 26

    for c in s:
        cnt[ord(c) - ord('a')] += 1    # 统计 s 的字符频次

    for c in t:
        cnt[ord(c) - ord('a')] -= 1    # 用 t 逐步抵消

    return all(x == 0 for x in cnt)    # 完全抵消 → 是异位词
```

#### 图解

```text
s = "anagram"  → cnt: a=3, n=1, g=1, r=1, m=1
t = "nagaram"  → 逐字母 -1 后 cnt: a=0, n=0, g=0, r=0, m=0

全为 0 → True
```

> 时间 $O(n)$，空间 $O(1)$（26 个槽位是常数）

### 383 赎金信

#### 核心思路

判断 `ransomNote` 的每个字母能否被 `magazine` "供应"。与 242 的区别：不要求相等，只要 `magazine` 的频次 ≥ `ransomNote` 即可。出现负数就立刻返回 False，不必扫完。

#### Python 代码

```python
def canConstruct(ransomNote: str, magazine: str) -> bool:
    cnt = [0] * 26

    for c in magazine:
        cnt[ord(c) - ord('a')] += 1    # 建立"库存"

    for c in ransomNote:
        idx = ord(c) - ord('a')
        cnt[idx] -= 1
        if cnt[idx] < 0:               # 库存不足，提前失败
            return False

    return True
```

#### 图解

```text
magazine  = "aab"  → cnt: a=2, b=1
ransomNote = "aa"  → 扣两次 a → cnt[a]=0 ✅
ransomNote = "aab" → 扣完后 cnt[b]=0 ✅
ransomNote = "aabc"→ 扣 c 时 cnt[c]=-1 < 0 → 立即返回 False
```

> 时间 $O(m+n)$，空间 $O(1)$

### 349 两个数组的交集

#### 核心思路

先把 `nums1` 存入 `set`，再遍历 `nums2`：在 set 里的元素即为交集。结果用 `set` 收集，自动去重。

#### Python 代码

```python
def intersection(nums1: list[int], nums2: list[int]) -> list[int]:
    set1 = set(nums1)
    result = set()

    for x in nums2:
        if x in set1:        # O(1) 查找
            result.add(x)    # set 自动去重

    return list(result)
```

#### 图解

```text
nums1 = [4, 9, 5]         → set1 = {4, 9, 5}
nums2 = [9, 4, 9, 8, 4]

遍历 nums2:
  9 in set1 → result = {9}
  4 in set1 → result = {9, 4}
  9 in set1 → result = {9, 4}   （set 自动去重）
  8 not in set1
  4 in set1 → result = {9, 4}

返回 [9, 4]（顺序不定）
```

> 时间 $O(m+n)$，空间 $O(m)$

### 202 快乐数

#### 核心思路

反复对各位数字的平方求和。若最终等于 1 则是快乐数；否则会陷入无限循环。用 `set` 记录出现过的中间值——一旦当前值已在 set 里，说明进入了环。

#### Python 代码

```python
def isHappy(n: int) -> bool:
    def get_next(num: int) -> int:
        total = 0
        while num > 0:
            digit = num % 10
            total += digit * digit
            num //= 10
        return total

    seen = set()
    while n != 1 and n not in seen:
        seen.add(n)          # 先记录当前值，防止下一步重复时漏检
        n = get_next(n)      # 再推进到下一个状态

    return n == 1
```

#### 图解

```text
n=19 → 82 → 68 → 100 → 1 ✅

n=2 → 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4
                                               ↑ seen 检测到 4 已存在
                                               退出循环，n != 1 → False
```

> 时间 $O(\log n)$（循环长度有理论上界），空间 $O(\log n)$

### 1 两数之和

#### 核心思路

题目返回下标，所以不能排序打乱顺序。边扫边查：扫到 `nums[i]` 时，立刻查 `target - nums[i]` 是否已在 dict 里。找到就返回，找不到就把当前数存进去供后续使用。

#### Python 代码

```python
def twoSum(nums: list[int], target: int) -> list[int]:
    table: dict[int, int] = {}    # {数值: 下标}

    for i, x in enumerate(nums):
        need = target - x
        if need in table:
            return [table[need], i]    # 找到补数，立即返回
        table[x] = i                   # 还没找到，先存入备用

    return []
```

#### 图解

```text
nums = [2, 7, 11, 15],  target = 9

i=0, x=2, need=7  → table={}      → 7 不在 → table={2:0}
i=1, x=7, need=2  → table={2:0}   → 2 在！→ 返回 [0, 1]
```

> 时间 $O(n)$，空间 $O(n)$

### 454 四数相加 II

#### 核心思路

四个**独立**数组，不涉及同数组去重。关键认知是**分治**：先枚举所有 A+B 的和并用 Counter 计数，再枚举所有 C+D 的和，查有多少个 `-(C+D)` 在前面的 Counter 里。四重循环拆成两个二重循环，从 $O(n^4)$ 降到 $O(n^2)$。

#### Python 代码

```python
from collections import Counter

def fourSumCount(nums1: list[int], nums2: list[int],
                 nums3: list[int], nums4: list[int]) -> int:
    # 统计 A+B 所有组合的和，Counter 缺失键自动返回 0
    ab_count = Counter(a + b for a in nums1 for b in nums2)

    ans = 0
    for c in nums3:
        for d in nums4:
            ans += ab_count[-(c + d)]    # 查有多少对 A+B 能与 C+D 抵消

    return ans
```

#### 图解

```text
nums1=[1,-2], nums2=[-2,1], nums3=[-1,2], nums4=[2,-1]

A+B 的所有和：
  1+(-2)=-1, 1+1=2, (-2)+(-2)=-4, (-2)+1=-1
→ ab_count = {-1:2, 2:1, -4:1}

枚举 C+D：
  (-1)+2=1    → 查 -1 → 命中 2 次
  (-1)+(-1)=-2 → 查  2 → 命中 1 次
  2+2=4       → 查 -4 → 命中 1 次
  2+(-1)=1    → 查 -1 → 命中 2 次

ans = 2+1+1+2 = 6
```

> 时间 $O(n^2)$，空间 $O(n^2)$（存 A+B 所有组合的计数）

## 统一对比

### 何时用哪种结构

| 场景 | 首选结构 | 核心理由 |
|------|----------|----------|
| Key 是小范围字符/数字（a-z、ASCII 128） | `list[n]` | 无哈希开销，空间 $O(1)$，速度最快 |
| 只判断「出现过没有」/ 去重 | `set` | API 最简洁，$O(1)$ 查询，自动去重 |
| 需要「值 → 下标」映射 | `dict` | Value 可存任意类型 |
| 统计频次，需要对缺失键返回 0 | `Counter` | 省去 `.get(key, 0)` 调用，更简洁 |
| 需要有序遍历 Key | `dict`（Python 3.7+ 插入有序）或 `sorted()` | 注意 `set` 无序，不可依赖遍历顺序 |

### Counter 和 dict 的区别

```python
from collections import Counter

# Counter：缺失键直接返回 0，适合计数查询
freq = Counter("aabbcc")
freq['z']              # → 0，不报错，适合 454 这种直接加的场景

# dict.get：需要显式指定默认值
freq = {}
freq.get('z', 0)       # → 0

# 计数场景二者等价，Counter 写法更紧凑
# dict 在需要存储非整数 Value（如下标）时更通用
```

### 哈希法 vs 排序 + 双指针

| 题型特征 | 典型题 | 推荐方法 | 原因 |
|----------|--------|----------|------|
| 返回下标，数组无序 | 1 两数之和 | `dict` | 排序会打乱下标 |
| 四个独立数组，只问次数 | 454 四数相加 II | `Counter` 分治 | 无组内去重，可拆成两个二重循环 |
| 同一数组选组合，必须去重 | 15 三数之和 / 18 四数之和 | 排序 + 双指针 | 哈希去重逻辑繁琐；排序让重复值连续可直接跳过 |

## 总结与扩展

哈希题写得快、写得稳，靠的不是"会不会用 dict"，而是**选对容器**。Key 范围小就用列表，只要存在性就用 set，需要映射就用 dict/Counter。正确选型后，代码通常极短，复杂度自然降到 $O(n)$；454 的分治写法把四重循环优化到 $O(n^2)$，是同类分组计数题的通用思路。

当题目从"四个独立数组"升级到"同一数组选组合"（15 三数之和 / 18 四数之和），去重用哈希处理会引入大量边界判断，此时应果断切换到**排序 + 双指针**体系。
