## 核心模板（速查）

三题共用"双指针相对运动"框架，各题区别只在指针的**初始位移差**或**移动速度差**：

```python
# ── 19 尺子法 ──────────────────────────────────────────────
dummy = ListNode(0, head)
fast = slow = dummy
for _ in range(n + 1):          # fast 先走 n+1 步 ← 各题唯一区别
    fast = fast.next
while fast:                      # 同步前进直到 fast 走完
    fast, slow = fast.next, slow.next
slow.next = slow.next.next       # slow 在前驱，删除目标节点

# ── 160 拼接法 ─────────────────────────────────────────────
pA, pB = headA, headB
while pA != pB:
    pA = headB if pA is None else pA.next   # 走完本链表切换到对方 ← 区别
    pB = headA if pB is None else pB.next

# ── 142 追及法 ─────────────────────────────────────────────
fast = slow = head
while fast and fast.next:
    fast, slow = fast.next.next, slow.next  # 速度 2:1 ← 区别
    if fast == slow:
        ptr = head
        while ptr != slow:
            ptr, slow = ptr.next, slow.next
        return ptr
```

| 题号 | 指针模式 | 核心原理 | 关键操作 | 是否需要 dummy |
|------|---------|---------|---------|--------------|
| 19   | 先后指针 | 距离差（间距固定 n+1） | `fast` 先走 `n+1` 步 | 是 |
| 160  | 拼接指针 | 路程差（A+B = B+A） | 走完本链表接着走对方链表 | 否 |
| 142  | 快慢指针 | 速度差（2:1 倍率） | 相遇后从头和相遇点同速再走 | 否 |

## ⚠️ 高频错误

**1. 19 题：fast 先走步数搞错（走 n 步而非 n+1 步）**

```python
# ❌ 错误：fast 只走 n 步，slow 最终停在目标节点本身，无法删除自身
for _ in range(n):
    fast = fast.next
while fast.next:            # 循环条件也要跟着改，极易出错
    fast, slow = fast.next, slow.next

# ✅ 正确：fast 走 n+1 步，slow 停在前驱，直接执行 slow.next = slow.next.next
for _ in range(n + 1):
    fast = fast.next
while fast:
    fast, slow = fast.next, slow.next
slow.next = slow.next.next
```

**2. 19 题：忘记 dummy，头节点被删时崩溃**

```python
# ❌ 错误：n 等于链表总长时，slow 需要停在"head 之前"，但没有 dummy 就无处停
fast = slow = head
# ... 当倒数第 n 个恰好是 head 时，无前驱可操作

# ✅ 正确：dummy 充当虚拟前驱，统一处理所有边界情况
dummy = ListNode(0, head)
fast = slow = dummy
# ...
return dummy.next
```

**3. 142 题：循环判断条件顺序写反，短路求值失效**

```python
# ❌ 错误：fast 为 None 时仍尝试访问 fast.next，抛出 AttributeError
while fast.next is not None and fast is not None:
    fast = fast.next.next

# ✅ 正确：先判断 fast 非空，再判断 fast.next（Python 短路求值从左到右）
while fast is not None and fast.next is not None:
    fast = fast.next.next
```

**4. 142 题：第一步循环结束后不加判断直接进入第二步**

```python
# ❌ 错误：无环时循环正常结束，slow 没有定位在相遇点，第二步死循环
while fast is not None and fast.next is not None:
    fast = fast.next.next
    slow = slow.next
ptr = head                   # 无环时也执行到这里，ptr 和 slow 永远不会相等
while ptr != slow:
    ptr, slow = ptr.next, slow.next

# ✅ 正确：相遇判断在第一步循环体内，确认有环才启动第二步
while fast is not None and fast.next is not None:
    fast = fast.next.next
    slow = slow.next
    if fast == slow:         # 有环，相遇点已找到
        ptr = head
        while ptr != slow:
            ptr, slow = ptr.next, slow.next
        return ptr
return None                  # 循环自然结束 → 无环
```

**5. 160 题：在 pA.next 为 None 时跳转，漏掉最后一个节点**

```python
# ❌ 错误：pA 到达末尾节点时就跳走，末尾节点未被参与比较，可能漏掉交点
if pA.next is None:
    pA = headB
else:
    pA = pA.next

# ✅ 正确：pA 走到 None（已越过最后一个节点）才跳转
pA = headB if pA is None else pA.next
```

## 详解

### 19. 删除链表倒数第 N 个节点（尺子法）

#### 核心思路

链表不支持下标，找"倒数第 n 个"通常要扫描两次：第一遍求总长 L，第二遍走 L-n 步定位。双指针只需一次扫描：先让 `fast` 走 `n+1` 步，建立一把长度固定为 `n+1` 的"尺子"，然后两指针同步前进。由于间距始终为 `n+1`，当 `fast` 走到 `None` 时，`slow` 恰好落在被删节点的**前驱**，执行 `slow.next = slow.next.next` 即可完成删除。

走 `n+1` 而非 `n` 步的原因：删除操作需要持有前驱节点，多走一步让 `slow` 指向前驱而非目标本身。

#### Python 代码

```python
from typing import Optional


class ListNode:
    def __init__(self, val: int = 0, next: "Optional[ListNode]" = None):
        self.val = val
        self.next = next


class Solution:
    def removeNthFromEnd(self, head: Optional[ListNode], n: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)   # 虚拟头节点，应对头节点被删的边界情况
        fast = slow = dummy

        # fast 先走 n+1 步，建立长度为 n+1 的"尺子"
        # 多走一步是为了让 slow 最终停在前驱而非目标本身
        for _ in range(n + 1):
            fast = fast.next

        # 两指针同步前进，fast 走到 None 时，slow 在倒数第 n+1 个节点（目标前驱）
        while fast is not None:
            fast = fast.next
            slow = slow.next

        # slow.next 是待删节点，跳过即完成删除
        slow.next = slow.next.next

        return dummy.next
```

**复杂度**：时间 $O(L)$，空间 $O(1)$（L 为链表长度）

#### 图解（1→2→3→4→5，n=2，预期删除节点 4）

```text
初始：dummy → 1 → 2 → 3 → 4 → 5 → None
      ↑
   fast/slow（均在 dummy）

fast 先走 n+1 = 3 步：
      dummy → 1 → 2 → 3 → 4 → 5 → None
      ↑              ↑
     slow            fast     （间距固定为 3）

两指针同步前进，直到 fast 到达 None：
      dummy → 1 → 2 → 3 → 4 → 5 → None
                        ↑              ↑
                    slow（前驱）       fast

执行 slow.next = slow.next.next（删除节点 4）：
      dummy → 1 → 2 → 3 → 5 → None

返回 dummy.next
```

### 160. 相交链表（拼接法）

#### 核心思路

两链表长度不同，从各自头部出发的指针无法自然同步到公共节点。利用恒等式 `len(A) + len(B) = len(B) + len(A)`：让 `pA` 走完 A 后接着走 B，让 `pB` 走完 B 后接着走 A，两者总路程相等、步速相同，必然同时抵达交点；若无交点则同时走到 `None`。

路程拆解（a = A 独有段长，b = B 独有段长，c = 公共段长）：

- `pA` 走：`a + c`（A 全程）+ `b`（B 独有部分）= `a + b + c`
- `pB` 走：`b + c`（B 全程）+ `a`（A 独有部分）= `a + b + c`

两者路程相等，步速相同，必在公共节点（或 None）相遇。

#### Python 代码

```python
class Solution:
    def getIntersectionNode(
        self, headA: ListNode, headB: ListNode
    ) -> Optional[ListNode]:
        pA, pB = headA, headB

        while pA != pB:
            # 走完本链表后立刻切换到对方链表头，消除路程差
            pA = headB if pA is None else pA.next
            pB = headA if pB is None else pB.next

        # pA == pB：有交点时为交点节点，无交点时两者同时走到 None
        return pA
```

**复杂度**：时间 $O(m + n)$，空间 $O(1)$（m、n 分别为两链表长度）

#### 图解（a=2, b=3, c=2）

```text
链表 A：a1 → a2 → c1 → c2 → None
链表 B：b1 → b2 → b3 → c1 → c2 → None
                         ↑ 交点

pA 路径：a1→a2→c1→c2→None→b1→b2→b3→[c1]  （共 7 步）
pB 路径：b1→b2→b3→c1→c2→None→a1→a2→[c1]  （共 7 步）

相遇点：c1（两者同时到达）✓

无交点时：pA 和 pB 各自走完两条链表后同时到达 None，
         while 条件 pA != pB 不成立，返回 None
```

### 142. 环形链表 II（追及法 + 数学推导）

#### 核心思路

**第一步 判断有环**：`fast` 每次走 2 步，`slow` 每次走 1 步。若有环，`fast` 最终一定在环内套圈追上 `slow`（龟兔赛跑）；若无环，`fast` 先走到 `None`，循环自然结束。

**第二步 找入口**：相遇后，从链表头出发一个新指针 `ptr`，与 `slow` 以相同速度同步前进，两者相遇处即为环入口。这依赖下节数学推导出的 `x = z` 结论。

#### 数学推导

设变量：

- `x`：链表头到环入口的距离
- `y`：环入口到相遇点的距离（沿环正向）
- `z`：相遇点沿环正向回到入口的距离

相遇时各自走过的路程：

- `slow` 走了：`x + y`
- `fast` 走了：`x + y + n(y + z)`，n ≥ 1（fast 在环内多绕了 n 圈）

由 `fast` 速度是 `slow` 的 2 倍得：

```text
2(x + y) = x + y + n(y + z)
    x + y = n(y + z)
        x = n(y + z) - y
        x = (n - 1)(y + z) + z
```

当 n = 1（fast 仅多绕一圈）时：**x = z**

**结论**：从链表头走 x 步到入口，从相遇点走 z 步也到入口，两者相等。因此 `ptr`（从头出发）和 `slow`（从相遇点出发）同速前进，必在入口相遇。n > 1 时结论同样成立：slow 在 ptr 到达入口时已额外绕了 n-1 圈，仍停在入口。

#### Python 代码

```python
class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        fast = slow = head

        # 第一步：快慢指针，寻找相遇点
        while fast is not None and fast.next is not None:
            fast = fast.next.next   # 速度 2
            slow = slow.next        # 速度 1

            if fast == slow:        # 有环，找到相遇点
                # 第二步：ptr 从头，slow 从相遇点，同速前进
                # x = z 保证两者恰好在环入口相遇
                ptr = head
                while ptr != slow:
                    ptr = ptr.next
                    slow = slow.next
                return ptr          # 环入口

        return None                 # fast 走到 None，无环
```

**复杂度**：时间 $O(n)$，空间 $O(1)$

#### 图解（head→1→2→3→4→2，入口为节点 2）

```text
链表结构：head → 1 → 2 → 3 → 4
                     ↑_______↑   （环：2 → 3 → 4 → 2）

变量标注：x=1（head→节点2），y=2（节点2→节点4），z=1（节点4→节点2）

第一步：slow 走 3 步，fast 走 6 步，相遇在节点 4
  slow: head → 1 → 2 → 3 → [4]
  fast: head → 1 → 2 → 3 → 4 → 2 → 3 → [4]  （多绕一圈）

  验证：2 × (x+y) = 2×3 = 6 = x+y+(y+z) = 1+2+3 ✓

第二步：ptr 从 head，slow 从相遇点（节点4），各走 1 步（x = z = 1）
  ptr : head → [节点2]
  slow: 节点4 → [节点2]

  相遇点：节点 2（环入口）✓
```

## 统一对比

| 维度 | 19 尺子法 | 160 拼接法 | 142 追及法 |
|------|---------|----------|---------|
| **指针模式** | 先后指针（一快一慢出发） | 等速拼接（同速不同路） | 快慢指针（同路不同速） |
| **核心差异** | 距离差：间距固定为 n+1 | 路程差：A+B = B+A | 速度差：2:1 倍率 |
| **物理类比** | 量布的尺子 | 两人互换跑道 | 操场追及问题 |
| **关键操作** | `fast` 先走 `n+1` 步 | 走完当前链表切换到对方头 | `fast` 每次移动 2 步 |
| **是否需要 dummy** | 是（头节点可能被删） | 否 | 否 |
| **终止条件** | `while fast` | `while pA != pB` | `while fast and fast.next` |
| **时间复杂度** | $O(L)$ | $O(m+n)$ | $O(n)$ |
| **空间复杂度** | $O(1)$ | $O(1)$ | $O(1)$ |

三题的统一抽象：

1. **定义不变量**：两指针之间的相对关系（间距恒为 n+1、路程和恒相等、速度比恒为 2:1）
2. **制造不变量**：通过初始位移差（19）、路径设计（160）或步速差（142）建立"测量工具"
3. **读取结果**：当不变量满足特定终态条件时，指针自动落在目标位置，无需额外计算

## 口诀

```text
【链表双指针三件套】

看到"倒数第 k 个" → 尺子法（19）
  fast 先走 k+1 步    ← 多一步换来前驱位置
  fast/slow 同步走    ← 间距始终固定为 k+1
  fast 到 None 停止   ← slow 此时恰在前驱
  必加 dummy 节点     ← 统一头节点被删的情况

看到"找两链表交点" → 拼接法（160）
  走完 A 立刻走 B     ← 消除路程差
  走完 B 立刻走 A     ← 总路程 A+B = B+A
  相遇即为交点        ← 同时到 None 代表无交点

看到"找环/环入口" → 追及法（142）
  fast 两步 slow 一步 ← 速度 2:1，有环必追上
  追上确认有环        ← 相遇点不是入口
  再用 x = z 结论     ← 从头和相遇点同速再走

【三个核心不变量】
  19  → fast 与 slow 的间距始终为 n+1
  160 → pA 与 pB 走过的总路程始终相等
  142 → fast 的速度始终是 slow 的 2 倍

【Python 细节提醒】
  判空用 is None / is not None，避免 __eq__ 重载陷阱
  快慢同步赋值：fast, slow = fast.next.next, slow.next
```

## 总结与扩展

链表双指针的本质是**相对运动**：链表无法随机访问，用两指针的位移差或速度差制造"隐形尺子"，让终态时指针自动指向目标，省去两次扫描的开销。三题的相对运动方式不同，但框架完全相同——定义不变量，制造不变量，终态读结果。掌握这套物理模型后，所有链表定位变体题均可归入三种模式推导，无需死记每道题的具体实现。

Python 实现中，`is None` 比 `== None` 更严谨；快慢指针同步赋值时 Python 会先求右侧所有值再统一赋左侧，不需要临时变量，也不存在 Java 版本里先赋 `fast` 再用 `fast` 的顺序陷阱。
