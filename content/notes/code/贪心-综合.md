s = list(str(n))

people.sort(key=lambda x: (-x[0], -x[1]))

people.sort(key=lambda x: (-x[0], x[1]))

## 详解

### 406. 根据身高重建队列

#### 核心思路

每个人 `[h, k]` 中，`h` 是身高，`k` 是排在其前面且**身高 ≥ h** 的人数。两个维度相互制约，不能同时贪，必须分步处理：

1. 先按身高**降序**排列（h 相同则 k 升序）：保证后续插入的人始终是当前队列中最矮的
2. 依次把每个人插入到下标 `k` 处：此时队列里只有不矮于他的人，下标 `k` 恰好等于约束要求的前置人数

矮个子后插入会把高个子向后移，但矮个子不被计入高个子的 `k` 统计，已就位的高个子不受影响——这正是"局部正确积累为全局正确"的贪心本质。

#### 图解：插入过程

```
输入：[[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]

排序后（身高降序，k 升序）：
  [7,0]  [7,1]  [6,1]  [5,0]  [5,2]  [4,4]

依次插入到 result[k] 位置：
  插入 [7,0] → [[7,0]]
  插入 [7,1] → [[7,0], [7,1]]
  插入 [6,1] → [[7,0], [6,1], [7,1]]
  插入 [5,0] → [[5,0], [7,0], [6,1], [7,1]]
  插入 [5,2] → [[5,0], [7,0], [5,2], [6,1], [7,1]]
  插入 [4,4] → [[5,0], [7,0], [5,2], [6,1], [4,4], [7,1]]

验证 [4,4]：前面 ≥ 4 的有 [5,0],[7,0],[5,2],[6,1] 共 4 人 ✓
```

#### Python 代码

```python
from typing import List

class Solution:
    def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
        # 身高降序：保证当前人是队列里最矮的，前面每个人都不矮于他
        # h 相同时 k 升序：k 小的先稳定占位，避免同身高者插入时互相干扰
        people.sort(key=lambda x: (-x[0], x[1]))

        result = []
        for p in people:
            # 此时 result 中只有不矮于 p 的人
            # 插到下标 p[1] 处，恰好满足"前面有 p[1] 个不矮于我的人"这一约束
            result.insert(p[1], p)

        return result
```

**复杂度**：排序 O(n log n)；每次 `list.insert` 最坏 O(n)，总体 O(n²)。

### 738. 单调递增的数字

#### 核心思路

要找不超过 `n` 的最大单调不降整数。一旦发现 `s[i-1] > s[i]`（违反单调），局部最优策略：

- `s[i-1]` 减 1：让高位尽量大，同时消除违规
- `s[i]` 及其后全部变为 `9`：低位尽量大

必须**从后向前**遍历：从右往左改时，每次减 1 只可能引发与更左侧位的新违规，而左侧尚未处理，下一轮循环自然覆盖；从左向右改则已处理过的右侧可能出现"回头违规"，需反复扫描。

`mark` 记录"从哪一位起全部置 9"，每次向左发现新违规就更新 `mark`，最后统一置 9。

#### 图解：mark 的移动轨迹

```
n = 332   →   s = ['3', '3', '2'],  mark = 3（末尾之后）

i=2：s[1]='3' > s[2]='2'  →  s[1]='2', mark=2   →  ['3','2','2']
i=1：s[0]='3' > s[1]='2'  →  s[0]='2', mark=1   →  ['2','2','2']

mark=1 起全置 9  →  ['2','9','9']  →  299  ✓

---

n = 10    →   s = ['1', '0'],  mark = 2

i=1：s[0]='1' > s[1]='0'  →  s[0]='0', mark=1   →  ['0','0']

mark=1 起全置 9  →  ['0','9']  →  9  ✓
```

#### Python 代码

```python
class Solution:
    def monotoneIncreasingDigits(self, n: int) -> int:
        s = list(str(n))
        mark = len(s)   # 初始设为末尾之后，意味着暂时不需要置 9

        # 从后向前扫描：每次只处理一对相邻位，不会引发已处理右侧的新违规
        for i in range(len(s) - 1, 0, -1):
            if s[i - 1] > s[i]:
                # 前一位减 1：保住高位尽量大
                s[i - 1] = str(int(s[i - 1]) - 1)
                # 更新 mark：从当前位起所有低位都应置 9
                mark = i

        # 统一将 mark 之后的位全置为 9
        for i in range(mark, len(s)):
            s[i] = '9'

        return int(''.join(s))
```

**复杂度**：O(d)，d 为数字位数（最多约 10 位）；空间 O(d)。

### 968. 监控二叉树

#### 核心思路

摄像头可覆盖自身、父节点和直接子节点三层。叶节点没有子节点，在叶节点装摄像头则向下覆盖完全浪费。贪心策略：**叶节点不装，让叶节点的父节点装**，一个摄像头可覆盖叶节点、父节点、祖父节点三层，效率最高。

要从叶子向根推导决策，必须用**后序遍历（左右中）**，子节点状态先返回给父节点，父节点再据此决定是否装摄像头。

#### 三状态定义

| 返回值 | 含义 | 触发条件 |
|--------|------|---------|
| `0` | 未被覆盖，等待父节点装摄像头 | 两子均已覆盖（left==2 且 right==2） |
| `1` | 本节点装了摄像头 | 任一子节点未覆盖（left==0 或 right==0） |
| `2` | 已被子节点的摄像头覆盖 | 任一子节点有摄像头（left==1 或 right==1） |

空节点返回 `2`（视为已覆盖），使叶节点两个空子均为 2，进入情况 3 返回 `0`，从而触发叶节点的父节点装摄像头。

#### 图解：状态向上传递

```
        A
       / \
      B   null(→2)
     / \
    D   E

空节点 → 2

叶节点 D：left=2(空), right=2(空)  →  情况 3  →  返回 0（未覆盖）
叶节点 E：left=2(空), right=2(空)  →  情况 3  →  返回 0（未覆盖）

节点 B（D、E 的父节点）：
  left=0(D未覆盖)  →  情况 1  →  装摄像头，result=1，返回 1

节点 A（根节点）：
  left=1(B有摄像头)  →  情况 2  →  返回 2（已覆盖）

根节点返回 2，无需额外处理  →  最终答案 1  ✓
```

#### Python 代码

```python
from typing import Optional

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class Solution:
    def minCameraCover(self, root: Optional[TreeNode]) -> int:
        self.result = 0

        def postorder(node: Optional[TreeNode]) -> int:
            # 空节点返回 2（视为已覆盖）
            # 目的：使叶节点两个空子都是 2，叶节点进入情况 3 返回 0
            if node is None:
                return 2

            left = postorder(node.left)    # 先处理左子树，拿到左子状态
            right = postorder(node.right)  # 再处理右子树，拿到右子状态

            # 情况 1：任一子节点未覆盖 → 当前节点必须装摄像头
            # 必须在情况 2 之前判断：left=1, right=0 时应触发情况 1，而非情况 2
            if left == 0 or right == 0:
                self.result += 1
                return 1

            # 情况 2：任一子节点有摄像头 → 当前节点已被子摄像头覆盖
            if left == 1 or right == 1:
                return 2

            # 情况 3：left==2 且 right==2 → 两子均已覆盖，当前节点等待父节点覆盖
            return 0

        # 根节点没有父节点；若根节点返回 0，必须额外在根节点装一个摄像头
        if postorder(root) == 0:
            self.result += 1

        return self.result
```

**复杂度**：O(n) 时间（每节点访问一次）；O(h) 空间（h 为树高，递归栈）。

## 统一对比

| 题号 | 贪心方向 | 核心技巧 | 常见错误 |
|------|----------|---------|---------|
| 406 根据身高重建队列 | 高 → 低 | 先固定身高维度排序，再按 k 值 `insert` | 升序排列；h 相同时 k 方向写反 |
| 738 单调递增的数字 | 后 → 前 | 减 1 + `mark` 记起点 + 补 9 | 从左向右遍历，级联违规 |
| 968 监控二叉树 | 叶 → 根 | 后序遍历 + 三状态 + 根节点特判 | 空节点返 0；漏根特判；情况顺序错 |

**共同规律**：贪心的难点不在"贪什么"，而在"**按什么顺序贪**"。

- 406 的顺序：高到低，高个子先插入队列，矮个子后插入不破坏高个子的约束
- 738 的顺序：后到前，从低位向高位修正，每次改动只向左传播，不回头
- 968 的顺序：叶到根，子节点状态决定父节点行为，后序遍历天然支持这一方向

## 总结与扩展

三题覆盖了贪心在不同场景下的三种典型形态：

- **多维度拆分**（406）：两个维度相互制约时，先排序固定一维，再处理另一维。核心原则是"先确定不变量，再贪变化量"。
- **方向敏感操作**（738）：贪心操作存在依赖关系时，选择能复用上一步结论的遍历方向。从右往左后每次减 1 的影响只向左传播，不会破坏已处理的右侧部分。
- **树形后序贪心**（968）：在树结构上，后序遍历让子节点决策先行，父节点按需跟进，实现全局最优覆盖。三状态的精髓在于"空节点返回 2"这一初始条件的设计，它把贪心起点自然落在叶节点的父层。

**相关扩展题目**：

- LeetCode 135 分发糖果：两次单向贪心（先左向右保证左约束，再右向左保证右约束），与 406 分步固定维度异曲同工
- LeetCode 452 用最少数量的箭引爆气球：区间贪心，按右端点排序后逐一合并，与 738 的"方向选择"问题同类
- LeetCode 1325 删除给定值的叶子节点：树形后序处理，与 968 的后序遍历 + 状态向上传递结构完全一致
