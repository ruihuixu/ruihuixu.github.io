## 核心模板（速查）

```python
# ───── 基础：三指针迭代（206 全局反转） ─────
prev, curr = None, head
while curr:
    nxt = curr.next    # ① 先保存后继，防止改了 curr.next 就断链
    curr.next = prev   # ② 反转：当前节点指向前驱
    prev = curr        # ③ 前驱前进一步
    curr = nxt         # ④ 当前前进一步
return prev            # curr=None 时，prev 就是新头节点

# ───── 进阶：分组反转（25 题通用框架） ─────
while n >= k:
    for _ in range(k):             # 组内：标准三指针，循环 k 次
        nxt = curr.next
        curr.next = prev
        prev, curr = curr, nxt
    # 结束：prev=新组头，curr=下一组起点
    nxt = p0.next                  # ① 暂存原组头（反转后它变成新组尾）
    p0.next.next = curr            # ② 新组尾 → 下一组
    p0.next = prev                 # ③ 前驱 → 新组头
    p0 = nxt                       # ④ p0 移到当前组尾（成为下一组前驱）
    prev = None                    # ⑤ 重置，为下一组做准备
    n -= k
```

| 题号 | 操作范围 | Dummy | 外层循环条件 | 组间重连 |
|------|---------|-------|------------|---------|
| 206  | 全局反转 | 非必须 | `curr is not None` | 无 |
| 24   | 每 2 个 | 必须 | `cur.next and cur.next.next` | 手动 3 步 |
| 25   | 每 k 个 | 必须 | `n >= k` | `p0.next` 技巧 4 步 |

## ⚠️ 高频错误

**1. 先修改指针再保存后继（断链最常见原因）**

```python
# ❌ 错误：curr.next 已指向 prev，nxt 拿到的根本不是后继节点
curr.next = prev
nxt = curr.next      # 此时 curr.next 已经是 prev！后继丢失

# ✅ 正确：先暂存后继，再安全地反转
nxt = curr.next      # 先保存真正的后继
curr.next = prev     # 再修改指针
```

**2. 24 题：循环条件顺序写反（AttributeError 空指针）**

```python
# ❌ 错误：cur.next 可能为 None，访问 None.next 直接抛异常
while cur.next.next and cur.next:
    ...

# ✅ 正确：短路求值必须先判 next，再判 next.next
while cur.next and cur.next.next:
    ...
```

**3. 25 题：先移动 p0 再做组间连接**

```python
# ❌ 错误：先移动 p0，后续 p0.next = prev 操作的是错误节点，前驱断链
p0 = p0.next
p0.next = prev       # 此时 p0 已经是新位置！

# ✅ 正确：先暂存 → 再连接 → 最后才移动 p0
nxt = p0.next        # 暂存原组头
p0.next.next = curr  # 组尾接下一组
p0.next = prev       # 前驱接新组头
p0 = nxt             # 最后才移动 p0
```

**4. 25 题：组间重连后忘记重置 prev**

```python
# ❌ 错误：prev 保留上一组的头节点，下一组组尾会错指上一组
while n >= k:
    for _ in range(k):
        ...
    # 组间重连...
    n -= k
    # 忘了重置 prev！

# ✅ 正确：每次组间重连后重置 prev = None
    p0 = nxt
    prev = None       # 下一组从 None 开始，组尾才能正确接续
    n -= k
```

**5. 返回 curr 而非 prev（206 题）**

```python
# ❌ 错误：循环结束时 curr 已走到 None
return curr

# ✅ 正确：prev 停在原链最后一个节点（新头节点）
return prev
```

## 详解

### 206. 反转链表

#### 核心思路

三指针同时维护三个视角：`prev`（前驱，初始为 `None`）、`curr`（当前节点）、`nxt`（必须在修改 `curr.next` 前保存）。每轮循环只做四件事：保存后继 → 反转指针 → 前驱前进 → 当前前进。

`prev` 初始为 `None` 的原因：原头节点反转后变成尾节点，尾节点的 `next` 必须是 `None`。

#### Python 代码（迭代）

```python
from __future__ import annotations


class ListNode:
    def __init__(self, val: int = 0, next: ListNode | None = None) -> None:
        self.val = val
        self.next = next


def reverseList(head: ListNode | None) -> ListNode | None:
    prev: ListNode | None = None
    curr = head

    while curr is not None:
        nxt = curr.next    # ① 先保存后继，防止断链
        curr.next = prev   # ② 反转：当前节点指向前驱
        prev = curr        # ③ 前驱前进
        curr = nxt         # ④ 当前前进

    # curr=None 时停止，prev 停在原链最后一个节点（新头）
    return prev
```

#### Python 代码（递归，了解）

```python
def reverseList_rec(head: ListNode | None) -> ListNode | None:
    if not head or not head.next:         # 空链或单节点，直接返回
        return head

    new_head = reverseList_rec(head.next) # 先递归反转后续子链
    head.next.next = head                 # 后继节点反指当前节点
    head.next = None                      # 当前节点成为新尾，断开原方向
    return new_head                       # 始终返回原链尾（新链头）
```

#### 图解

```text
初始：None ← prev    1 → 2 → 3 → None
                     curr

Step 1：nxt=2,  1.next=None,  prev=1, curr=2
  None ← 1    2 → 3 → None

Step 2：nxt=3,  2.next=1,    prev=2, curr=3
  None ← 1 ← 2    3 → None

Step 3：nxt=None, 3.next=2,  prev=3, curr=None
  None ← 1 ← 2 ← 3    (curr 已到 None)

返回 prev = 3（新头节点）
```

**复杂度**：时间 O(n)，空间 O(1)（迭代）/ O(n)（递归栈）

### 24. 两两交换链表中的节点

#### 核心思路

两两交换本质是 k=2 的分组反转。头节点在操作后会变（原第 2 个节点变成第 1 个），必须加 Dummy 节点来统一处理，避免对头节点单独特判。

`cur` 始终指向**待交换一对的前驱节点**，每轮交换分三步：

1. `cur.next = node2`：前驱跳过 node1 直接指向 node2
2. `node2.next = node1`：完成两节点的反转
3. `node1.next = nxt`：node1 接续后方链表

#### Python 代码

```python
def swapPairs(head: ListNode | None) -> ListNode | None:
    dummy = ListNode(-1)
    dummy.next = head
    cur = dummy

    while cur.next and cur.next.next:          # 剩余至少 2 个节点才能交换
        node1 = cur.next
        node2 = cur.next.next
        nxt = node2.next                        # 暂存下一对的起点，防止断链

        cur.next = node2                        # 前驱 → node2（跨越 node1）
        node2.next = node1                      # node2 反指 node1
        node1.next = nxt                        # node1 接续后方

        cur = node1                             # node1 是这对的尾，也是下对的前驱

    return dummy.next
```

#### 图解

```text
初始：dummy → 1 → 2 → 3 → 4 → None
      cur

第 1 对（node1=1, node2=2, nxt=3）：
  cur.next  = 2  →  dummy → 2 ...
  node2.next = 1  →  2 → 1 ...
  node1.next = 3  →  1 → 3 → 4 → None
  cur = 1
  → dummy → 2 → 1 → 3 → 4 → None
                 cur

第 2 对（node1=3, node2=4, nxt=None）：
  cur.next  = 4  →  1 → 4 ...
  node2.next = 3  →  4 → 3 ...
  node1.next = None
  cur = 3
  → dummy → 2 → 1 → 4 → 3 → None
                          cur

cur.next = None，退出循环
返回 dummy.next = 2
```

**复杂度**：时间 O(n)，空间 O(1)

### 25. K 个一组翻转链表

#### 核心思路

这是前两题的通用抽象，206（k = 链表长度）和 24（k = 2）都是它的特例。

算法分三层：

- **分组**：先遍历计数得总长度 n，外层循环 `n >= k` 控制是否还有完整的组可处理
- **组内反转**：内层标准三指针迭代，循环恰好 k 次；结束时 `prev` 是新组头，`curr` 是下一组首节点
- **组间拼接**：`p0.next` 在反转前是原组头，反转后变成新组尾——利用这个身份变化完成 4 步重连，无需额外变量追踪组尾

#### Python 代码

```python
def reverseKGroup(head: ListNode | None, k: int) -> ListNode | None:
    # 先统计链表总长度，用于控制外层循环
    n, node = 0, head
    while node:
        n += 1
        node = node.next

    dummy = ListNode(0)
    dummy.next = head
    p0 = dummy               # p0：当前组的前驱节点，始终是"已完成组的尾"
    prev: ListNode | None = None
    curr = head

    while n >= k:
        # ── 组内反转：标准三指针，执行 k 次 ──────────────
        for _ in range(k):
            nxt = curr.next
            curr.next = prev
            prev = curr
            curr = nxt
        # 结束时：prev = 新组头，curr = 下一组起点

        # ── 组间拼接：4 步重连 ────────────────────────────
        nxt = p0.next            # ① 暂存原组头（反转后它是新组尾）
        p0.next.next = curr      # ② 新组尾 → 下一组
        p0.next = prev           # ③ 上组尾（前驱） → 新组头
        p0 = nxt                 # ④ p0 移到新组尾（成为下一组的前驱）
        prev = None              # 重置 prev，下一组从 None 出发
        n -= k

    return dummy.next
```

#### 图解

```text
输入：1 → 2 → 3 → 4 → 5，k=2，n=5
初始：dummy → 1 → 2 → 3 → 4 → 5
      p0=dummy, prev=None, curr=1

──── 第 1 组（反转节点 1, 2）────
内层 2 次后：2 → 1 → None，prev=2，curr=3

4 步拼接：
  nxt = p0.next = 1          (dummy.next 当前是 1)
  1.next = 3                  (原组头 1 变组尾，接下一组 3)
  dummy.next = 2              (前驱 dummy 接新组头 2)
  p0 = 1                      (p0 移到组尾 1)

结果：dummy → 2 → 1 → 3 → 4 → 5
                    p0=1, prev→None, curr=3

──── 第 2 组（反转节点 3, 4）────
内层 2 次后：4 → 3 → None，prev=4，curr=5

4 步拼接：
  nxt = p0.next = 3          (p0=1，1.next=3)
  3.next = 5                  (原组头 3 变组尾，接尾节点 5)
  1.next = 4                  (前驱 1 接新组头 4)
  p0 = 3                      (p0 移到组尾 3)

结果：dummy → 2 → 1 → 4 → 3 → 5
                         p0=3, curr=5

──── n=1 < k=2，外层循环结束 ────
最终：2 → 1 → 4 → 3 → 5（节点 5 不足一组，保持原顺序）
```

**复杂度**：时间 O(n)（计数一遍 + 分组反转一遍），空间 O(1)

## 统一对比

| 维度 | 206 反转链表 | 24 两两交换 | 25 K 组翻转 |
|------|------------|------------|------------|
| **抽象层次** | 基础模板 | k=2 特例 | 通用抽象 |
| **Dummy 节点** | 非必须 | 必须 | 必须 |
| **外层循环** | `curr is not None` | `cur.next and cur.next.next` | `n >= k` |
| **组间重连** | 无 | 手动 3 步 | `p0.next` 技巧 4 步 |
| **尾段处理** | 无尾段概念 | 奇数时最后 1 个自动保留 | `n < k` 自动停止 |
| **时间复杂度** | O(n) | O(n) | O(n) |
| **空间复杂度** | O(1) | O(1) | O(1) |

三道题共享同一个**原子操作**，差异只在于是否分组、是否需要 Dummy，以及如何处理组间连接：

```text
① nxt = curr.next    保存后继（断链前的保险）
② curr.next = prev   反转指针（核心操作）
③ prev = curr        前驱前进
④ curr = nxt         当前前进
```

分组题额外增加一步：将反转好的子链**插回原链表的正确位置**（组间拼接）。

## 口诀

```text
【链表反转三部曲口诀】

● 基础（206）：三指针舞蹈
  prev=None 打底，curr 从头出发
  先存 nxt 防断链，再翻 curr.next 方向
  前驱与当前齐步走，curr=None 时 prev 是新头

● 进阶（24）：Dummy + 两两结对
  头节点会变必须加 Dummy
  先判 cur.next 再判 cur.next.next，顺序不能倒
  node2 跨越 node1，node1 接续后方，cur 移到 node1

● 通用（25）：计数 → 分组 → 拼接
  先数总长 n，外层 n≥k 继续，每组 n-=k 递减
  内层 k 次三指针，结束时 prev=新组头，curr=下一组起
  4 步拼接：暂存原头 → 组尾接下组 → 前驱接新头 → p0 移组尾
  每轮 prev=None 重置，为下一组清零

【核心不变量】
  每轮结束后，已处理部分指针方向正确，未处理部分保持原样
  p0 始终是"已完成组的尾节点"，是下一组对外的唯一入口
  p0.next 的身份变化是精髓：反转前=原组头，反转后=新组尾
```

## 总结与扩展

这三道题构成链表反转的完整升级路径：206 建立三指针迭代的肌肉记忆，24 引入 Dummy 和分组拼接（k=2 特例），25 完成通用抽象。理解了 25 题的"分组 + 组内反转 + 组间拼接"三层结构后，206 和 24 都自然退化为特例，链表反转类的所有变体都能用这套框架推导出来。

做链表指针题的铁律：**先画图，先保存，再修改**。头节点可能变化时无条件加 Dummy；组间连接时先暂存原组头（`nxt = p0.next`），再安全操作，最后才移动 `p0`。
