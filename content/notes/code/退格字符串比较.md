## 识别信号

> 看到什么特征，想到这篇笔记？

- 字符串中含有特殊符号（`#`）表示"删除前一个字符"，要求比较两个处理后的结果
- 追求 **O(1) 空间**，不允许使用辅助栈
- 退格只影响当前字符**左侧**的字符，正向模拟需要"回头"，而逆向扫描天然匹配
- 题目出现"退格""删除""消除"等字样，且删除方向向左

## 核心模板（速查）

```python
def backspace_compare(s: str, t: str) -> bool:
    i, j = len(s) - 1, len(t) - 1
    skip_s = skip_t = 0

    while i >= 0 or j >= 0:
        while i >= 0:               # ① 找 s 的下一个有效字符
            if s[i] == '#':
                skip_s += 1         # 获得一枚退格令牌
                i -= 1
            elif skip_s > 0:
                skip_s -= 1         # 消耗令牌，此字符被删
                i -= 1
            else:
                break               # skip_s==0 且非 '#'，有效字符

        while j >= 0:               # ② 同理处理 t
            if t[j] == '#':
                skip_t += 1
                j -= 1
            elif skip_t > 0:
                skip_t -= 1
                j -= 1
            else:
                break

        cs = s[i] if i >= 0 else '' # ③ 越界时用空串做哨兵
        ct = t[j] if j >= 0 else ''
        if cs != ct:
            return False            # ④ 字符不同，立即返回

        i -= 1                      # ⑤ 消费有效字符，继续左移
        j -= 1

    return True
```

**关键变量一览**

| 变量 | 含义 | 变化规则 |
|------|------|---------|
| `i / j` | 当前扫描位置（从末尾向左） | 单调递减，最终到 -1 |
| `skip_s / skip_t` | 待消耗的退格令牌数 | 遇 `#` 加一；遇普通字符减一（若 > 0） |
| `cs / ct` | 当前轮次的有效字符 | `i < 0` 时用 `''` 代表"已耗尽" |

## ⚠️ 高频错误

**1. 外层循环条件用 `and` 而非 `or`**

```python
# ❌ 错误：一方耗尽就停，会漏掉另一方剩余的有效字符
while i >= 0 and j >= 0:
    ...

# ✅ 正确：只要任意一方还有字符就继续扫描
while i >= 0 or j >= 0:
    ...
```

**2. 越界时直接取字符，触发 Python 负索引**

```python
# ❌ 错误：i=-1 时 s[-1] 是最后一个字符，逻辑完全错误
cs = s[i]

# ✅ 正确：越界时返回空串，与任何有效字符都不相等
cs = s[i] if i >= 0 else ''
```

**3. 比较后忘记移动指针，导致死循环**

```python
# ❌ 错误：i、j 停在有效字符上，下轮内层 while 立即 break，无限重复
if cs != ct:
    return False
# 漏掉了 i -= 1 和 j -= 1

# ✅ 正确：无论是否相等，都必须移动指针消费这对有效字符
if cs != ct:
    return False
i -= 1
j -= 1
```

**4. 两个字符串共用一个 skip 变量**

```python
# ❌ 错误：s 的退格令牌污染 t 的扫描
skip = 0
# 处理 s 时 skip 被修改 → 处理 t 时读到的值已错误

# ✅ 正确：两个字符串独立维护各自的计数器
skip_s = 0
skip_t = 0
```

**5. 内层循环缺少边界保护**

```python
# ❌ 错误：Python 中 i=-1 时 s[-1] 合法但语义错，且可能无限循环
while True:
    if s[i] == '#':
        skip_s += 1; i -= 1
    ...

# ✅ 正确：内层 while 必须同时检查下标边界
while i >= 0:
    if s[i] == '#':
        skip_s += 1; i -= 1
    ...
```

## 详解

### 方法一：栈模拟（直觉解法，O(n) 空间）

#### 核心思路

从左到右扫描字符串，维护一个栈：

- 普通字符：入栈
- `#`：若栈非空则弹出栈顶（退格），空栈时忽略

最终将两个字符串各自还原为最终状态，比较是否相等。

#### Python 代码

```python
def backspace_compare(s: str, t: str) -> bool:
    def build(string: str) -> str:
        stack: list[str] = []
        for c in string:
            if c != '#':
                stack.append(c)       # 普通字符，入栈等待
            elif stack:
                stack.pop()           # 退格，删除上一个字符
            # 空栈时遇到 '#' 直接忽略
        return ''.join(stack)

    return build(s) == build(t)
```

#### 复杂度

- 时间：$O(N + M)$，各扫描一遍
- 空间：$O(N + M)$，栈最坏情况存储整个字符串

### 方法二：逆向双指针（最优解，O(1) 空间）

#### 核心思路

退格是"向左删"，所以**从右往左扫描**天然契合：

- 正向扫描时，遇到 `#` 需要"回头"删掉之前入栈的字符
- 逆向扫描时，`#` 出现的位置总是在它所删字符的右侧，扫到 `#` 时只需记下"还要跳过多少个字符"

`skip` 计数器本质上是一个"退格令牌池"：

| 事件 | 动作 |
|------|------|
| 遇到 `#` | `skip += 1`，获得令牌 |
| 遇到普通字符且 `skip > 0` | `skip -= 1`，消耗令牌，此字符被删 |
| 遇到普通字符且 `skip == 0` | 令牌耗尽，这是真正的有效字符 |

两个字符串各自找到下一个有效字符后比较；任何一方越界则用空串 `''` 作为哨兵。

#### Python 代码

```python
def backspace_compare(s: str, t: str) -> bool:
    i, j = len(s) - 1, len(t) - 1
    skip_s, skip_t = 0, 0

    while i >= 0 or j >= 0:

        # ── 找 s 的下一个有效字符 ──
        while i >= 0:
            if s[i] == '#':
                skip_s += 1           # 收到退格令牌
                i -= 1
            elif skip_s > 0:
                skip_s -= 1           # 当前字符被退格消耗，跳过
                i -= 1
            else:
                break                 # s[i] 是真正的有效字符

        # ── 找 t 的下一个有效字符 ──
        while j >= 0:
            if t[j] == '#':
                skip_t += 1
                j -= 1
            elif skip_t > 0:
                skip_t -= 1
                j -= 1
            else:
                break

        # ── 取有效字符（越界时用空串） ──
        cs = s[i] if i >= 0 else ''
        ct = t[j] if j >= 0 else ''

        if cs != ct:
            return False              # 对应位置字符不同，直接返回

        i -= 1                        # 消费这对有效字符，继续向左
        j -= 1

    return True
```

#### 图解（以 s = "ab##", t = "c#d#" 为例）

```text
s = "a  b  #  #"
      0  1  2  3

逆向扫描 s：
  i=3: s[3]='#'  → skip_s=1, i=2
  i=2: s[2]='#'  → skip_s=2, i=1
  i=1: s[1]='b'  skip_s=2>0 → 被退格，skip_s=1, i=0
  i=0: s[0]='a'  skip_s=1>0 → 被退格，skip_s=0, i=-1
  结果：i=-1，s 无有效字符 → cs=''

t = "c  #  d  #"
      0  1  2  3

逆向扫描 t：
  j=3: t[3]='#'  → skip_t=1, j=2
  j=2: t[2]='d'  skip_t=1>0 → 被退格，skip_t=0, j=1
  j=1: t[1]='#'  → skip_t=1, j=0
  j=0: t[0]='c'  skip_t=1>0 → 被退格，skip_t=0, j=-1
  结果：j=-1，t 无有效字符 → ct=''

比较：cs='' == ct='' ✓  →  i=-2, j=-2
外层循环条件 i>=0 or j>=0 不满足，退出，return True
```

另一个例子（s = "a#c", t = "b" → False）：

```text
s = "a  #  c"      t = "b"
      0  1  2            0

逆向扫描 s：
  i=2: s[2]='c'  skip_s=0 → 有效字符，break
  cs = 'c'

逆向扫描 t：
  j=0: t[0]='b'  skip_t=0 → 有效字符，break
  ct = 'b'

比较：cs='c' ≠ ct='b'  →  return False
```

#### 复杂度

- 时间：$O(N + M)$，每个字符最多被访问一次（指针单调递减）
- 空间：$O(1)$，只用 `i, j, skip_s, skip_t` 四个整数

## 统一对比

| 维度 | 方法一：栈模拟 | 方法二：逆向双指针 |
|------|--------------|------------------|
| 扫描方向 | 从左往右 | 从右往左 |
| 辅助结构 | 栈（Python list） | `skip` 计数器（整数） |
| 时间复杂度 | $O(N+M)$ | $O(N+M)$ |
| 空间复杂度 | $O(N+M)$ | $O(1)$ |
| 代码量 | ~10 行，极简 | ~25 行，需处理边界 |
| 理解难度 | 低，直接模拟 | 中，需要理解令牌语义 |
| 面试推荐场景 | 快速出解、验证思路 | 追求空间最优时使用 |

**选择建议**：
- 只要求正确结果：方法一，代码清晰，出错率低
- 面试题要求 O(1) 空间：方法二，并能清晰解释 skip 令牌的语义

## 口诀

```text
【退格字符串口诀】

正向用栈空间费，逆向计数最经济。

遇到井号拿令牌（skip++），
遇到字母看余额：
  余额大于零 → 消耗（skip--），此字符被删，继续左移；
  余额等于零 → 有效，停下来等待比较。

外层 or 保证扫完，
越界取空串做哨兵，
比完记得双指针左移，
两串独立 skip 莫混淆。

【决策树】
  退格/删除 + O(1) 空间  →  逆向扫描 + skip 计数器
  只要求结果正确         →  栈模拟，简单可靠
  看到 '#' 表示消除      →  先考虑逆向，再看空间约束
```

## 总结与扩展

这道题的精髓在于**用 skip 计数器替代栈**，将空间从 O(n) 压缩到 O(1)。核心洞察：退格是"向左删"，逆向扫描时 `#` 的影响对象自然落在其左侧，配合令牌计数即可精确跳过被删字符，无需真正维护栈结构。

从"栈模拟"到"逆向计数器"的思维跳跃是这道题的考察重点——能否在 O(1) 空间约束下找到等价的信息表示方式。
