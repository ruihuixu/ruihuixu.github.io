## 识别信号

> 看到什么特征，想到这篇笔记？

- 题目要求**原地修改**字符串或字符数组（in-place，O(1) 额外空间）
- 操作是**反转**某段区间，或把某类字符**替换/扩展**成更长的序列
- 字符串按**固定周期**（如每 2k 个字符）切段，只处理每段的某一部分
- 暴力做法会因**元素搬移**导致 O(N²)，需要找到只扫描一次的方向

## 核心模板（速查）

```python
# ---- 模板 A：对撞指针反转 ----
left, right = 0, len(s) - 1
while left < right:
    s[left], s[right] = s[right], s[left]
    left += 1
    right -= 1
```

```python
# ---- 模板 B：倒序双指针填充（扩容场景）----
# 先统计需要扩展的字符数，再从后向前写，每个字符只移动一次
count = sum(1 for c in s if c == ' ')
chars = list(s) + [''] * (count * 2)  # 净增 count * 2 个位置

left = len(s) - 1          # 指向原字符串末尾
right = len(chars) - 1     # 指向新数组末尾

while left >= 0:
    if chars[left] == ' ':
        chars[right - 2] = '%'   # 倒序：先写 '%'，再写 '2'，再写 '0'
        chars[right - 1] = '2'
        chars[right]     = '0'
        right -= 3
    else:
        chars[right] = chars[left]
        right -= 1
    left -= 1
```

```python
# ---- 模板 C：步长跳跃反转 ----
chars = list(s)
n = len(chars)
for i in range(0, n, 2 * k):      # 步长直接设为 2k，省去计数器
    left  = i
    right = min(i + k - 1, n - 1) # min 统一处理末尾不足 k 个的情况
    while left < right:
        chars[left], chars[right] = chars[right], chars[left]
        left  += 1
        right -= 1
```

**各题对比速查：**

| 题号 | 模板 | 指针运动方向 | 核心难点 |
|------|------|-------------|---------|
| 344  | A（对撞） | L→ 和 ←R，向中间靠拢 | 终止条件 `left < right` |
| LCR 122 | B（倒序填充） | L← 和 R←，同向向左 | 扩容量 = `count * 2`，不是 `count * 3` |
| 541  | C（步长跳跃） | i += 2k，跨区间 | `min(i+k-1, n-1)` 防越界 |

## ⚠️ 高频错误

**1. 541 题右边界没有用 `min`，直接写 `i + k - 1`**

```python
# ❌ 错误：最后一组字符不足 k 个时，i + k - 1 超出数组范围
right = i + k - 1

# ✅ 正确：min 自动处理"剩余不足 k 个"的情况
right = min(i + k - 1, len(chars) - 1)
```

**2. LCR 122 扩容量算成 `count * 3`，把原空格也算了进去**

```python
# ❌ 错误：空格原来占 1 位，替换为 3 位，净增 2 位；count * 3 多分配了空间
chars = list(s) + [''] * (count * 3)

# ✅ 正确：每个空格净增 2 个位置
chars = list(s) + [''] * (count * 2)
```

**3. LCR 122 从前向后遍历，每次遇到空格都整体右移后续元素**

```python
# ❌ 错误：逐字符前插替换，每次右移 O(N)，总计 O(N²)
result = ""
for ch in s:
    result += '%20' if ch == ' ' else ch  # 字符串拼接同样有隐性 O(N) 开销

# ✅ 正确：倒序填充，每个字符只写入一次，总计 O(N)
# 见模板 B
```

**4. 344 终止条件写成 `left <= right`，中间元素与自身交换**

```python
# ❌ 错误：奇数长度时，left == right 时中间元素自交换（结果无误，但逻辑不清晰）
# 偶数长度无问题，但在变体题中 off-by-one 风险增加
while left <= right:
    ...

# ✅ 正确：严格小于，相遇即停
while left < right:
    ...
```

## 详解

### 344. 反转字符串

#### 核心思路

对撞指针：`left` 从头向右，`right` 从尾向左，每次交换两端元素，直到相遇。

**核心不变量**：`[left, right]` 是尚未完成反转的区域，每次交换后区间收缩一格。

#### Python 代码

```python
def reverseString(s: list[str]) -> None:
    """
    原地反转字符列表，O(1) 额外空间。
    s 是 list[str]，Python 中不需要 temp 变量，直接元组解包交换。
    """
    left, right = 0, len(s) - 1

    # 核心不变量：[left, right] 是尚未反转的区间
    while left < right:
        s[left], s[right] = s[right], s[left]  # Python 无需 temp
        left  += 1
        right -= 1
```

**复杂度**：时间 O(N)，空间 O(1)

#### 图解

```text
初始：  ['h', 'e', 'l', 'l', 'o']
         L                   R

第 1 步：交换 s[0] 和 s[4]
        ['o', 'e', 'l', 'l', 'h']
               L         R

第 2 步：交换 s[1] 和 s[3]
        ['o', 'l', 'l', 'e', 'h']
                   LR
        （left == right，中间元素无需处理，停止）

结果：  ['o', 'l', 'l', 'e', 'h']
```

### LCR 122. 路径加密（替换空格）

#### 核心思路

将每个空格替换为 `%20`，每个空格从 1 个字符膨胀到 3 个，需要扩容。

**为什么不能从前向后填充**：从头遍历遇到空格时，后续所有字符都要整体右移 2 位。N 个空格 → N 次整体右移 → O(N²)。

**倒序方案**：先统计空格数、计算新长度；`left` 指向原末尾，`right` 指向新末尾，同步向左写入。每个字符只被写入一次，时间 O(N)。

**物理直觉**：像整理书架，先把架子加长，从最后一本书开始倒序搬运，后面的书挪到位之后前面才跟上，不会发生"推挤"。

#### Python 代码

```python
def replaceSpace(s: str) -> str:
    count = s.count(' ')
    if count == 0:
        return s

    # 扩容：Python 字符串不可变，转 list 后追加占位符
    # 每个空格净增 2 个位置（1→3），追加 count * 2 个空字符串
    chars = list(s) + [''] * (count * 2)

    left  = len(s) - 1          # 指向原字符串末尾（扩容前的最后一位）
    right = len(chars) - 1      # 指向新数组末尾

    while left >= 0:
        if chars[left] == ' ':
            # 倒序写入 '%20'：right 位置写 '0'，前一位写 '2'，再前一位写 '%'
            chars[right]     = '0'
            chars[right - 1] = '2'
            chars[right - 2] = '%'
            right -= 3          # 消耗了 3 个新位置
        else:
            chars[right] = chars[left]  # 普通字符：平移到新位置
            right -= 1
        left -= 1               # 无论哪种情况，原指针都前移 1 步

    return ''.join(chars)
```

**复杂度**：时间 O(N)，空间 O(N)（Python 字符串不可变，必须构造新列表）

#### 图解

```text
输入：s = "hi there"（1 个空格），count=1，扩容 2 位

扩容后 chars：
  ['h','i',' ','t','h','e','r','e','','']
                                    L=7  R=9

left=7 'e'（非空格）→ chars[9]='e'，  left=6, right=8
left=6 'r'（非空格）→ chars[8]='r'，  left=5, right=7
left=5 'e'（非空格）→ chars[7]='e'，  left=4, right=6
left=4 'h'（非空格）→ chars[6]='h'，  left=3, right=5
left=3 't'（非空格）→ chars[5]='t'，  left=2, right=4

left=2 ' '（空格）→ chars[4]='0', chars[3]='2', chars[2]='%'
                      left=1, right=1

left=1 'i'（非空格）→ chars[1]='i'，  left=0, right=0
left=0 'h'（非空格）→ chars[0]='h'，  left=-1，循环结束

结果：'hi%20there'
```

### 541. 反转字符串 II

#### 核心思路

每 2k 个字符为一组，只反转每组的**前 k 个**字符。

**关键洞察**：不需要计数器标记当前在第几段，直接让循环步长等于 2k，每次 `i` 自然跳到下一组起点，再对 `[i, i+k-1]` 段做对撞反转即可。

**边界保护**：最后一组可能不足 k 个，右边界用 `min(i+k-1, n-1)` 统一处理。

#### Python 代码

```python
def reverseStr(s: str, k: int) -> str:
    chars = list(s)
    n = len(chars)

    # 步长直接设为 2k：i 每次跳到下一组起点，省去手动计数
    for i in range(0, n, 2 * k):
        left  = i
        right = min(i + k - 1, n - 1)  # 不足 k 个时，反转剩余所有字符

        # 复用对撞反转逻辑（同 344）
        while left < right:
            chars[left], chars[right] = chars[right], chars[left]
            left  += 1
            right -= 1

    return ''.join(chars)
```

**复杂度**：时间 O(N)，空间 O(N)

#### 图解

```text
输入：s = "abcdefg"，k = 2

初始：  ['a','b','c','d','e','f','g']

i=0：反转 [0, 1]（前 k=2 个）
        ['b','a','c','d','e','f','g']
         ←→

i=4：反转 [4, 5]（前 k=2 个，第三组）
        ['b','a','c','d','f','e','g']
                        ←→

i=8：8 >= n=7，range 结束

结果：  "bacdfeg"

注意：i=2 时反转 [2, 3]，这是第二组的后 k 个，不需要反转，跳过。
```

## 统一对比

| 题号 | 核心模型 | 指针运动方式 | 核心难点 | Python 关键语句 |
|------|----------|-------------|---------|----------------|
| 344 | 对撞反转 | L→ ←R，向中间 | 终止 `L < R` | `s[l], s[r] = s[r], s[l]` |
| LCR 122 | 扩容倒序 | L← R←，同向向左 | 扩容量 `count*2` | `chars = list(s) + ['']*count*2` |
| 541 | 步长跳跃 | i+=2k，跨组 | `min(i+k-1, n-1)` | `range(0, n, 2*k)` |

**抽象规律**：字符串原地操作的本质是对**下标（Index）的精细控制**：

- 344 控制**值**（两端元素对调）
- LCR 122 控制**容量**（先扩后写，避免右移堆积）
- 541 控制**步长**（跳跃循环，消灭计数器）

## 总结与扩展

这三道题建立了字符串原地操作的三个基础模型。它们本质上都是双指针的变体，差别只在于指针的移动方向与步长：**对撞**（两端向中间）、**同向倒序**（两指针均从右向左）、**跳跃**（按固定周期推进）。

掌握这套框架后，遇到字符串题的第一反应应该是：「这个操作用哪种指针运动可以实现，从哪个方向扫描可以让每个字符只被访问一次？」而不是立即求助于库函数。
