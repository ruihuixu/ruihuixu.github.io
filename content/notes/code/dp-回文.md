## 识别信号

> 看到什么特征，想到这篇笔记？

- 题目围绕「回文」展开，要求统计数量或求最长长度
- 操作对象是字符串的某个**连续区间** `[i, j]`，且答案依赖更短区间的结果
- 关键转移：`s[i] == s[j]` 时向内缩一圈，即需要 `dp[i+1][j-1]` 的结果
- 出现「子串（连续）」或「子序列（不连续）」字样，两者转移公式不同

## 核心模板（速查）

两题共用同一套**区间 DP + 对角线填充**框架，差异只在 dp 值类型和不匹配时的处理。

### 区间 DP 通用骨架

```python
n = len(s)
dp = [[初始值] * n for _ in range(n)]
# 对角线初始化（视题目而定）

# 对角线填充：i 从大到小，j 从 i 开始向右
for i in range(n - 1, -1, -1):      # ← 从下往上（保证 dp[i+1] 已算好）
    for j in range(i, n):            # ← 从左往右（保证 dp[j-1] 已算好）
        if s[i] == s[j]:
            # 两题在此分叉（见下表）
            ...
        else:
            # 两题在此分叉（见下表）
            ...
```

### 两题核心差异速查

| 题号 | dp 含义 | 初始化 | s[i]==s[j] 时 | s[i]!=s[j] 时 | 结果取法 |
|------|---------|--------|---------------|---------------|---------|
| 647 | `bool`，`s[i..j]` 是否回文 | 全 `False` | `j-i<=1` → `True`；否则 `dp[i+1][j-1]` | 保持 `False` | 累计 `True` 的总数 |
| 516 | `int`，最长回文子序列长度 | 对角线 `dp[i][i]=1`，其余 `0` | `dp[i+1][j-1] + 2` | `max(dp[i+1][j], dp[i][j-1])` | `dp[0][n-1]` |

### 对角线填充方向说明

```text
       j=0  j=1  j=2  j=3
i=0  [  .    .    .    .  ]   ← 最后才填（最长区间）
i=1  [       .    .    .  ]
i=2  [            .    .  ]
i=3  [                 .  ]   ← 最先填（对角线，长度=1）

填表方向：i 从底部往上，j 在每行从 i 向右。
当填 dp[i][j] 时，dp[i+1][j-1]（左下角）已经算好。
若 i 从上往下遍历，左下角尚未计算，结果必错。
```

## ⚠️ 高频错误

**1. 遍历方向写反：i 从 0 到 n-1（从上往下）**

这是最常见的错误，导致依赖的 `dp[i+1][j-1]` 尚未计算。

```python
# ❌ 错误：i 从小到大，dp[i+1][j-1] 此时还是初始值
for i in range(n):
    for j in range(i, n):
        if s[i] == s[j] and j - i >= 2:
            dp[i][j] = dp[i + 1][j - 1]  # dp[i+1][j-1] 未计算！

# ✅ 正确：i 从大到小，保证下方行已经填好
for i in range(n - 1, -1, -1):
    for j in range(i, n):
        if s[i] == s[j] and j - i >= 2:
            dp[i][j] = dp[i + 1][j - 1]  # 左下角已就绪
```

**2. 516 忘记初始化对角线 `dp[i][i] = 1`**

单个字符本身就是长度为 1 的回文子序列，不初始化会让所有结果偏小。

```python
# ❌ 错误：全部初始化为 0，单字符情况丢失
dp = [[0] * n for _ in range(n)]

# ✅ 正确：对角线置 1
dp = [[0] * n for _ in range(n)]
for i in range(n):
    dp[i][i] = 1  # 单字符回文子序列长度为 1
```

**3. 516 不匹配时误写为 0 或直接跳过**

`s[i] != s[j]` 不代表区间 `[i,j]` 没有回文子序列，内部可能有很长的回文。

```python
# ❌ 错误：不匹配直接置 0，内部已有的回文子序列全丢失
if s[i] != s[j]:
    dp[i][j] = 0

# ✅ 正确：分别尝试去掉左端或右端，取二者最大值
if s[i] != s[j]:
    dp[i][j] = max(dp[i + 1][j], dp[i][j - 1])
```

**4. 647 长度为 2 时未特判，直接访问 dp[i+1][j-1]**

当 `j - i == 1` 时，`dp[i+1][j-1]` 即 `dp[i+1][i]`，是非法区间（右端小于左端）。

```python
# ❌ 错误：没有区分长度 1/2 和长度 ≥ 3
if s[i] == s[j]:
    dp[i][j] = dp[i + 1][j - 1]   # j-i==1 时 dp[i+1][i] 未定义！

# ✅ 正确：长度 ≤ 2 时直接置 True，长度 ≥ 3 才看内部
if s[i] == s[j]:
    if j - i <= 1:
        dp[i][j] = True            # 单字符或相邻两字符
    else:
        dp[i][j] = dp[i + 1][j - 1]
```

**5. 混淆子串与子序列的不匹配处理**

子串要求连续：首尾不匹配，`dp[i][j]` 直接是 `False`（无法延伸）。
子序列允许跳跃：首尾不匹配，仍可向内缩一端继续寻找。

```python
# 647 子串：不匹配 = 直接 False（初始值，无需赋值）
# 516 子序列：不匹配 = max(dp[i+1][j], dp[i][j-1])
# 口诀：子串断就断，子序列断了找两侧
```

## 详解

### 647. 回文子串（统计数量）

**题意**：给定字符串 `s`，计算其中所有回文子串的数量，单字符也算。

```text
示例：s = "aaa"
回文子串："a"×3 + "aa"×2 + "aaa"×1 = 6 个
```

#### 核心思路

定义 `dp[i][j]` 为 bool，表示 `s[i..j]` 是否是回文子串。

递推关系基于「缩圈」：若 `s[i] == s[j]`，则 `s[i..j]` 是否回文取决于内部 `s[i+1..j-1]`。
统计答案：每发现一个 `dp[i][j] == True`，计数器加一。

#### Python 代码

```python
class Solution:
    def countSubstrings(self, s: str) -> int:
        n = len(s)
        dp = [[False] * n for _ in range(n)]
        result = 0

        # i 从大到小：保证填 dp[i][j] 时，左下角 dp[i+1][j-1] 已就绪
        for i in range(n - 1, -1, -1):
            for j in range(i, n):
                if s[i] == s[j]:
                    if j - i <= 1:
                        # 单字符（i==j）或相邻两字符（j-i==1），直接是回文
                        dp[i][j] = True
                    else:
                        # 长度 ≥ 3：首尾相同且内部也是回文，才是回文
                        dp[i][j] = dp[i + 1][j - 1]
                # s[i] != s[j] 时 dp[i][j] 保持 False

                if dp[i][j]:
                    result += 1

        return result
        # 时间 O(n²)，空间 O(n²)
```

#### dp 表格图解（s = "aaa"）

```text
初始（全 False）：       填表后（T = True）：

     j=0 j=1 j=2             j=0  j=1  j=2
i=0 [ F   F   F ]      i=0 [  T    T    T  ]
i=1 [ F   F   F ]  →   i=1 [       T    T  ]
i=2 [ F   F   F ]      i=2 [            T  ]

填表顺序（i 从下往上）：
  i=2: dp[2][2]=T（单字符 'a'）
  i=1: dp[1][1]=T，dp[1][2]=T（"aa"，长度2直接True）
  i=0: dp[0][0]=T，dp[0][1]=T（"aa"），dp[0][2]=T（需 dp[1][1]=T，成立）

共 6 个 True → 答案 = 6
```

#### 双指针中心扩展法（空间最优解）

DP 需要 O(n²) 空间存储整张表，而双指针只需 O(1)。
核心思路：枚举每个可能的回文中心（奇数中心 = 单字符，偶数中心 = 相邻两字符），
从中心向两侧扩展，每扩展成功一次就计一个回文。

```python
class Solution:
    def countSubstrings(self, s: str) -> int:
        n = len(s)
        result = 0

        def expand(left: int, right: int) -> int:
            """从 (left, right) 向两侧扩展，统计扩展途中经过的回文数量"""
            count = 0
            while left >= 0 and right < n and s[left] == s[right]:
                count += 1
                left -= 1    # 向左扩
                right += 1   # 向右扩
            return count

        for i in range(n):
            result += expand(i, i)       # 奇数长度：以单字符 i 为中心
            result += expand(i, i + 1)   # 偶数长度：以 i 和 i+1 为中心

        return result
        # 时间 O(n²)，空间 O(1)
```

> 647 推荐双指针法：时间复杂度相同，但空间从 O(n²) 降到 O(1)，代码更直观。
> 区间 DP 的价值在于理解「对角线填充」的遍历模式，为 516 等题打下基础。

### 516. 最长回文子序列

**题意**：给定字符串 `s`，找出其中最长的回文子序列（不要求连续），返回长度。

```text
示例：s = "bbbab"
最长回文子序列 = "bbbb"（跳过 'a'），长度 = 4
```

#### 核心思路

定义 `dp[i][j]` 为整数，表示 `s[i..j]` 中最长回文子序列的长度。

与 647 的本质区别：子序列**可以不连续**，因此首尾不匹配时不能直接放弃，
而是分别尝试去掉左端或右端，选保留更长的那一侧。

递推公式：
- `s[i] == s[j]`：两端字符同时纳入，`dp[i][j] = dp[i+1][j-1] + 2`
- `s[i] != s[j]`：不能同时纳入，`dp[i][j] = max(dp[i+1][j], dp[i][j-1])`

#### Python 代码

```python
class Solution:
    def longestPalindromeSubseq(self, s: str) -> int:
        n = len(s)
        dp = [[0] * n for _ in range(n)]

        # 对角线初始化：单字符是长度 1 的回文子序列
        for i in range(n):
            dp[i][i] = 1

        # i 从大到小（从下往上），j 从 i+1 开始向右
        for i in range(n - 1, -1, -1):
            for j in range(i + 1, n):
                if s[i] == s[j]:
                    # 首尾字符相同：夹住内部子序列，长度 +2
                    # 当 j == i+1 时，dp[i+1][i] 是非法区间，值为 0，结果 = 0+2 = 2，正确
                    dp[i][j] = dp[i + 1][j - 1] + 2
                else:
                    # 首尾不同：分别去掉左端或右端，保留更长的
                    dp[i][j] = max(dp[i + 1][j], dp[i][j - 1])

        # 整个字符串的最长回文子序列长度
        return dp[0][n - 1]
        # 时间 O(n²)，空间 O(n²)
```

#### dp 表格图解（s = "cbbd"，n = 4）

```text
初始化（对角线为 1，其余为 0）：
       c    b    b    d
  c  [ 1    0    0    0 ]
  b  [ 0    1    0    0 ]
  b  [ 0    0    1    0 ]
  d  [ 0    0    0    1 ]

填表过程（i 从 2 到 0，j 从 i+1 向右）：

i=2, j=3: s[2]='b', s[3]='d'，不同
    dp[2][3] = max(dp[3][3], dp[2][2]) = max(1, 1) = 1

i=1, j=2: s[1]='b', s[2]='b'，相同
    dp[1][2] = dp[2][1] + 2 = 0 + 2 = 2    （"bb"）

i=1, j=3: s[1]='b', s[3]='d'，不同
    dp[1][3] = max(dp[2][3], dp[1][2]) = max(1, 2) = 2

i=0, j=1: s[0]='c', s[1]='b'，不同
    dp[0][1] = max(dp[1][1], dp[0][0]) = max(1, 1) = 1

i=0, j=2: s[0]='c', s[2]='b'，不同
    dp[0][2] = max(dp[1][2], dp[0][1]) = max(2, 1) = 2

i=0, j=3: s[0]='c', s[3]='d'，不同
    dp[0][3] = max(dp[1][3], dp[0][2]) = max(2, 2) = 2

最终 dp 表：
       c    b    b    d
  c  [ 1    1    2    2 ]
  b  [ 0    1    2    2 ]
  b  [ 0    0    1    1 ]
  d  [ 0    0    0    1 ]

答案 dp[0][3] = 2，最长回文子序列为 "bb"
```

## 统一对比

| 对比维度 | 647 回文子串（计数） | 516 最长回文子序列 |
|---------|-------------------|-----------------|
| 子串/子序列 | 子串（**连续**） | 子序列（**不连续**） |
| dp 值类型 | `bool` | `int`（长度） |
| 初始化 | 全 `False` | 对角线 `dp[i][i]=1`，其余 `0` |
| `s[i]==s[j]` 且长度 ≤ 2 | 直接 `True` | 直接 `0 + 2 = 2`（自然正确） |
| `s[i]==s[j]` 且长度 ≥ 3 | `dp[i+1][j-1]` | `dp[i+1][j-1] + 2` |
| `s[i]!=s[j]` | 保持 `False`（无操作） | `max(dp[i+1][j], dp[i][j-1])` |
| 遍历方向 | i 从大到小，j 从小到大 | 同左 |
| 结果取法 | 遍历中累计 `True` 的个数 | `dp[0][n-1]` |
| 空间优化 | 可用双指针降至 O(1) | 无简单替代，DP 是标准解 |
| 时间复杂度 | O(n²) | O(n²) |
| 空间复杂度 | O(n²)，双指针 O(1) | O(n²) |

**一句话总结区别**：子串连续用 bool，不匹配就断；子序列不连续用整数，不匹配还要找两侧最大。

## 口诀

```text
【区间 DP 遍历口诀】
dp[i][j] 看左下，左下先算才靠谱。
i 从大到小（从下往上），j 从小到大（从左往右）。
从上往下遍历 i，左下角空，必出错。

【647 回文子串口诀】
首尾相同看长度：长度 1 或 2 直接 True。
长度 ≥ 3 看内部：内部为 True 才算 True。
遍历途中数 True，统计完就是答案。

【516 回文子序列口诀】
对角线先置 1，单字符是基础。
首尾相同加两端（+2），首尾不同取两侧最大。
答案在左上角 dp[0][n-1]。

【子串 vs 子序列一句话】
连续（子串）断就 False，不连续（子序列）断了找两侧。

【易错检查三问】
i 是从大到小吗？（遍历方向）
516 的对角线初始化了吗？（初始化）
647 的 j-i==1 特判了吗？（区间边界）
```

## 总结与扩展

回文类 DP 本质上都是**区间 DP**，整个框架只有两个核心点：

1. **定义**：`dp[i][j]` 描述区间 `[i, j]` 的回文信息（bool 或 int）
2. **遍历**：i 从大到小，j 从小到大，确保左下角 `dp[i+1][j-1]` 先于 `dp[i][j]` 计算

647 和 516 的分叉仅在于「连续性」：子串（连续）首尾不匹配直接放弃；子序列（不连续）首尾不匹配仍可向内缩减一端继续搜索。

**相关扩展题目**：

- **LeetCode 5（最长回文子串）**：与 647 几乎相同的 DP（dp 值为 bool，额外记录最长区间起点）；双指针中心扩展更优，空间 O(1)。
- **LeetCode 1312（让字符串成为回文串的最少插入次数）**：等价于 `n - dp[0][n-1]`，直接复用 516 的结果。
- **LeetCode 132（分割回文串 II）**：先用 647 的区间 DP 预处理所有区间是否为回文，再做一维 DP 求最少切割数。
