## 核心模板（速查）

### 子集模板（78）

```python
def backtrack(start: int):
    result.append(path[:])          # 进入即收集，每个节点都是答案
    for i in range(start, n):
        path.append(nums[i])
        backtrack(i + 1)            # i+1：不回头选已选过的
        path.pop()
```

### 去重子集模板（90）

```python
nums.sort()                         # 去重前提：必须排序

def backtrack(start: int):
    result.append(path[:])
    for i in range(start, n):
        if i > start and nums[i] == nums[i - 1]:  # 树层去重：i>start，不是i>0
            continue
        path.append(nums[i])
        backtrack(i + 1)
        path.pop()
```

### 排列模板（46 / 47）

```python
used = [False] * n

def backtrack():
    if len(path) == n:              # 叶节点：path 选满才收集
        result.append(path[:])
        return
    for i in range(n):              # 从 0 开始，无 start 参数
        if used[i]:                 # 树枝去重：同路径内不重复选同一位置
            continue
        # 47 题额外加一行树层去重（见详解）
        used[i] = True
        path.append(nums[i])
        backtrack()
        path.pop()
        used[i] = False
```

| 对比项 | 子集（78/90） | 排列（46/47） | 组合（77/40） |
|--------|-------------|-------------|--------------|
| 收集时机 | 每进入一层 | 叶节点（path 满） | 叶节点（满足条件） |
| start 参数 | 有，防止回头 | 无 | 有，防止回头 |
| used 数组 | 不需要 | 需要 | 不需要 |
| 去重工具 | `i>start` 或本层 set | `not used[i-1]` | `i>start` |
| 答案数量级 | $2^n$ | $n!$ | $C(n,k)$ |

## ⚠️ 高频错误（先看这里）

### 错误 1：子集只收集叶节点

```python
# ❌ 错误：加了长度判断才收集，漏掉所有中间节点
def backtrack(start):
    if len(path) == k:
        result.append(path[:])
        return
    for i in range(start, n): ...

# ✅ 正确：进入函数就无条件收集，空集也是答案
def backtrack(start):
    result.append(path[:])
    for i in range(start, n): ...
```

---

### 错误 2：90 题去重写成 `i > 0` 而非 `i > start`

```python
# ❌ 错误：i > 0 会错误跳过递归深层的合法起点
if i > 0 and nums[i] == nums[i - 1]:
    continue

# ✅ 正确：i > start 才能精确判断"本层已出现过相同值"
if i > start and nums[i] == nums[i - 1]:
    continue
```

反例：`nums = [1, 2, 2]`，`start=2` 时 `i=2` 是本层第一个元素。`i > 0` 成立会错误跳过这个合法起点，漏掉子集 `[2, 2]` 的 `[2]` 分支。

### 错误 3：491 题对输入排序

```python
# ❌ 错误：排序破坏原始序列结构，递增子序列含义完全改变
nums.sort()   # 绝对不能做！

# ✅ 正确：不排序，改用本层 set 去重（见 491 详解）
```

`[4, 7, 6, 7]` 排序后变成 `[4, 6, 7, 7]`，`[7, 6]` 这类原本不合法的路径在排序后的序列里根本不会出现，大量结果被改变。

### 错误 4：491 题对 `used_this_level` 做撤销

```python
# ❌ 错误：加了 remove，本层后续相同值无法被拦截，去重失效
used_this_level.add(nums[i])
path.append(nums[i])
backtrack(i + 1)
path.pop()
used_this_level.remove(nums[i])    # 不能有这行！

# ✅ 正确：set 只加不删；它是局部变量，函数返回后自动消亡
used_this_level.add(nums[i])
path.append(nums[i])
backtrack(i + 1)
path.pop()
```

### 错误 5：47 题排列去重方向写反

```python
# ❌ 错误：used[i-1]==True 时跳 -> 树枝去重，漏掉合法排列
if i > 0 and nums[i] == nums[i - 1] and used[i - 1]:
    continue
# 后果：nums=[1,1,2] 中 [1(pos0),1(pos1),2] 被错误跳过

# ✅ 正确：not used[i-1] 时跳 -> 树层去重，只跳重复分支
if i > 0 and nums[i] == nums[i - 1] and not used[i - 1]:
    continue
```

记忆方法：**`used[i-1]` 是 False 跳（树层重复），是 True 留（树枝延伸）**。

## 详解

### 78. 子集

#### 核心思路

树上**每个节点**都是合法子集（含空集）。在递归入口无条件 `append(path[:])`，for 循环范围自然收缩，无需显式终止条件。

```python
def subsets(nums: list[int]) -> list[list[int]]:
    result, path = [], []
    n = len(nums)

    def backtrack(start: int):
        result.append(path[:])      # 每进入一层立即收集

        for i in range(start, n):
            path.append(nums[i])
            backtrack(i + 1)
            path.pop()

    backtrack(0)
    return result
```

#### 图解（决策树，nums = [1, 2, 3]）

```
收集 []
  选1 -> 收集 [1]
    选2 -> 收集 [1,2]
      选3 -> 收集 [1,2,3]
    选3 -> 收集 [1,3]
  选2 -> 收集 [2]
    选3 -> 收集 [2,3]
  选3 -> 收集 [3]

结果（共 2^3=8 个）：[], [1], [1,2], [1,2,3], [1,3], [2], [2,3], [3]
```

### 90. 子集 II

#### 核心思路

先排序让相同元素相邻，for 循环中加 `i > start and nums[i] == nums[i-1]` 跳过同层重复值。78 的代码只多了 `sort()` 和这一行 `continue`。

#### Python 代码

```python
def subsetsWithDup(nums: list[int]) -> list[list[int]]:
    nums.sort()
    result, path = [], []
    n = len(nums)

    def backtrack(start: int):
        result.append(path[:])
        for i in range(start, n):
            if i > start and nums[i] == nums[i - 1]:  # 树层去重
                continue
            path.append(nums[i])
            backtrack(i + 1)
            path.pop()

    backtrack(0)
    return result
```

#### 图解（nums = [1, 2, 2] 排序后）

```
收集 []
  选1(i=0) -> 收集 [1]
    选2a(i=1) -> 收集 [1,2]
      选2b(i=2) -> 收集 [1,2,2]
    选2b(i=2)  i>start(1) 且 nums[2]==nums[1] -> 跳过
  选2a(i=1) -> 收集 [2]
    选2b(i=2) -> 收集 [2,2]
  选2b(i=2)  i>start(0) 且 nums[2]==nums[1] -> 跳过

结果：[], [1], [1,2], [1,2,2], [2], [2,2]
```

### 491. 非递减子序列

#### 核心思路

两个限制：① path 需非递减；② 结果不含重复子集，且**不能排序**。

用 `used_this_level = set()` 在每层记录已用过的值实现树层去重。满足长度 ≥ 2 时收集，但**不 `return`**（还要继续向下找更长的子序列）。

#### Python 代码

```python
def findSubsequences(nums: list[int]) -> list[list[int]]:
    result, path = [], []
    n = len(nums)

    def backtrack(start: int):
        if len(path) >= 2:
            result.append(path[:])  # 收集但不 return

        used_this_level = set()     # 每层独立创建，不跨层，不做撤销

        for i in range(start, n):
            if path and nums[i] < path[-1]:         # 破坏非递减，跳过
                continue
            if nums[i] in used_this_level:          # 本层重复值，跳过
                continue
            used_this_level.add(nums[i])
            path.append(nums[i])
            backtrack(i + 1)
            path.pop()

    backtrack(0)
    return result
```

#### 图解（nums = [4, 7, 6, 7]）

```
start=0, used={}
  选4 -> path=[4]
    start=1, used={}
      选7 -> path=[4,7]  收集[4,7]
        6<7 跳; 选7 -> path=[4,7,7]  收集[4,7,7]
      选6 -> path=[4,6]  收集[4,6]
        选7 -> path=[4,6,7]  收集[4,6,7]
      选7(i=3): 7 in used{7} -> 跳过（树层去重）
  选7(i=1) -> path=[7]
    6<7 跳; 选7 -> path=[7,7]  收集[7,7]
  选6(i=2) -> path=[6]
    选7 -> path=[6,7]  收集[6,7]
  选7(i=3): 7 in used{7} -> 跳过（树层去重）
```

### 46. 全排列

#### 核心思路

`[1,2]` 和 `[2,1]` 是不同答案，所以每层都从 0 枚举（无 `start`）。`used[i]` 标记当前路径上哪些位置已占用，防止同一路径重复选同一位置。`len(path)==n` 时才收集（叶节点）。

#### Python 代码

```python
def permute(nums: list[int]) -> list[list[int]]:
    result, path = [], []
    n = len(nums)
    used = [False] * n

    def backtrack():
        if len(path) == n:
            result.append(path[:])
            return
        for i in range(n):
            if used[i]:
                continue
            used[i] = True
            path.append(nums[i])
            backtrack()
            path.pop()
            used[i] = False

    backtrack()
    return result
```

#### 图解（nums = [1, 2, 3]）

```
[]
├─ 选1 -> [1]
│   ├─ 选2 -> [1,2] -> 选3 -> [1,2,3]*
│   └─ 选3 -> [1,3] -> 选2 -> [1,3,2]*
├─ 选2 -> [2]
│   ├─ 选1 -> [2,1] -> 选3 -> [2,1,3]*
│   └─ 选3 -> [2,3] -> 选1 -> [2,3,1]*
└─ 选3 -> [3]
    ├─ 选1 -> [3,1] -> 选2 -> [3,1,2]*
    └─ 选2 -> [3,2] -> 选1 -> [3,2,1]*

* 叶节点收集，共 3!=6 个结果
```

### 47. 全排列 II

#### 核心思路

排序后在 46 基础上增加一行树层去重：

```
if i > 0 and nums[i] == nums[i-1] and not used[i-1]: continue
```

`used[i-1] == False` 说明 `nums[i-1]` 在本层已被选并回溯撤销，当前 `nums[i]`（同值）继续选会产生完全一样的子树，跳过。`used[i-1] == True` 说明前者还在路径中（树枝延伸），合法，不跳。

#### Python 代码

```python
def permuteUnique(nums: list[int]) -> list[list[int]]:
    nums.sort()
    result, path = [], []
    n = len(nums)
    used = [False] * n

    def backtrack():
        if len(path) == n:
            result.append(path[:])
            return
        for i in range(n):
            if used[i]:
                continue
            if i > 0 and nums[i] == nums[i - 1] and not used[i - 1]:  # 树层去重
                continue
            used[i] = True
            path.append(nums[i])
            backtrack()
            path.pop()
            used[i] = False

    backtrack()
    return result
```

#### 图解（nums = [1, 1, 2]，1a=nums[0], 1b=nums[1]）

```
[]
├─ 选1a(used[0]=T) -> [1a]
│   ├─ 选1b(used[1]=T) -> [1a,1b] -> 选2 -> [1,1,2]*
│   └─ 选2(used[2]=T)  -> [1a,2]  -> 选1b -> [1,2,1]*
├─ 选1b(i=1): not used[0] 且 nums[1]==nums[0] -> 跳过（树层去重）
└─ 选2(used[2]=T) -> [2]
    ├─ 选1a -> [2,1a] -> 选1b -> [2,1,1]*
    └─ 选1b(i=1): not used[0]? used[0]=F -> 跳过（树层去重）

结果：[1,1,2], [1,2,1], [2,1,1]
```

## 统一对比

| 题目 | 元素有重复 | 能否排序 | 收集时机 | 去重方式 | 去重核心代码 |
|------|----------|---------|---------|---------|------------|
| 78 子集 | 无 | 不需要 | 每个节点 | 无需去重 | — |
| 90 子集 II | 有 | 可以 | 每个节点 | 排序 + 跳同层重复 | `i > start and nums[i]==nums[i-1]` |
| 491 非递减子序列 | 有 | **不能** | 节点（长度≥2） | 本层 set | `nums[i] in used_this_level` |
| 46 全排列 | 无 | 不需要 | 叶节点 | used 防重复选位置 | `if used[i]: continue` |
| 47 全排列 II | 有 | 可以 | 叶节点 | 排序 + used 判树层 | `not used[i-1]` |

**两大规律**：子集/组合有 `start`，排列无 `start`（用 `used` 代替）；去重本质相同——同一树层不能用相同值开头，能排序则用 `i>start` 跳，不能排序则用本层 `set`。

## 总结与扩展

五道题的知识地图：

```
回溯三类问题
├── 子集类（收集每个节点，有 start）
│   ├── 78 无重复  -> 直接回溯，startIndex 防回头
│   ├── 90 有重复  -> 排序 + i>start 树层去重
│   └── 491 不能排序 -> 本层 set 去重 + 非递减剪枝
└── 排列类（收集叶节点，无 start）
    ├── 46 无重复  -> used 数组防同路径重选
    └── 47 有重复  -> 排序 + not used[i-1] 树层去重
```

**扩展**：491 的本层 set 方案可推广到所有"不能排序但需树层去重"的场景；47 的 `not used[i-1]` 是排列去重的标准写法，排列没有 `start`，不能用 `i > start` 替代。

**写代码前五个自检**：① 子集（每节点收集）vs 排列/组合（叶节点收集）？② 有重复且能排序用 `i>start`，不能排序用本层 set？③ 排列去重用 `not used[i-1]`？④ 491 的 `used_this_level` 未误加 `remove`？⑤ 收集时用 `path[:]` 而非 `path`？
