## 识别信号

> 看到什么特征，想到这篇笔记？

- 对数组每个位置求**右侧（或左侧）第一个比它大的元素**，暴力双重循环是 O(n²)
- 结果数组与输入数组等长，每个位置独立地找"最近更大值"——**值本身**或**距离**
- 题目出现「下一个更大」「等待几天」「循环数组」「绕圈搜索」等关键词
- 两个数组有**子集关系**（496），或数组**首尾相连**（503）

### O(n²) 暴力 → O(n) 单调栈的关键洞察

```python
# 暴力：每个元素向右扫描，最坏（严格递减序列）每次扫到末尾
for i in range(n):
    for j in range(i + 1, n):   # 嵌套循环 → O(n²)
        if nums[j] > nums[i]:
            result[i] = nums[j]
            break
```

单调栈：**已结算元素不再检查**。维护一个"尚未找到答案的候选者"栈，遇到更大新元素时批量结算，每个元素最多入栈一次、出栈一次，总操作 2n 次 → O(n)。

### 两种方向的单调栈

| 目标 | 栈的维护顺序（栈底→栈顶） | 弹出条件 |
|------|------------------------|---------|
| 找右侧第一个**更大**值 | 递减（大→小） | 新元素 **>** 栈顶对应值 |
| 找右侧第一个**更小**值 | 递增（小→大） | 新元素 **<** 栈顶对应值 |

本文三题均为"找右侧第一个更大值"，使用**单调递减栈**。

## 核心模板（速查）

单调递减索引栈：栈底对应值大、栈顶对应值小；新元素更大则循环弹出结算：

```python
def next_greater(nums: list[int]) -> list[int]:
    n = len(nums)
    result = [-1] * n          # 739 改为 [0]*n（默认含义不同）
    stack: list[int] = []      # 存索引！对应值保持单调递减（大→小）

    for i in range(n):
        # 新元素更大：弹出所有小于它的候选者，批量给出答案
        while stack and nums[i] > nums[stack[-1]]:   # 严格大于，相等不触发
            idx = stack.pop()
            result[idx] = nums[i]   # ← 三题唯一的区别点在这一行
        stack.append(i)             # 当前元素入栈，等待右侧更大值

    # 遍历结束，stack 中剩余的索引右侧无更大值，保持默认值
    return result
```

三题变化对比——**弹出时写入逻辑是唯一差异**：

| 题号 | result 初始值 | 弹出时写入 `result[idx]` | 额外结构 | 遍历范围 |
|------|-------------|------------------------|---------|---------|
| 739 每日温度 | `[0] * n` | `i - idx`（等待天数） | 无 | `range(n)` |
| 496 下一个更大元素 I | `[-1] * len(nums1)` | `nums2[i]`（字典映射） | 字典 `{num1值: 下标}` | `range(len(nums2))` |
| 503 下一个更大元素 II | `[-1] * n` | `nums[i % n]` | 无 | `range(2*n)`，`i<n` 才入栈 |

**三种入栈情况**（以"找右侧更大值"为例，新元素为 `nums[i]`，栈顶元素为 `nums[stack[-1]]`）：

| 情况 | 条件 | 动作 |
|------|------|------|
| 新元素更小 | `nums[i] < nums[stack[-1]]` | 直接入栈，维持单调递减 |
| 新元素相等 | `nums[i] == nums[stack[-1]]` | 直接入栈（相等不算"更大"） |
| 新元素更大 | `nums[i] > nums[stack[-1]]` | 循环弹出结算，直到栈空或不满足条件，再入栈 |

## ⚠️ 高频错误

**1. 栈里存值而不是索引（最常见）**

```python
# ❌ 错误：存值后无法计算距离（739），也无法做字典反查（496）
stack.append(nums[i])
idx = stack.pop()
result[idx] = i - idx       # idx 是值不是下标，这里会用值作为数组索引

# ✅ 正确：存索引，通过 nums[idx] 随时取值，同时支持 i - idx 计算距离
stack.append(i)
idx = stack.pop()
result[idx] = i - idx       # 739: 等待天数
result[idx] = nums[i]       # 496/503: 值本身
```

**2. 弹出条件用 `>=`（把相等也当"更大"处理）**

```python
# ❌ 错误：相等也弹出，违反"严格更大"语义，等值元素被错误结算
while stack and nums[i] >= nums[stack[-1]]:
    result[stack.pop()] = nums[i]

# ✅ 正确：严格大于才触发，相等时新元素直接入栈等待
while stack and nums[i] > nums[stack[-1]]:
    result[stack.pop()] = nums[i]
```

**3. 503 第二轮（i >= n）继续入栈，同一索引二次入栈**

```python
# ❌ 错误：两轮都 append，i=n 时 i%n=0，0 号索引第二次入栈导致重复结算
for i in range(2 * n):
    while stack and nums[i % n] > nums[stack[-1]]:
        result[stack.pop()] = nums[i % n]
    stack.append(i % n)         # ← 第二轮不能再 append

# ✅ 正确：第二轮只弹出结算，不再入栈
for i in range(2 * n):
    while stack and nums[i % n] > nums[stack[-1]]:
        result[stack.pop()] = nums[i % n]
    if i < n:                   # 只有第一轮才入栈
        stack.append(i)
```

**4. 496 用 `len(nums2)` 初始化 result**

```python
# ❌ 错误：result 是给 nums1 的，却用 nums2 的长度初始化
result = [-1] * len(nums2)    # nums1、nums2 长度通常不同

# ✅ 正确：result 大小 = nums1 的长度（查询数组）
result = [-1] * len(nums1)
```

**5. 739 默认值照抄 496/503 的 -1**

```python
# ❌ 错误：739 题目要求"无更大值时输出 0"，-1 会导致 WA
result = [-1] * n

# ✅ 正确：739 输出等待天数，无更大值时天数为 0
result = [0] * n
```

## 详解

### 739. 每日温度

#### 核心思路

对每个位置求"右侧第一个更高温度距当前的天数"。单调递减索引栈，弹出时用索引差 `i - idx` 记录等待天数。

#### Python 代码

```python
class Solution:
    def dailyTemperatures(self, temperatures: list[int]) -> list[int]:
        n = len(temperatures)
        result = [0] * n              # 默认 0：右侧没有更高温度，等待 0 天
        stack: list[int] = []         # 存索引，对应温度值保持单调递减

        for i in range(n):
            # 新温度比栈顶更高：栈顶那天终于等到了更暖的一天
            while stack and temperatures[i] > temperatures[stack[-1]]:
                idx = stack.pop()
                result[idx] = i - idx     # 等待天数 = 当前索引 - 栈顶索引
            stack.append(i)               # 当前位置入栈，等待右侧更高温

        return result
        # 时间 O(n)：每个索引最多入栈一次、出栈一次，总操作 2n 次
        # 空间 O(n)：最坏情况（严格递减序列）栈存全部 n 个索引
```

#### 图解（关键步骤）


### 496. 下一个更大元素 I

#### 核心思路

`nums1` 是 `nums2` 的子集，求 `nums1` 每个数在 `nums2` 中右侧第一个更大的数。对 `nums2` 跑单调栈，弹出时通过字典 `{num1的值: 它在nums1的下标}` 判断该值是否属于 `nums1`，若是则写入结果。字典是连接两个数组的桥梁。

三步走：
1. 建字典 `{nums1[i]: i}`，记录 nums1 中每个数的位置
2. 对 `nums2` 维护单调递减索引栈
3. 弹出时通过字典判断是否需要写入结果

#### Python 代码

```python
class Solution:
    def nextGreaterElement(self, nums1: list[int], nums2: list[int]) -> list[int]:
        result = [-1] * len(nums1)                       # 结果对应 nums1 的每个元素
        num_to_idx = {v: i for i, v in enumerate(nums1)} # nums1 值 → 在nums1中的下标
        stack: list[int] = []                            # 存 nums2 的索引，值单调递减

        for i in range(len(nums2)):
            while stack and nums2[i] > nums2[stack[-1]]:
                val = nums2[stack.pop()]            # 被弹出元素的值
                if val in num_to_idx:               # 该值是 nums1 的查询对象
                    result[num_to_idx[val]] = nums2[i]  # 在 nums1 对应位置写入答案
            stack.append(i)

        return result
        # 时间 O(m+n)：建字典 O(m)，遍历 nums2 O(n)，m=len(nums1)，n=len(nums2)
        # 空间 O(m+n)：字典 O(m)，栈最坏 O(n)
```

#### 图解（关键步骤）

```
nums1 = [4, 1, 2]，nums2 = [1, 3, 4, 2]
字典 num_to_idx = {4:0, 1:1, 2:2}
result = [-1, -1, -1]

i=0 val=1  栈空，入栈                  stack nums2索引: [0]   对应值: [1]
i=1 val=3  3 > nums2[0]=1，弹出 0
           val=1，在字典中（下标1）→   result[1] = 3
           栈空，入栈                  stack: [1]             对应值: [3]
i=2 val=4  4 > nums2[1]=3，弹出 1
           val=3，不在字典，跳过
           栈空，入栈                  stack: [2]             对应值: [4]
i=3 val=2  2 < nums2[2]=4，直接入栈   stack: [2,3]           对应值: [4,2]

遍历结束，栈剩余 [2,3]：
  val=4 → 在字典（下标0），未弹出 → result[0] 保持 -1
  val=2 → 在字典（下标2），未弹出 → result[2] 保持 -1
result = [-1, 3, -1]  ✓
```

### 503. 下一个更大元素 II

#### 核心思路

循环数组：末尾元素的右侧可绕回开头继续查找更大值。虚拟遍历 `2n` 次（用 `i % n` 访问真实元素），让原本在末尾的元素也能"看到"前面的元素。第一轮（`i < n`）正常入栈，第二轮（`i >= n`）只弹出结算不入栈——第一轮已将所有元素入栈一次，第二轮再入会导致同一索引重复处理。

#### Python 代码

```python
class Solution:
    def nextGreaterElements(self, nums: list[int]) -> list[int]:
        n = len(nums)
        result = [-1] * n              # 默认 -1：循环一圈也找不到更大值
        stack: list[int] = []          # 存原始索引（0 到 n-1），值单调递减

        for i in range(2 * n):
            real = i % n               # 映射到真实数组下标
            # 发现更大值：弹出所有等待中的候选者并结算
            while stack and nums[real] > nums[stack[-1]]:
                result[stack.pop()] = nums[real]
            # 第二轮不入栈：每个原始索引只能在栈中出现一次
            if i < n:
                stack.append(real)

        return result
        # 时间 O(n)：虽然循环 2n 次，但每个原始索引入栈/出栈各一次
        # 空间 O(n)
```

#### 图解（关键步骤）

```
nums = [1, 2, 1]，n=3
result = [-1, -1, -1]

--- 第一轮 i=0..2（正常入栈）---
i=0 real=0 val=1  栈空，入栈            stack: [0]    对应值: [1]
i=1 real=1 val=2  2>nums[0]=1，弹 0    result[0]=2
                  栈空，入栈            stack: [1]    对应值: [2]
i=2 real=2 val=1  1<nums[1]=2，入栈    stack: [1,2]  对应值: [2,1]

--- 第二轮 i=3..5（只弹不入）---
i=3 real=0 val=1  1 <= nums[2]=1，不弹（相等不触发）
i=4 real=1 val=2  2 > nums[2]=1，弹 2  result[2]=2
                  2 <= nums[1]=2，停止（相等不触发）
i=5 real=2 val=1  1 < nums[1]=2，不弹

遍历结束，栈剩余 [1]（val=2），右侧无更大值，result[1] 保持 -1
result = [2, -1, 2]  ✓
```

## 统一对比

| 维度 | 739 每日温度 | 496 下一个更大 I | 503 下一个更大 II |
|------|------------|----------------|-----------------|
| 数组结构 | 单数组，线性 | 双数组（子集关系） | 单数组，循环 |
| 结果含义 | 距离（等待天数） | 值（nums2 中的数） | 值（循环中找到的数） |
| result 默认值 | `0` | `-1` | `-1` |
| 弹出时写入 | `i - idx` | `nums2[i]`（字典映射） | `nums[i % n]` |
| 额外数据结构 | 无 | 字典 `{num1值: 下标}` | 无 |
| 遍历范围 | `range(n)` | `range(len(nums2))` | `range(2*n)`，`i<n` 才入栈 |
| 时间复杂度 | O(n) | O(m+n) | O(n) |
| 空间复杂度 | O(n) | O(m+n) | O(n) |

**演化关系**：

- 739 是最基础的模板，建立"存索引 + 弹出时写距离"的直觉
- 496 将单数组扩展到双数组——字典在两个数组之间充当桥梁，弹出时查字典决定是否写结果
- 503 将线性遍历扩展到循环遍历——两倍遍历 + 取模 + 第二轮不入栈，三要素缺一不可
- **三题核心栈逻辑完全相同**，差异只在弹出时写什么和遍历范围

## 总结与扩展

单调栈的本质是**空间换时间**：用一个栈记录"尚未找到答案的候选者"，每个元素至多入栈一次、出栈一次，将 O(n²) 暴力降至 O(n)。核心不变量——**栈内元素按值单调排列，新元素破坏单调性时批量结算**——在三道题中始终成立。

三题覆盖了"找下一个更大元素"方向的三种常见变体：

- **739**：建立核心模板，熟悉"弹出时结算距离"的写法
- **496**：双数组场景，字典作为索引中转的思路可迁移到其他跨数组查询问题
- **503**：循环数组场景，"两倍遍历 + 取模 + 第二轮不入栈"是处理循环的通用技巧

**相关扩展题**：

- LeetCode 84「柱状图中最大的矩形」：单调**递增**栈，找左右第一个**更小**柱，计算面积，O(n)
- LeetCode 42「接雨水」：单调栈或双指针，找左右方向最大高度，经典面试题
- LeetCode 316「去除重复字母」：单调递增栈控制字典序，同时维护剩余字符集合
