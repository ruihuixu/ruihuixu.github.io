## 识别信号

> 看到什么特征，想到这篇笔记？

- 要求**原地**修改数组，不能使用额外数组空间（O(1) 空间）
- 题目要求"移除/过滤某个值""去重""把某类元素移到末尾"
- 操作完成后返回新的**有效长度**，或不改变剩余元素的相对顺序
- 有序数组需要去重（有序意味着重复元素必然相邻，可以相邻比较）

## 核心模板（速查）

```python
def fast_slow_template(nums: list[int]) -> int:
    slow = 0                           # slow：有效区间的写入位置（边界守卫）
    for fast in range(len(nums)):
        if 符合条件(nums[fast]):        # ← 三题唯一区别点在这里
            nums[slow] = nums[fast]    # 覆盖写入（或交换，见 283 题）
            slow += 1
    return slow                        # 新长度，或 slow + 1（见 26 题）
```

**三题在模板里的对比：**

| 题号 | 判断条件 | 写入方式 | slow 起点 | fast 起点 | 返回值 |
|------|---------|---------|----------|----------|--------|
| 27   | `nums[fast] != val` | 覆盖 | 0 | 0 | `slow` |
| 26   | `nums[fast] != nums[slow]` | 先移 slow 再覆盖 | 0 | 1 | `slow + 1` |
| 283  | `nums[fast] != 0` | 交换（保顺序） | 0 | 0 | 无返回值 |

## ⚠️ 高频错误

**1. 26 题 fast 从 0 开始，导致首元素和自身比较**

```python
# ❌ 错误：fast=0 时 nums[0] 与 nums[slow]=nums[0] 比较，永远相等，逻辑出错
slow = 0
for fast in range(len(nums)):        # fast 从 0 开始
    if nums[fast] != nums[slow]:
        slow += 1
        nums[slow] = nums[fast]

# ✅ 正确：首元素已占据 slow=0 位置，fast 从 1 开始，跳过首元素
slow = 0
for fast in range(1, len(nums)):     # fast 从 1 开始
    if nums[fast] != nums[slow]:
        slow += 1
        nums[slow] = nums[fast]
```

**2. 26 题返回 slow 而非 slow + 1**

```python
# ❌ 错误：slow 是"最后一个有效元素的索引"，不是长度
return slow

# ✅ 正确：长度 = 最后一个有效元素的索引 + 1
return slow + 1
```

**3. 混淆 27 题与 26 题的 slow 语义（off-by-one 根源）**

```python
# 27 题：slow 语义 = "下一个写入位置"，有效区间是 [0, slow)（左闭右开）
# 26 题：slow 语义 = "最后一个有效元素"，有效区间是 [0, slow]（左闭右闭）
#
# 27 题先写入再移动 slow：
#   nums[slow] = nums[fast]; slow += 1
#
# 26 题先移动 slow 再写入：
#   slow += 1; nums[slow] = nums[fast]
#
# 这个顺序差异导致返回值和 fast 起点都不同，混淆是最常见的 bug。
```

**4. 283 题用覆盖替代交换，破坏非零元素相对顺序**

```python
# ❌ 错误：直接覆盖会把 slow 处的非零值抹掉
if nums[fast] != 0:
    nums[slow] = nums[fast]   # slow 原有的非零值丢失
    slow += 1

# ✅ 正确：交换保证 slow 处的零移到 fast 位置，非零元素有序前移
if nums[fast] != 0:
    nums[slow], nums[fast] = nums[fast], nums[slow]
    slow += 1
```

**5. 26 题未处理空数组，导致 range(1, 0) 访问 nums[slow=0] 越界**

```python
# ❌ 错误：空数组时 slow=0，range(1,0) 不执行循环但返回 slow+1=1，错误
def removeDuplicates(nums: list[int]) -> int:
    slow = 0
    for fast in range(1, len(nums)):
        ...
    return slow + 1   # 空数组时错误地返回 1

# ✅ 正确：提前判空
def removeDuplicates(nums: list[int]) -> int:
    if not nums:
        return 0
    slow = 0
    for fast in range(1, len(nums)):
        ...
    return slow + 1
```

## 详解

### 27. 移除元素（基础模板）

#### 核心思路

`slow` 指向"下一个有效元素的写入位置"，`fast` 从头扫描全数组。遇到 `nums[fast] != val`，将其覆盖写入 `nums[slow]`，然后 `slow` 前进一步。

**核心不变量**：`[0, slow)` 区间内的元素均不等于 `val`。

因为 `slow <= fast` 始终成立，覆盖不会丢失尚未扫描的元素（`fast` 始终走在 `slow` 前面或同位置）。

#### Python 代码

```python
def removeElement(nums: list[int], val: int) -> int:
    slow = 0
    for fast in range(len(nums)):
        if nums[fast] != val:
            # slow <= fast 恒成立：fast 扫描前方，slow 写入后方，互不干扰
            nums[slow] = nums[fast]
            slow += 1
    return slow   # [0, slow) 是有效区间，slow 即新长度
```

#### 图解（nums = [3, 2, 2, 3], val = 3）

```text
初始：[3, 2, 2, 3]
       ↑            slow=0, fast=0
       sf

fast=0, nums[0]=3  (= val，跳过)
      [3, 2, 2, 3]   slow=0, fast=1
       ↑  ↑
       s  f

fast=1, nums[1]=2  (≠ val，写入 slow=0，slow 前进)
      [2, 2, 2, 3]   slow=1, fast=2
          ↑  ↑
          s  f

fast=2, nums[2]=2  (≠ val，写入 slow=1，slow 前进)
      [2, 2, 2, 3]   slow=2, fast=3
             ↑  ↑
             s  f

fast=3, nums[3]=3  (= val，跳过) → 遍历结束

结果：[2, 2, ?, ?]  返回 slow = 2
      ↑────↑
      有效区间 [0, 2)
```

**复杂度**：时间 O(n)，空间 O(1)

### 26. 删除有序数组中的重复项（有序优化）

#### 核心思路

数组有序，重复元素必然相邻。不需要和固定目标值比较，改为比较 `nums[fast]` 与 `nums[slow]`（已确定的最后一个唯一元素）。遇到新值时，先让 `slow` 前进腾出位置，再写入。

**核心不变量**：`[0, slow]` 区间内的元素均唯一且保持原顺序。

与 27 题关键区别：`slow` 语义从"下一个写入位置"变为"最后一个有效元素位置"，因此写入顺序反转（先移 slow，再写入），返回值也变为 `slow + 1`。

#### Python 代码

```python
def removeDuplicates(nums: list[int]) -> int:
    if not nums:
        return 0
    slow = 0                          # 指向最后一个已确定的唯一元素
    for fast in range(1, len(nums)):  # 首元素已在 slow=0 位置，从第二个比较
        if nums[fast] != nums[slow]:
            slow += 1                 # 先移动 slow（为新唯一元素腾出位置）
            nums[slow] = nums[fast]   # 再写入（不覆盖当前 slow 处的有效值）
    return slow + 1                   # slow 是索引，长度 = 索引 + 1
```

#### 图解（nums = [1, 1, 2, 2, 3]）

```text
初始：[1, 1, 2, 2, 3]
       ↑  ↑            slow=0, fast=1
       s  f

fast=1, nums[1]=1  (= nums[slow]=1，重复，跳过)
      [1, 1, 2, 2, 3]   slow=0, fast=2
       ↑     ↑
       s     f

fast=2, nums[2]=2  (≠ nums[slow]=1，slow 先进 → 写入)
      [1, 2, 2, 2, 3]   slow=1, fast=3
          ↑     ↑
          s     f

fast=3, nums[3]=2  (= nums[slow]=2，重复，跳过)
      [1, 2, 2, 2, 3]   slow=1, fast=4
          ↑        ↑
          s        f

fast=4, nums[4]=3  (≠ nums[slow]=2，slow 先进 → 写入) → 遍历结束
      [1, 2, 3, 2, 3]   slow=2

结果：[1, 2, 3, ?, ?]  返回 slow + 1 = 3
      ↑────────↑
      有效区间 [0, 3)
```

**复杂度**：时间 O(n)，空间 O(1)

### 283. 移动零（稳定交换）

#### 核心思路

目标：把所有 0 移到末尾，非零元素保持相对顺序。

若沿用 27 题的覆盖写法，`slow` 处原有的非零值会被 `fast` 处的非零值覆盖，但 `slow` 处原来存放的 0 并没有去任何地方——最终尾部需要额外补零，或者用"覆盖 + 填零"两轮扫描。

更优雅的方式是**交换**：把 `nums[slow]`（此时一定是 0）和 `nums[fast]`（非零值）对调，非零值前移，0 被踢到后方，相对顺序天然保持。

**核心不变量**：`[0, slow)` 区间内全为非零元素，且保持原有相对顺序。

#### Python 代码

```python
def moveZeroes(nums: list[int]) -> None:
    slow = 0
    for fast in range(len(nums)):
        if nums[fast] != 0:
            # slow == fast 时所有元素均非零，无需自我交换
            if slow != fast:
                nums[slow], nums[fast] = nums[fast], nums[slow]
            slow += 1
    # 原地修改，无返回值
```

#### 图解（nums = [0, 1, 0, 3, 12]）

```text
初始：[0, 1, 0, 3, 12]
       ↑             slow=0, fast=0
       sf

fast=0, nums[0]=0  (= 0，跳过)
      [0, 1, 0, 3, 12]   slow=0, fast=1
       ↑  ↑
       s  f

fast=1, nums[1]=1  (≠ 0，交换 slow=0 ↔ fast=1，slow 前进)
      [1, 0, 0, 3, 12]   slow=1, fast=2
          ↑  ↑
          s  f

fast=2, nums[2]=0  (= 0，跳过)
      [1, 0, 0, 3, 12]   slow=1, fast=3
          ↑     ↑
          s     f

fast=3, nums[3]=3  (≠ 0，交换 slow=1 ↔ fast=3，slow 前进)
      [1, 3, 0, 0, 12]   slow=2, fast=4
             ↑     ↑
             s     f

fast=4, nums[4]=12  (≠ 0，交换 slow=2 ↔ fast=4) → 遍历结束
      [1, 3, 12, 0, 0]   slow=3

结果：[1, 3, 12, 0, 0]  非零元素 1→3→12 顺序完整保留
```

**复杂度**：时间 O(n)，空间 O(1)

## 统一对比

| 特性 | 27. 移除元素 | 26. 删除重复项 | 283. 移动零 |
|------|-------------|--------------|------------|
| 判断条件 | `nums[fast] != val` | `nums[fast] != nums[slow]` | `nums[fast] != 0` |
| 写入方式 | 覆盖 | 先移 slow 再覆盖 | 交换 |
| slow 语义 | 下一个写入位置 | 最后一个有效元素 | 下一个写入位置 |
| fast 起点 | 0 | 1 | 0 |
| 返回值 | `slow` | `slow + 1` | 无 |
| 有效区间 | `[0, slow)` 左闭右开 | `[0, slow]` 左闭右闭 | `[0, slow)` 左闭右开 |
| 数组前提 | 任意顺序 | 必须**有序** | 任意顺序 |
| 为何用交换 | 不需要 | 不需要 | 需要保持非零顺序 |

**抽象升级路径：**

```text
27（纯过滤，基础模板：覆盖 + slow 即长度）
  ↓ 利用有序性，比较对象从固定值改为 nums[slow]
26（有序去重：先移 slow 再写，返回 slow+1）
  ↓ 需要保持非零元素顺序，覆盖改为交换
283（稳定交换：零被"踢"到后方，顺序自然保留）
```

三题本质完全相同：`fast` 向前扫描，`slow` 守住有效边界，区别仅在**判断条件**和**写入方式**。

## 口诀

```text
【快慢指针原地修改口诀】

slow 守边界，fast 向前探；
符合条件写 slow，否则跳过往前走。

27：不等于 val 就覆盖，返回 slow（slow = 下一写入位）；

覆盖用于过滤（无需保留 slow 处原值）；
交换用于保序（slow 处的零要移走，不能丢）。

有效区间开闭决定返回值：
  [0, slow) → 返回 slow
  [0, slow] → 返回 slow + 1

```

## 总结与扩展

三道题共享**快慢指针维护有效边界**这一框架，差异点清单如下：

- **27** 是最纯粹的基础模板：`slow` 指向下一个写入位置，`fast` 遇到非 val 就覆盖，返回 `slow`
- **26** 利用有序性：`slow` 改为"最后一个有效元素"，写入顺序反转（先移 slow 再写），fast 从 1 开始，返回 `slow + 1`
- **283** 为保持顺序：覆盖改为交换，`slow` 处的零随着每次交换自然后移，无需额外补零

掌握核心不变量后，这类题目的难点从"想到方法"缩减为"确认 slow 语义和返回值"——本质上就是有效区间是左闭右开还是左闭右闭的判断。

**扩展练习：**

- **80. 删除有序数组中的重复项 II**：每个元素至多保留两次，将判断条件改为 `nums[fast] != nums[slow - 1]`（与 slow 前一位比较而非 slow 本身），其余逻辑与 26 题完全相同
- **75. 颜色分类（荷兰国旗）**：三值排序，使用三指针（low / mid / high），是快慢指针向三分区的扩展
- **88. 合并两个有序数组**：双指针从后向前合并，避免向前覆盖时丢失尚未处理的元素
