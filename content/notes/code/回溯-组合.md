## 识别信号

> 看到什么特征，想到这篇笔记？

- 题目要求返回"所有满足条件的组合"，而不是一个最优解
- 需要从一个集合中**选若干个元素**，组合顺序不重要（[1,2] 和 [2,1] 算同一个）
- 答案数量可能很大，暴力 k 层嵌套循环因 k 不确定而无法写出
- 元素来自同一集合，但各题对"能否复选"和"是否含重复元素"有不同限制

## 核心模板（速查）

### 通用回溯框架

```python
result = []
path = []

def backtrack(start: int):
    if 终止条件:
        result.append(path[:])   # 必须拷贝，不是引用
        return
    for i in range(start, 上界 + 1):
        if 剪枝条件:
            break                # 排序后可以 break；无序只能 continue
        path.append(i)           # 做选择
        backtrack(i + 1)         # 递归纵向深入（可复选时传 i）
        path.pop()               # 撤销，恢复现场
```

三个动作永远配对：**append → 递归 → pop**，缺少 pop 就变成单向 DFS。

## ⚠️ 高频错误（先看这里）

**1. 77 题剪枝条件写反——用 `len(path) + (n - i + 1) >= k` 而非更直观的提前 break**

```python
# ❌ 错误：没有剪枝，当剩余元素不足以填满 path 时仍然继续循环
for i in range(start, n + 1):
    path.append(i)
    backtrack(i + 1)
    path.pop()

# ✅ 正确：剩余可选数量 n-i+1 < 还需选数量 k-len(path) 时，直接 break
for i in range(start, n + 1):
    if n - i + 1 < k - len(path):
        break
    path.append(i)
    backtrack(i + 1)
    path.pop()
```

验证：n=4, k=3，path 为空时需再选 3 个。i=3 时剩余 4-3+1=2 < 3，break。正确跳过无意义分支。

---

**2. 40 题树层去重写成 `i > 0` 而不是 `i > start`**

```python
# ❌ 错误：i > 0 会把树枝上（递归层）合法的相同值也跳掉，导致漏解
if i > 0 and candidates[i] == candidates[i - 1]:
    continue

# ✅ 正确：i > start 才是"同一 for 循环层"的判断，只跳过树层重复
if i > start and candidates[i] == candidates[i - 1]:
    continue
```

例：candidates=[1,1,2]，target=3。若写 `i > 0`，path=[1] 时 i=1 对应第二个 1，触发 continue，漏掉 [1,1,2]。

---

**3. 39 题可复选时递归传 `i+1` 而不是 `i`**

```python
# ❌ 错误：传 i+1 意味着当前元素只选一次，不满足"可无限复选"
backtrack(i + 1, current_sum + candidates[i])

# ✅ 正确：传 i，下层 for 仍从 i 开始，允许再次选当前元素
backtrack(i, current_sum + candidates[i])
```

39 题与 40 题的唯一区别就在这一个参数：传 `i` 还是 `i+1`。

```python
# ❌ 错误：未排序时相同元素不相邻，无法通过 candidates[i]==candidates[i-1] 去重
def combinationSum2(candidates, target):
```

## 详解

### 77. 组合

#### 核心思路

从 1..n 中取 k 个不同的数，返回所有组合。用 `start` 保证每次只向后选，不回头取已选过的数，从而避免 [1,2] 和 [2,1] 重复。

**剪枝关键**：当 `n - i + 1 < k - len(path)` 时，剩余数字数量不足以填满 path，这一层及后续分支全部无效，直接 `break`。

#### 图解（决策树，n=4, k=2）

```
                      []
         /        /       \       \
       [1]       [2]      [3]     [4]
      / | \      | \       |
  [1,2][1,3][1,4][2,3][2,4] [3,4]
   *    *    *    *    *     *
```

叶子节点（path 长度 = k）处收集结果（标 `*`）。当 i=4 而 k-len(path)=2 时，剩余 4-4+1=1 < 2，剪枝跳过。

#### Python 代码

```python
def combine(n: int, k: int) -> list[list[int]]:
    result = []
    path = []

    def backtrack(start: int):
        # 终止条件：path 已选满 k 个
        if len(path) == k:
            result.append(path[:])
            return
        for i in range(start, n + 1):
            # 剪枝：剩余数量不足以填满 path，直接终止本层后续循环
            if n - i + 1 < k - len(path):
                break
            path.append(i)           # 做选择
            backtrack(i + 1)         # 递归，i+1 保证不重复取
            path.pop()               # 撤销

    backtrack(1)
    return result
    # 时间 O(C(n,k)×k)，空间 O(k)
```

### 216. 组合总和 III

#### 核心思路

从 1-9 中取恰好 k 个不同的数，使其和等于 n，每个数只能用一次。在 77 题基础上叠加"sum 约束"，形成双重剪枝：数量剪枝（同 77）+ sum 超标剪枝。

#### 图解（k=2, n=4）

```
                        []
       /        /       /    \   ...
     [1]       [2]    [3]   [4]
    / | \      |  \    |
[1,2][1,3][1,4][2,3][2,4] [3,4]
 3    4*   5x   5x   6x    7x
```

`*` 标记 sum=4 处收集 [1,3]；`x` 表示 sum 已超 n 被 break 剪掉。

#### Python 代码

```python
def combinationSum3(k: int, n: int) -> list[list[int]]:
    result = []
    path = []

    def backtrack(start: int, current_sum: int):
        # 终止条件：选够 k 个数
        if len(path) == k:
            if current_sum == n:
                result.append(path[:])
            return
        for i in range(start, 10):
            # 数量剪枝（同 77 题）
            if 9 - i + 1 < k - len(path):
                break
            # sum 剪枝：加上 i 已超 n，后续更大，直接 break
            if current_sum + i > n:
                break
            path.append(i)
            backtrack(i + 1, current_sum + i)
            path.pop()

    backtrack(1, 0)
    return result
    # 时间 O(C(9,k)×k)，空间 O(k)
```

### 17. 电话号码的字母组合

#### 核心思路

本题是**多个集合之间**的组合，而非同一集合内的组合。每个数字对应一组字母，层与层之间集合完全不同，所以不需要 startIndex，而是用 `index` 记录当前处理第几个数字。

#### 图解（digits="23"）

```
                    []
         /          |          \
       [a]         [b]         [c]         ← 处理 '2' 对应 abc
      / | \       / | \       / | \
  [ad][ae][af] [bd][be][bf] [cd][ce][cf]   ← 处理 '3' 对应 def
   *   *   *    *   *   *    *   *   *
```

树的深度 = digits 的长度，每层的宽度 = 当前数字对应的字母数量。

#### Python 代码

```python
def letterCombinations(digits: str) -> list[str]:
    if not digits:
        return []   # 边界：空字符串直接返回 []，否则会得到 ['']

    phone = {
        '2': 'abc', '3': 'def', '4': 'ghi', '5': 'jkl',
        '6': 'mno', '7': 'pqrs', '8': 'tuv', '9': 'wxyz'
    }
    result = []
    path = []

    def backtrack(index: int):
        # 终止条件：处理完所有数字
        if index == len(digits):
            result.append(''.join(path))
            return
        # 取当前数字对应的字母集合，逐一选取
        for letter in phone[digits[index]]:
            path.append(letter)        # 做选择
            backtrack(index + 1)       # 处理下一个数字
            path.pop()                 # 撤销

    backtrack(0)
    return result
    # 时间 O(4^n × n)，空间 O(n)，n 为 digits 长度
```

与 77/216 对比：77/216 的 for 循环遍历**同一个数字集合**（需要 startIndex 防回头）；本题每层 for 遍历**当前数字对应的字母集合**，层间集合不同，index 仅作层数计数器。

### 39. 组合总和

#### 核心思路

candidates 无重复元素，每个元素可以**无限次**使用，找和为 target 的所有组合。可复选的关键：递归时传 `i` 而非 `i+1`，允许下层继续从当前元素开始选取。

#### 图解（candidates=[2,3,6,7], target=7）

```
                           []
              /           /        \        \
           [2]           [3]       [6]      [7]*
          / | \         /   \       |
      [2,2][2,3][2,6] [3,3][3,6] [6,6]x
       / \    |    x     |    x
  [2,2,2][2,2,3]* [2,3,3]x [3,3,3]x
    /
[2,2,2,2]x
```

`*` 处 sum==target 收集；`x` 处 sum 超标 break 剪枝。

#### Python 代码

```python
def combinationSum(candidates: list[int], target: int) -> list[list[int]]:
    candidates.sort()   # 排序是 break 剪枝的前提
    result = []
    path = []

    def backtrack(start: int, current_sum: int):
        if current_sum == target:
            result.append(path[:])
            return
        for i in range(start, len(candidates)):
            # 剪枝：排序后若当前元素已超，后续更大，直接 break
            if current_sum + candidates[i] > target:
                break
            path.append(candidates[i])
            # 关键：传 i 而非 i+1——允许下层再次选当前元素（可复选）
            backtrack(i, current_sum + candidates[i])
            path.pop()

    backtrack(0, 0)
    return result
    # 时间 O(n × 2^target)，空间 O(target)
```

### 40. 组合总和 II

#### 核心思路

candidates 含重复元素，每个元素只能用一次，结果集中不能含重复组合。难点在于**树层去重**：相同值在同一 for 循环层（树的同一层）只能被选一次，但在不同层（父子递归层）可以各自选。

**树层去重 vs 树枝去重**：

- **树层**（同一 for 循环的不同迭代）：`i > start and candidates[i] == candidates[i-1]` → `continue` 跳过
- **树枝**（父子递归层）：不跳过，允许路径内连续出现相同值（如 [1,1,2]）

#### 图解（candidates=[1,1,2], target=3）

```
                              []
               /               |              \
            [1a]            [1b] 跳过         [2]
           /    \                              |
        [1a,1b] [1a,2]*                    [2,2]x
           |
       [1a,1b,2]*
```

第二个 1（标 1b）在树层被 `i > start` 条件跳过，避免产生与 [1a] 开头完全重复的结果。收集：[1,2] 和 [1,1,2]。

#### Python 代码

```python
def combinationSum2(candidates: list[int], target: int) -> list[list[int]]:
    candidates.sort()   # 必须排序！去重依赖相邻元素比较
    result = []
    path = []

    def backtrack(start: int, current_sum: int):
        if current_sum == target:
            result.append(path[:])
            return
        for i in range(start, len(candidates)):
            # sum 剪枝：排序后超标，后续更大，直接 break
            if current_sum + candidates[i] > target:
                break
            # 树层去重：同层（i > start）出现相同值，跳过
            if i > start and candidates[i] == candidates[i - 1]:
                continue
            path.append(candidates[i])
            # 关键：传 i+1——每个元素只用一次（区别于 39 题传 i）
            backtrack(i + 1, current_sum + candidates[i])
            path.pop()

    backtrack(0, 0)
    return result
    # 时间 O(2^n × n)，空间 O(n)
```

## 统一对比

```
回溯组合家族
├── 同集合内选，不可复选，无重复元素
│   ├── 77（组合）          start 防回头，上界剪枝
│   └── 216（组合总和 III） 额外 sum 超标 break
├── 多集合之间选（不涉及 start）
│   └── 17（电话号码）      index 逐层切换字母集合
├── 同集合内选，可复选，无重复元素
│   └── 39（组合总和）      递归传 i（不是 i+1）
└── 同集合内选，不可复选，有重复元素
    └── 40（组合总和 II）   排序 + i>start 树层去重
```

| 核心差异 | 表现 |
|---------|------|
| 39 vs 40 | 39 传 `i`（可复选），40 传 `i+1`（不可复选）+ 树层去重 |
| 树层去重 | `i > start`：跳过同 for 循环内相同元素，防止重复组合 |
| 树枝去重 | 若写 `i > 0`，连同一路径内合法的相同值也跳了，漏解 |
| break vs continue | 排序后 sum 超标用 `break`；无法排序时只能用 `continue` |

## 口诀

```
回溯三要素：参数 / 终止 / 单层循环
模板三配对：append → 递归 → pop

组合四分法：
  不复选无重 → startIndex 防回头，传 i+1
  可复选无重 → startIndex 防回头，传 i（39）
  不复选有重 → 先排序，树层去重 i>start（40）
  多集合选   → index 计层，每层遍历对应集合（17）

剪枝两招：
  数量不足  → n-i+1 < k-len(path)，break
  sum 超标  → current_sum + candidates[i] > target，break（需先排序）

去重一句话：
  树层用 i>start，树枝不管——两者搞混必漏解
```

## 总结与扩展

五道组合题是同一回溯模板在不同约束下的变体，差异只集中在三个维度：

- **77**：最基础的组合，startIndex 防回头，for 上界做剪枝（`n-i+1 >= k-len(path)`）。
- **216**：在 77 上叠加 sum 约束，双重剪枝（数量不足 + sum 超标），本质与 77 相同。
- **17**：唯一一道跨集合的组合题，用 index 替代 startIndex，每层 for 遍历当前数字对应的字母集，无需去重。
- **39**：可复选——递归传 `i` 而非 `i+1`，这是与 77/216/40 的唯一代码差异；排序后 sum 超标直接 break。
- **40**：最难点在树层去重——`candidates.sort()` 是前提，`i > start and candidates[i] == candidates[i-1]` 是核心，`i > 0` 是经典错误。

理解了"树层去重"与"树枝去重"的区别，组合类回溯题就没有盲区。其余需要记住的，只有传 `i` 还是 `i+1` 这一个细节。
