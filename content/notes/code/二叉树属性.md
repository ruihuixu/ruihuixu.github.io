## 核心模板（速查）

### 模板一：单树后序递归——修改结构（226）

```python
def postorder_modify(root: Optional[TreeNode]) -> Optional[TreeNode]:
    if not root:
        return None
    root.left  = postorder_modify(root.left)    # 后序：先递归左子树
    root.right = postorder_modify(root.right)   # 后序：再递归右子树
    # ← 后序时机：此处两棵子树已处理完毕，再操作当前节点
    root.left, root.right = root.right, root.left   # 226：交换左右引用
    return root
```

### 模板二：双树同步递归——对比结构（100 / 101 / 572）

```python
def compare(node1: Optional[TreeNode], node2: Optional[TreeNode]) -> bool:
    # 终止条件三步，顺序固定不可颠倒
    if not node1 and not node2: return True    # 都空 → 匹配
    if not node1 or  not node2: return False   # 一空 → 不匹配
    if node1.val != node2.val:  return False   # 值异 → 不匹配
    # ← 各题唯一区别：递归时 node2 侧传哪个方向
    return (compare(node1.left,  node2.???)    # 左子树往哪对
        and compare(node1.right, node2.???))   # 右子树往哪对
```

**各题填空对照**：

| 题号 | 调用入口 | node2 左侧传 | node2 右侧传 | 语义 |
|------|---------|-------------|-------------|------|
| **100** 相同 | `compare(p, q)` | `q.left` | `q.right` | 同向：左配左、右配右 |
| **101** 对称 | `compare(root.left, root.right)` | `r.right` | `r.left` | 反向：外侧配外侧 |
| **572** 子树 | 外层单树 DFS 中触发 | `q.left` | `q.right` | 复用 100 的模板 |

## ⚠️ 高频错误（先看这里）

**1. 226 翻转：用中序遍历，左子树被翻转两遍**

```python
# ❌ 中序：先翻左子树，交换后 root.right = 原 left，再对它递归 = 翻两遍
def invertTree(root):
    if not root: return None
    self.invertTree(root.left)
    root.left, root.right = root.right, root.left   # 交换
    self.invertTree(root.right)  # ← 此时 root.right 已是原 left，再翻一遍！

# ✅ 后序：先把两棵子树各自翻好，最后在当前节点交换引用
root.left  = self.invertTree(root.left)
root.right = self.invertTree(root.right)
root.left, root.right = root.right, root.left       # 最后交换
```

**2. 101 对称：把「镜像递归」写成了「相同树递归」**

```python
# ❌ 同向递归 = 判断相同树，不是对称
return isMirror(left.left,  right.left) \
   and isMirror(left.right, right.right)

# ✅ 反向递归：外侧（左左 vs 右右），内侧（左右 vs 右左）
return isMirror(left.left,  right.right) \
   and isMirror(left.right, right.left)
```

**3. 双树递归：「都空」被「一空」条件吞掉，返回错误 False**

```python
# ❌ 先写 "not p or not q"：p=None, q=None 时 not p=True → 进来返回 False
if not p or not q:
    return False   # 两者都空时也执行了这行，错误！

# ✅ 先判「都空」，再判「一空」（顺序固定）
if not p and not q: return True    # 都空 = 匹配
if not p or  not q: return False   # 一空 = 不匹配
```

**4. 572 子树：忘记 root 为空的提前返回，访问 root.left 崩溃**

```python
# ❌ root=None 时，下一行访问 root.left 触发 AttributeError
def isSubtree(root, subRoot):
    if isSameTree(root, subRoot): return True
    return isSubtree(root.left, subRoot) or ...   # root=None → 崩溃

# ✅ 最先处理 root 为空
def isSubtree(root, subRoot):
    if not root: return False              # 空树不含任何非空子树
    if isSameTree(root, subRoot): return True
    return isSubtree(root.left, subRoot) or isSubtree(root.right, subRoot)
```

## 详解

### 226. 翻转二叉树

#### 核心思路

后序递归：先把左子树和右子树各自翻转，翻好后再交换当前节点的左右指针。子树已处理完毕，当前节点的操作只是"调换两根引用"，自底向上完成整棵树的翻转。

前序（先交换再递归）同样正确。**中序不可用**：左子树翻转完毕后再交换指针，`root.right` 此时已指向原左子树，等于对同一子树翻了两遍。

#### 图解

```text
原树：              后序翻转过程（自底向上）：
      4               步骤 1：叶节点 1、3、6、9 无子树，直接返回
     / \              步骤 2：节点 2：swap(1, 3) → left=3, right=1
    2   7             步骤 3：节点 7：swap(6, 9) → left=9, right=6
   / \ / \            步骤 4：节点 4：swap(子树2 ↔ 子树7)
  1  3 6  9

结果：
      4
     / \
    7   2
   / \ / \
  9  6 3  1
```

#### Python 代码

```python
from typing import Optional

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val; self.left = left; self.right = right

class Solution:
    def invertTree(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        if not root:
            return None

        # 后序：先翻转两棵子树（子问题先解决）
        root.left  = self.invertTree(root.left)
        root.right = self.invertTree(root.right)

        # 两棵子树翻好后，交换当前节点的左右引用
        root.left, root.right = root.right, root.left

        return root
```

> **复杂度**：时间 $O(n)$，每节点恰好访问一次；空间 $O(h)$，$h$ 为树高（递归栈深度）。

### 101. 对称二叉树

#### 核心思路

不是判断"左子树和右子树是否相同"，而是"左子树和右子树是否**互为镜像**"。镜像比较必须**反向传参**：

- 外侧配对：`left.left` vs `right.right`
- 内侧配对：`left.right` vs `right.left`

#### 图解

```text
对称的树：                      不对称的树：
        1                               1
       / \                             / \
     L=2   R=2                       L=2   R=2
     / \ / \                           \     \
    3  4 4  3                           3     3

isMirror(L=2, R=2) 调用展开：
  L.val == R.val?                ✓  (2 == 2)
  isMirror(L.left=3,  R.right=3)?   外侧  ✓
  isMirror(L.right=4, R.left=4)?    内侧  ✓
  → True

与「相同树」的方向对比：
  isSameTree:   compare(p.left,  q.left)   compare(p.right, q.right)  ← 同向
  isSymmetric:  compare(l.left,  r.right)  compare(l.right, r.left)   ← 反向（关键！）
```

#### Python 代码（递归版）

```python
class Solution:
    def isSymmetric(self, root: Optional[TreeNode]) -> bool:
        def isMirror(left: Optional[TreeNode], right: Optional[TreeNode]) -> bool:
            if not left and not right:  return True
            if not left or  not right or left.val != right.val:  return False
            # 外侧：左左 vs 右右；内侧：左右 vs 右左
            return (isMirror(left.left,  right.right)
                and isMirror(left.right, right.left))

        if not root:
            return True
        return isMirror(root.left, root.right)
```

#### Python 代码（迭代 BFS 版，面试备用）

```python
from collections import deque

class Solution:
    def isSymmetric(self, root: Optional[TreeNode]) -> bool:
        if not root:
            return True
        queue = deque([root.left, root.right])   # 成对入队

        while queue:
            left  = queue.popleft()
            right = queue.popleft()

            if not left and not right:
                continue
            if not left or not right or left.val != right.val:
                return False

            # 外侧一对入队，内侧一对入队
            queue.append(left.left);  queue.append(right.right)
            queue.append(left.right); queue.append(right.left)

        return True
```

> **复杂度**：时间 $O(n)$；空间递归版 $O(h)$，BFS 版 $O(n)$（最宽层节点数）。

### 100. 相同的树

#### 核心思路

双树同步递归的**最简基础版**，是 101 和 572 的核心子逻辑。同时从两棵树的根向下走，每步比对当前节点的值和结构。终止条件的三步顺序固定，须烂熟于心。

#### Python 代码

```python
class Solution:
    def isSameTree(self, p: Optional[TreeNode], q: Optional[TreeNode]) -> bool:
        # 终止条件：三步顺序固定
        if not p and not q:   return True    # 都空 = 完全匹配
        if not p or  not q:   return False   # 一空 = 结构不同
        if p.val != q.val:    return False   # 值不同

        # 同向递归：左对左，右对右
        return (self.isSameTree(p.left,  q.left)
            and self.isSameTree(p.right, q.right))
```

#### 终止情况图示

```text
p=○  q=○  且值相同  → 继续向下递归
p=□  q=□            → True   （两棵子树同时走完，完全匹配）
p=□  q=○            → False  （q 比 p 深，结构不同）
p=○  q=□            → False  （p 比 q 深，结构不同）
p.val ≠ q.val       → False  （值不同，立即剪枝）
```

> **复杂度**：时间 $O(\min(m, n))$，遇到第一个差异即剪枝；空间 $O(\min(h_p, h_q))$。

### 572. 另一棵树的子树

#### 核心思路

**复用 100 的逻辑**：对 `root` 的每一个节点，触发一次 `isSameTree(root, subRoot)`。找到即返回 `True`，找不到继续往左右子树递归。本题是"**单树 DFS 遍历**"与"**双树比较**"的直接组合。

```text
isSubtree(root, subRoot) 决策链：
  root 为空？        → return False     （空树不含子树）
  isSameTree 匹配？  → return True      （当前节点完全匹配）
  否则               → 递归左子树 or 递归右子树
```

#### Python 代码

```python
class Solution:
    def isSubtree(self, root: Optional[TreeNode], subRoot: Optional[TreeNode]) -> bool:
        def isSameTree(p: Optional[TreeNode], q: Optional[TreeNode]) -> bool:
            if not p and not q: return True
            if not p or  not q or p.val != q.val: return False
            return isSameTree(p.left, q.left) and isSameTree(p.right, q.right)

        if not root:
            return False                             # 空树不包含任何非空子树
        if isSameTree(root, subRoot):
            return True                              # 当前节点完全匹配，直接返回
        return (self.isSubtree(root.left,  subRoot)
             or self.isSubtree(root.right, subRoot))
```

> **复杂度**：时间 $O(m \times n)$，$m$/$n$ 分别为两树节点数，每个 root 节点最多触发一次 $O(n)$ 的比较；空间 $O(h_m + h_n)$，两层递归栈叠加。
>
> **进阶**：树序列化为字符串后用 KMP 匹配，时间复杂度可降至 $O(m + n)$，面试中暴力解法已足够。

## 统一对比

| 题号 | 题型 | 递归类型 | 遍历时机 | 核心递归调用 |
|------|------|---------|---------|-------------|
| **226** 翻转 | 修改结构 | 单树后序 | 返回前交换 | `swap(root.left, root.right)` |
| **100** 相同 | 判断属性 | 双树同向 | 同向向下 | `same(p.left, q.left) and same(p.right, q.right)` |
| **101** 对称 | 判断属性 | 双树反向 | 反向向下 | `mirror(l.left, r.right) and mirror(l.right, r.left)` |
| **572** 子树 | 查找匹配 | 单树 DFS + 双树 | 逐节点触发 100 | `isSame(root, sub) or sub(root.left) or sub(root.right)` |

**抽象升级链**：

```text
100（双树同向） ──→ 101（双树反向，改传参方向即可）
               └──→ 572（单树 DFS 中每个节点触发 100）

226 是独立分支：单树后序修改，不涉及双树比较
```

## 口诀

```text
树属性四题速记：

  后序翻转改结构，翻后再交换；          ← 226：后序 swap，中序不可用
  双树同向比相同，左配左右配右；        ← 100：同向递归
  双树反向比镜像，外外内内对；          ← 101：反向（外=左左/右右，内=左右/右左）
  子树遍历触比较，每点调百题；          ← 572：单树 DFS + 每点 isSameTree

终止条件三步（顺序固定）：
  ① 都空 → True
  ② 一空 → False
  ③ 值异 → False
  ④ 否则递归

三个易错点：
  中序不翻树（左翻完再交换，右侧已是原左，翻了两遍）
  对称别同向（方向写反 = 判断相同树，不是对称）
  子树先判空（root=None 先返 False，再触发比较）
```

## 总结与扩展

这四道题覆盖了树形递归的两种基本范式：**单树后序修改**（226）和**双树同步对比**（100/101/572）。掌握两套模板后，解题流程简化为三步：确认终止条件（都空/一空/值异，顺序不变）→ 确认递归方向（同向/反向/逐节点触发）→ 合并子问题结果（`and` 要求全部满足，`or` 找到一个即可）。

572 复用 100 的逻辑是"**函数组合**"思维的典型案例：大问题（在树中查找匹配子树）通过组合已有小工具（判断两树是否相同）来解决，无需从头设计新逻辑。这种拆分方式在树题中普遍适用。
