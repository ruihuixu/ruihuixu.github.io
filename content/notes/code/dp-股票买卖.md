## 识别信号

> 看到什么特征，想到这篇笔记？

- 题目给出「每日股票价格数组」，求「最大利润」
- 操作为买入 / 卖出，同一时刻最多持有一只股票
- 每天持仓状态只有两种：**持有**（已花钱在等）和**不持有**（空仓）
- 出现「只能买一次」「不限次数」「最多 k 次」「冷冻期」「手续费」等约束变体

## 核心模板（速查）

每天状态只有两格，转移各看两条路，六题共用同一骨架——区别仅在 `[买入来源]` 和状态数量：

```python
# 核心状态机（以 122 不限次数为基础）
# dp[i][0]：第 i 天不持有股票，账户最大现金
# dp[i][1]：第 i 天持有股票，账户最大现金（已花钱，所以初始为负）

dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])   # 不持有
dp[i][1] = max(dp[i-1][1], [买入来源] - prices[i])    # 持有
#                                ↑ 六题唯一区别在这里

# 初始化
dp[0][0] = 0             # 第 0 天不持有，利润为 0
dp[0][1] = -prices[0]   # 第 0 天买入，花掉 prices[0]

# 答案：最后一天不持有时，利润必然最大
return dp[n-1][0]
```

### 六题变化对比表

| 题号 | 买卖限制 | 状态数 | `[买入来源]` | 其他变化 | 返回值 |
|------|---------|--------|-------------|---------|--------|
| 121 | 只买一次 | 2 | `-prices[i]`（不带前置利润） | — | `dp[n-1][0]` |
| 122 | 不限次数 | 2 | `dp[i-1][0]`（带前置利润） | — | `dp[n-1][0]` |
| 123 | 最多两次 | 5 | 两套买卖状态依次推进 | 答案取最后一个状态 | `dp[n-1][4]` |
| 188 | 最多 k 次 | 2k+1 | 奇数下标：`dp[i-1][j-1] - prices[i]` | 偶数下标做卖出 | `dp[n-1][2k]` |
| 309 | 冷冻期 | 4 | 来源限定为"可买"或"冷冻结束" | 返回三不持有状态最大 | `max(dp[n-1][1..3])` |
| 714 | 手续费 | 2 | `dp[i-1][0]`（同 122） | 卖出时额外 `-fee` | `dp[n-1][0]` |

## ⚠️ 高频错误

**1. 121 买入公式误用了 122 的写法**

121 只允许一笔交易，买入时账户里没有任何前置利润，买入公式必须固定为 `-prices[i]`。

```python
# ❌ 错误：带入前面交易的利润（这是 122 不限次数的写法）
dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])

# ✅ 正确：只能买一次，买入时手上只有 0 元
dp[i][1] = max(dp[i-1][1], -prices[i])
```

两者差异：若 `dp[i-1][0] > 0`（账户已有前次利润），错误写法会把它叠加进持仓成本，
隐式地允许了多次交易，结果偏高。

---

**2. 123 初始化只写了 `dp[0][1]`，漏掉 `dp[0][3]`**

123 题有两个买入状态（下标 1 = 第一次持有，下标 3 = 第二次持有），第 0 天都需要初始化。

```python
# ❌ 错误：dp[0][3] 未初始化，保持默认值 0，比真实值 -prices[0] 偏高
dp[0][1] = -prices[0]
# dp[0][3] 漏写，后续第二次买入状态从 0 出发，答案偏高

# ✅ 正确：第 0 天可理解为"买入立刻卖出再买入"，净效果同样是 -prices[0]
dp[0][1] = -prices[0]
dp[0][3] = -prices[0]
```

188 题推广：所有奇数下标都需初始化为 `-prices[0]`：

```python
for j in range(1, 2 * k + 1, 2):
    dp[0][j] = -prices[0]
```

---

**3. 309 返回值只取一个不持有状态**

309 题有三种不持有状态（可买入 / 今日刚卖出 / 冷冻期中），最后一天可以停在任意一种，
漏掉任何一种都可能丢失最优解。

```python
# ❌ 错误：只返回"可买"状态，遗漏"今日卖出"和"冷冻期"两种终态
return dp[n-1][1]

# ✅ 正确：三种不持有状态都可能是最终最优解
return max(dp[n-1][1], dp[n-1][2], dp[n-1][3])
```

---

**4. 714 手续费加在买入和卖出两处，重复扣费**

手续费每笔交易只扣一次，位置必须统一——只减在卖出或只减在买入，不能混用。

```python
# ❌ 错误：买入和卖出各减一次，等于每笔交易重复扣两倍手续费
dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i] - fee)   # 卖出减
dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i] - fee)   # 买入又减

# ✅ 正确：统一在卖出时扣，位置唯一，逻辑清晰
dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i] - fee)
dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])
```

## 详解

### LeetCode 121：只买卖一次

#### 核心思路

「最多一笔交易」意味着买入时账户里只有初始的 0 元，不存在任何前置利润可叠加。
买入公式固定为 `-prices[i]`，即直接花当前价格买入，不带前面任何收益。

这是最基础的状态机版本，理解这一题，才能理解 121→122 买入来源的核心差别。

#### Python 代码

```python
class Solution:
    def maxProfit(self, prices: list[int]) -> int:
        n = len(prices)
        # dp[i][0]: 第 i 天不持有股票的最大利润
        # dp[i][1]: 第 i 天持有股票的最大利润
        dp = [[0, 0] for _ in range(n)]
        dp[0][0] = 0            # 第 0 天不持有，利润为 0
        dp[0][1] = -prices[0]  # 第 0 天买入，花掉 prices[0]

        for i in range(1, n):
            # 不持有：继续观望 or 今天卖出
            dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])
            # 持有：继续持有 or 今天买入（不能带前置利润，固定为 -prices[i]）
            dp[i][1] = max(dp[i-1][1], -prices[i])

        return dp[n-1][0]  # 最后一天不持有，利润最大
```

时间复杂度 O(n)，空间复杂度 O(n)。

#### 图解（prices = [7, 1, 5, 3, 6, 4]）

```text
       i=0    i=1    i=2    i=3    i=4    i=5
price:  7      1      5      3      6      4

dp[·][0] (不持有):
        0      0      4      4      5      5
                     ↑             ↑
                  -1+5=4        -1+6=5

dp[·][1] (持有):
       -7     -1     -1     -1     -1     -1
              ↑
           max(-7,-1)=-1   （第1天价格1，持有成本变小）

答案 dp[5][0] = 5（第 1 天买 1，第 4 天卖 6，利润 5）
```

### LeetCode 122：不限次数买卖

#### 核心思路

允许多次交易，买入前可能已有前次利润在账户里。买入来源从固定的 `0` 改为
`dp[i-1][0]`（昨天不持有时的最大现金），自然叠加前置收益。

与 121 唯一区别：持有的买入来源从 `-prices[i]` 改为 `dp[i-1][0] - prices[i]`。

#### Python 代码

```python
class Solution:
    def maxProfit(self, prices: list[int]) -> int:
        n = len(prices)
        dp = [[0, 0] for _ in range(n)]
        dp[0][0] = 0
        dp[0][1] = -prices[0]

        for i in range(1, n):
            # 不持有：继续观望 or 今天卖出
            dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])
            # 持有：继续持有 or 今天买入（带入前置利润，允许多次交易）
            dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])

        return dp[n-1][0]
```

时间复杂度 O(n)，空间复杂度 O(n)。

### LeetCode 123：最多两次交易

#### 核心思路

两次买卖形成 5 个阶段，状态编号 0~4，只能从低编号向高编号单向推进：

```text
状态 0：无操作（初始，始终为 0）
状态 1：第一次持有（第一次买入后）
状态 2：第一次卖出（第一次卖出后，等待或结束）
状态 3：第二次持有（第二次买入后）
状态 4：第二次卖出（最终目标状态）
```

每个状态的转移逻辑统一：**继续保持当前状态** vs **执行操作进入下一状态**，取最大。

#### Python 代码

```python
class Solution:
    def maxProfit(self, prices: list[int]) -> int:
        n = len(prices)
        dp = [[0] * 5 for _ in range(n)]
        dp[0][1] = -prices[0]   # 第 0 天第一次买入
        dp[0][3] = -prices[0]   # 第 0 天视为"买卖再买"，净效果同样是 -prices[0]

        for i in range(1, n):
            dp[i][0] = dp[i-1][0]                                # 无操作，始终为 0
            dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])  # 第一次买入
            dp[i][2] = max(dp[i-1][2], dp[i-1][1] + prices[i])  # 第一次卖出
            dp[i][3] = max(dp[i-1][3], dp[i-1][2] - prices[i])  # 第二次买入
            dp[i][4] = max(dp[i-1][4], dp[i-1][3] + prices[i])  # 第二次卖出

        return dp[n-1][4]  # 第二次卖出后，利润最大
```

时间复杂度 O(n)，空间复杂度 O(n)。

#### 状态转移示意

```text
dp[·][0] ──买入──> dp[·][1] ──卖出──> dp[·][2] ──买入──> dp[·][3] ──卖出──> dp[·][4]
  (无)             (第一次持有)        (第一次卖出)        (第二次持有)        (第二次卖出)
  自循环              自循环              自循环              自循环              自循环
```

### LeetCode 188：最多 k 次交易

#### 核心思路

123 题的直接推广：固定 5 个状态 → `2k+1` 个状态。规律统一：
**奇数下标 = 持有（买入），偶数下标 = 不持有（卖出）**，用循环替代手写每个状态。

初始化同样按规律推广：所有奇数下标均初始化为 `-prices[0]`。

#### Python 代码

```python
class Solution:
    def maxProfit(self, k: int, prices: list[int]) -> int:
        n = len(prices)
        if n == 0:
            return 0

        # j=0 无操作; j=1,3,5,... 第1/2/.../k次持有; j=2,4,6,... 第1/2/.../k次卖出
        dp = [[0] * (2 * k + 1) for _ in range(n)]

        # 初始化：所有奇数下标（买入状态）设为 -prices[0]
        for j in range(1, 2 * k + 1, 2):
            dp[0][j] = -prices[0]

        for i in range(1, n):
            for j in range(1, 2 * k + 1):
                if j % 2 == 1:
                    # 奇数：持有状态 — 继续持有 or 从上一卖出状态买入
                    dp[i][j] = max(dp[i-1][j], dp[i-1][j-1] - prices[i])
                else:
                    # 偶数：卖出状态 — 继续不持有 or 从上一持有状态卖出
                    dp[i][j] = max(dp[i-1][j], dp[i-1][j-1] + prices[i])

        return dp[n-1][2 * k]  # 第 k 次卖出后的最大利润
```

时间复杂度 O(kn)，空间复杂度 O(kn)。
令 `k=2` 代入，此代码与 123 题完全等价，可直接验证。

### LeetCode 309：含冷冻期

#### 核心思路

卖出后次日不可买入（冷冻 1 天）。如果只用两状态，「不持有」无法区分"昨天才卖"和
"可以买入"——买入来源会混入非法状态，导致冷冻期约束失效。

解决方案：将「不持有」一分为三，明确每种情况的来源与去向。

#### 四状态定义

```text
状态 0：持有股票（今天买入 or 之前买入一直持有）
状态 1：不持有 + 可以买入（冷冻期已过，两天以上前卖出）
状态 2：不持有 + 今天刚卖出（明天进入冷冻期）
状态 3：不持有 + 今天冷冻期中（昨天刚卖出，今天不能操作）
```

#### Python 代码

```python
class Solution:
    def maxProfit(self, prices: list[int]) -> int:
        n = len(prices)
        if n == 0:
            return 0

        dp = [[0] * 4 for _ in range(n)]
        dp[0][0] = -prices[0]   # 第 0 天持有，必须是当天买入

        for i in range(1, n):
            # 持有：昨天持有继续 / 昨天可买今天买 / 昨天冷冻结束今天买
            dp[i][0] = max(dp[i-1][0],
                           dp[i-1][1] - prices[i],
                           dp[i-1][3] - prices[i])
            # 可买：昨天可买继续观望 / 昨天冷冻结束不操作
            dp[i][1] = max(dp[i-1][1], dp[i-1][3])
            # 今天卖出：昨天持有，今天卖（明天进入冷冻）
            dp[i][2] = dp[i-1][0] + prices[i]
            # 今天冷冻：昨天刚卖出（只能由状态 2 转入）
            dp[i][3] = dp[i-1][2]

        # 最后一天三种不持有状态均可作为终态，取最大
        return max(dp[n-1][1], dp[n-1][2], dp[n-1][3])
```

时间复杂度 O(n)，空间复杂度 O(n)。

#### 状态流转图

```text
                    ┌──────────────────────────────────────────┐
                    │                                          │
  状态1(可买) ─买入─> 状态0(持有) ─卖出─> 状态2(今日卖出) ─次日─> 状态3(冷冻期)
      ↑  自循环         ↑  自循环                                     │
      │                │                                             │
      └─────────────── 冷冻期结束，可重新买入 ──────────────────────┘
```

### LeetCode 714：含手续费

#### 核心思路

与 122 题框架完全相同，仅在卖出时额外减去手续费 `fee`。手续费位置固定在卖出公式，
避免与买入公式混用导致重复扣费（见高频错误第 4 条）。

#### Python 代码

```python
class Solution:
    def maxProfit(self, prices: list[int], fee: int) -> int:
        n = len(prices)
        dp = [[0, 0] for _ in range(n)]
        dp[0][0] = 0
        dp[0][1] = -prices[0]   # 第 0 天买入

        for i in range(1, n):
            # 不持有：继续观望 or 今天卖出（卖出时扣手续费）
            dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i] - fee)
            # 持有：继续持有 or 今天买入（带入前置利润，不限次数）
            dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])

        return dp[n-1][0]
```

时间复杂度 O(n)，空间复杂度 O(n)。
与 122 代码仅一处差异：卖出公式中 `+ prices[i]` 改为 `+ prices[i] - fee`。

## 统一对比

| 题号 | 限制 | 状态数 | 买入公式 | 卖出公式 | 返回值 |
|------|------|--------|----------|----------|--------|
| 121 | 只买一次 | 2 | `max(hold, -prices[i])` | `max(cash, hold + prices[i])` | `dp[n-1][0]` |
| 122 | 不限次数 | 2 | `max(hold, cash - prices[i])` | `max(cash, hold + prices[i])` | `dp[n-1][0]` |
| 123 | 最多两次 | 5 | 状态1,3：前一状态 `-prices[i]` | 状态2,4：前一状态 `+prices[i]` | `dp[n-1][4]` |
| 188 | 最多 k 次 | 2k+1 | 奇数：`max(dp[j], dp[j-1] - price)` | 偶数：`max(dp[j], dp[j-1] + price)` | `dp[n-1][2k]` |
| 309 | 冷冻期 | 4 | 来源限定状态1或状态3 | 转入状态2，次日为状态3 | `max(dp[n-1][1..3])` |
| 714 | 手续费 | 2 | `max(hold, cash - prices[i])` | `max(cash, hold + prices[i] - fee)` | `dp[n-1][0]` |

**核心演化链**：

- 121 → 122：买入来源从 `0`（固定无利润）改为 `dp[i-1][0]`（允许叠加前置利润）
- 122 → 123/188：二维状态展开为多维，枚举每一次买入/卖出的阶段
- 122 → 309：不持有状态一分为三，约束买入来源，排除冷冻期中的非法买入
- 122 → 714：卖出公式额外减 `fee`，其余框架不变

## 总结与扩展

六道股票题本质上是同一状态机的六种约束变体，约束类型决定改动位置：

| 约束类型 | 影响位置 |
|---------|---------|
| 交易次数限制（1 / 2 / k） | 买入公式 + 状态维度扩展 |
| 冷冻期 | 不持有状态一分为三，买入来源受限 |
| 手续费 | 卖出公式额外减 `fee` |

**学习路径**：先吃透 122（基础两状态框架）→ 对比 121（理解次数限制对买入公式的影响）
→ 扩展 123 / 188（将固定状态通用化）→ 学习 309（状态细分的思路）→ 最后 714（一行变体，最简）。

**相关延伸题**：
- [LC 1014. 最佳观光组合](https://leetcode.cn/problems/best-sightseeing-pair/)：同类「拆分式」状态机，`preMax + scores[i]` 与本题买入来源思路完全一致
- [LC 1911. 最大子序列交替和](https://leetcode.cn/problems/maximum-alternating-subsequence-sum/)：持有 / 不持有交替转移，与 122 题结构同构
- 进阶：若手续费按成交额百分比收取，卖出公式改为 `prices[i] * (1 - rate) + hold`，状态转移逻辑不变
