## 识别信号

看到下列特征，选对框架事半功倍：

- **求树的高度 / 节点计数 / 判平衡** → 后序 DFS，子树先返回数字，父节点汇总
- **从根到叶的路径（收集或判断）** → 前序 DFS + 回溯，路径列表随节点进出同步增删
- **利用特殊树结构加速**（完全二叉树） → 左高 == 右高时直接套满树公式，跳过遍历
- **找某层特定元素**（最后一行第一个） → BFS 层序，逐层结算比 DFS 更直接

## 核心模板（速查）

后序 DFS 深度骨架（559 / 222 / 110 均由此演化）：

```python
def dfs(node):
    if not node:
        return 0                         # 空节点：基准深度 0
    left  = dfs(node.left)              # 1. 先拿左子树结果
    right = dfs(node.right)             # 2. 再拿右子树结果
    return max(left, right) + 1         # 3. 汇总后向上返回
```

三类解法速查：

| 类型 | 信息流方向 | 遍历顺序 | 函数返回值 | 代表题 |
|------|-----------|---------|-----------|--------|
| 深度 / 属性计算 | 子 → 父（向上传数字） | 后序 | `int` | 559 / 222 / 110 |
| 路径收集 / 判断 | 父 → 子（向下传列表） | 前序 + 回溯 | `None` | 257 / 404 / 112 / 113 |
| 按层查找 | 逐层广搜 | BFS | — | 513 |

## ⚠️ 高频错误（先看这里）

### 错误 1：559 — `max()` 对空序列报 ValueError

```python
# ❌ children 为空列表时，generator 无元素，max() 直接抛出 ValueError
return max(self.maxDepth(c) for c in root.children) + 1

# ✅ 加 default=0，叶节点的空 children 视作高度 0
return max((self.maxDepth(c) for c in root.children), default=0) + 1
```

### 错误 2：222 — 高度定义混淆导致公式错误

```python
# ❌ 高度从 0 开始计数（边数），但套了节点数公式，差 1
left_h = 0          # 根节点高度 = 0（边数定义）
if left_h == right_h:
    return (1 << left_h) - 1    # 公式错：应为 (1 << (left_h + 1)) - 1

# ✅ 统一用节点数高度（根节点 = 1），公式始终是 2^h - 1
left_h = 0
while left_node:
    left_h += 1                 # 根节点 = 1
    left_node = left_node.left
if left_h == right_h:
    return (1 << left_h) - 1   # 正确：2^left_h - 1
```

### 错误 3：110 — 收到 -1 哨兵后未及时短路

```python

right_h = get_height(node.right)
if abs(left_h - right_h) > 1: return -1    # left_h 可能已经是 -1！

# ✅ 每次收到 -1 立即透传，不再做无效计算
left_h = get_height(node.left)
if left_h == -1: return -1                 # 提前短路
right_h = get_height(node.right)
if right_h == -1: return -1                # 提前短路
if abs(left_h - right_h) > 1: return -1
```

### 错误 4：257 / 113 — 收集路径时存引用而非副本

```python

# ✅ 切片拷贝，存当前快照
```

### 559. N 叉树的最大深度

#### 核心思路

与二叉树最大深度（104 题）完全相同，唯一区别是把 `left` / `right` 两个指针换成 `children` 列表。后序遍历：对所有子树深度取 max，再加 1。

#### Python 代码

```python
class Solution:
    def maxDepth(self, root: 'Node') -> int:
        if not root:
            return 0
        # default=0：children 为空时（叶节点）max() 不报错
        return max((self.maxDepth(c) for c in root.children), default=0) + 1
```

#### 图解

```text
      1
    / | \
   3  2  4          maxDepth(1):
  / \                 maxDepth(3) = 2
 5   6                maxDepth(2) = 1
                      maxDepth(4) = 1
                    → max(2, 1, 1) + 1 = 3  ✓
```

### 222. 完全二叉树的节点个数

#### 核心思路

暴力后序遍历是 O(n)。利用完全二叉树的性质可优化到 O(log²n)：

1. 从当前节点分别沿**最左路径**和**最右路径**各自统计高度
2. 若两高度相等 → 该子树是**满二叉树** → 节点数 = `2^h - 1`，直接返回，无需遍历
3. 若不等 → 递归左右子树（递归过程中必有一棵子树命中条件 2）

#### Python 代码

```python
class Solution:
    def countNodes(self, root: Optional[TreeNode]) -> int:
        if not root:
            return 0
        left_h = right_h = 0
        left_node, right_node = root, root
        while left_node:                        # 沿最左路径统计高度
            left_h += 1
            left_node = left_node.left
        while right_node:                       # 沿最右路径统计高度
            right_h += 1
            right_node = right_node.right
        if left_h == right_h:                   # 满二叉树，套公式
            return (1 << left_h) - 1
        # 未填满：递归左右子树，+1 计当前节点
        return self.countNodes(root.left) + self.countNodes(root.right) + 1
```

#### 图解

```text
满二叉树（7 节点）：         未填满（5 节点）：
      1                           1
     / \                         / \
    2   3                       2   3
   / \ / \                     / \
  4  5 6  7                   4   5

left_h=3, right_h=3           left_h=3, right_h=2
→ 相等 → 2^3-1 = 7  ✓         → 不等 → 递归左右子树
```

### 110. 平衡二叉树

#### 核心思路

用 `-1` 作哨兵值：递归函数正常情况返回子树高度（≥ 0），一旦发现不平衡就返回 `-1`。父节点收到 `-1` 后立即短路向上透传，避免重复计算，整体仅 O(n) 一次扫描。

#### Python 代码

```python
class Solution:
    def isBalanced(self, root: Optional[TreeNode]) -> bool:
        def height(node: Optional[TreeNode]) -> int:
            if not node:
                return 0
            left = height(node.left)
            if left == -1: return -1             # 左子树已不平衡，立即透传
            right = height(node.right)
            if right == -1: return -1            # 右子树已不平衡，立即透传
            if abs(left - right) > 1: return -1  # 当前节点高度差超标
            return max(left, right) + 1          # 平衡，返回真实高度
        return height(root) != -1
```

#### 图解

```text
不平衡示例：           平衡示例：
    1                      3
   /                      / \
  2                       9  20
 /                          /  \
3                          15   7

height(3)=1               所有节点左右高度差 ≤ 1
height(2): |1-0|=1 → 2   → height(3) 返回 3（非 -1）
height(1): |2-0|=2 → -1  → isBalanced = True
→ isBalanced = False
```

### 257. 二叉树的所有路径

#### 核心思路

前序 DFS：先将当前节点加入路径，到叶节点时收集结果，离开节点时弹出（回溯）。`append → 递归 → pop` 三步必须配对，缺少 `pop` 就不是回溯而是单向 DFS。

#### Python 代码

```python
class Solution:
    def binaryTreePaths(self, root: Optional[TreeNode]) -> List[str]:
        result = []
        def dfs(node: Optional[TreeNode], path: List[int]) -> None:
            path.append(node.val)                         # 前序：先加入当前节点
            if not node.left and not node.right:          # 叶节点：收集路径
                result.append("->".join(map(str, path)))
            if node.left:  dfs(node.left, path)
            if node.right: dfs(node.right, path)
            path.pop()                                    # 回溯：离开时弹出
        if root:
            dfs(root, [])
        return result
```

#### 图解

```text
    1
   / \
  2   3
   \
    5

路径追踪（path 变化）：
  进 1 → [1]
    进 2 → [1,2]
      进 5 → [1,2,5] → 叶节点 → 收集 "1->2->5"
      出 5 → [1,2]
    出 2 → [1]
    进 3 → [1,3] → 叶节点 → 收集 "1->3"
    出 3 → [1]
  出 1 → []
```

### 404. 左叶子之和

#### 核心思路

节点本身无法判断自己是不是"左叶子"，必须由**父节点**来判断。解法：递归时传入布尔标志 `is_left`，到达叶节点时根据标志决定是否计入。根节点调用时传 `False`，因为根节点不属于任何人的左子树。

#### Python 代码

```python
class Solution:
    def sumOfLeftLeaves(self, root: Optional[TreeNode]) -> int:
        def dfs(node: Optional[TreeNode], is_left: bool) -> int:
            if not node:
                return 0
            if not node.left and not node.right:          # 叶节点
                return node.val if is_left else 0
            # 递归左孩子时传 True，递归右孩子时传 False
            return dfs(node.left, True) + dfs(node.right, False)
        return dfs(root, False)
```

#### 图解

```text
    3
   / \
  9  20
    /  \
   15   7

dfs(3, False)
  dfs(9, True)    → 叶节点 + is_left=True  → 返回 9
  dfs(20, False)
    dfs(15, True) → 叶节点 + is_left=True  → 返回 15
    dfs(7, False) → 叶节点 + is_left=False → 返回 0
  → 9 + 15 + 0 = 24  ✓
```

### 513. 找树左下角的值

#### 核心思路

BFS 最直观：层序遍历，每层只记录**第一个节点**的值（`i == 0` 时更新 `result`）。遍历结束后 `result` 自然是最后一层第一个节点的值，不需要额外的"最左"判断。

#### Python 代码

```python
from collections import deque

class Solution:
    def findBottomLeftValue(self, root: Optional[TreeNode]) -> int:
        queue = deque([root])
        result = root.val
        while queue:
            level_size = len(queue)
            for i in range(level_size):
                node = queue.popleft()
                if i == 0:
                    result = node.val         # 每层只取第一个节点
                if node.left:  queue.append(node.left)
                if node.right: queue.append(node.right)
        return result
```

#### 图解

```text
      2
     / \
    1   3
   / \
  4   5
     /
    6

BFS 逐层处理：
  第 1 层 → result=2
  第 2 层 → result=1  （第一个节点）
  第 3 层 → result=4  （第一个节点）
  第 4 层 → result=6  （第一个节点，最后一层）

返回 6  ✓
```

### 112. 路径总和

#### 核心思路

前序 DFS，向下传递"剩余目标值"：每经过一个节点就减去该节点的值。到达叶节点时若 `remaining == 0` 则找到目标路径。用 `or` 的短路特性实现"找到一条立即停止"，不需要额外的标志变量。

**注意**：必须在**叶节点**处判断，不能在中间节点提前判断——中间节点的值也可能恰好让 `remaining` 归零，但此时路径还未走完。

#### Python 代码

```python
class Solution:
    def hasPathSum(self, root: Optional[TreeNode], targetSum: int) -> bool:
        if not root:
            return False
        remaining = targetSum - root.val
        if not root.left and not root.right:    # 叶节点
            return remaining == 0
        # or 短路：左侧找到答案则右侧不再搜索
        return (self.hasPathSum(root.left, remaining) or
                self.hasPathSum(root.right, remaining))
```

#### 图解

```text
targetSum = 22，树如下：
      5
     / \
    4   8
   /   / \
  11  13   4
 / \        \
7   2        1

路径 5→4→11→2：remaining 变化
  22 → 17 → 6 → -5 → 0  ← 叶节点，remaining==0 → True  ✓

中间节点陷阱：若节点值恰好等于当前 remaining，但该节点不是叶节点，不能提前返回 True
```

### 113. 路径总和 II

#### 核心思路

与 112 的两个关键区别：
1. 需要遍历**全部**路径，不能用 `or` 短路，左右子树都要递归
2. 收集时必须用 `path[:]` 拷贝快照，直接 `append(path)` 存的是引用，回溯时会被修改

#### Python 代码

```python
class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> List[List[int]]:
        result = []
        def dfs(node: Optional[TreeNode], remaining: int, path: List[int]) -> None:
            if not node:
                return
            path.append(node.val)
            remaining -= node.val
            if not node.left and not node.right and remaining == 0:
                result.append(path[:])          # 必须拷贝！不能存引用
            dfs(node.left, remaining, path)     # 左右都要递归，不能短路
            dfs(node.right, remaining, path)
            path.pop()                          # 回溯
        dfs(root, targetSum, [])
        return result
```

#### 图解

```text
112 vs 113 核心差异：

| 维度         | 112（判断存在）         | 113（收集所有路径）        |
|-------------|----------------------|--------------------------|
| 递归返回值   | bool                 | None（用 result 收集）    |
| 左右子树处理 | left or right（短路） | 分别调用，不能短路         |
| 路径存储     | 不需要                | result.append(path[:])   |
| 叶节点逻辑   | return True/False    | if remaining==0 则收集   |
```

## 总结与扩展

**三条主线**将八道题串联：

**主线一：后序 DFS 传数字**（559 / 222 / 110）
子树先算，结果向上汇报。559 是最基础的形式，`children` 列表只是二叉树的推广；222 利用完全二叉树的结构把 O(n) 优化到 O(log²n)，满树公式是核心；110 用 `-1` 哨兵巧妙地在一次遍历中同时完成"求高度"和"判平衡"两件事。

**主线二：前序 DFS + 回溯**（257 / 404 / 112 / 113）
状态向下携带，叶节点处理，返回时撤销。`append → 递归 → pop` 是路径类题目的固定三件套。404 的 `is_left` 标志演示了"父节点给子节点打标签"这一通用思路；113 的 `path[:]` 是防引用污染的必备写法。

**主线三：BFS 层序**（513）
当问题天然按层定义（"最后一行"）时，BFS 比 DFS 直接。513 只需每层更新首节点值，代码逻辑自然正确，不需要任何特殊的"最左"判断。
