## 识别信号

> 看到什么特征，想到这篇笔记？

- 题目要求实现一个链表类，支持按索引增、删、查多种操作
- 头部插入和中间插入看起来需要写两套逻辑 → 虚拟头节点可以统一
- 合法索引范围因操作类型不同：增允许 `index == size`，查/删必须 `index < size`
- 每次增删后链表长度变化，需要 O(1) 快速访问当前长度

---

## 核心模板（速查）

### Node 类 + 虚拟头节点模式

```python
class Node:
    def __init__(self, val: int = 0):
        self.val = val
        self.next: 'Node | None' = None


class MyLinkedList:
    def __init__(self):
        self.dummy = Node()  # 虚拟头节点，val 无意义，仅作哨兵
        self.size = 0        # 链表真实长度，O(1) 获取，所有边界判断依赖它
```

核心设计：**所有增删操作统一归结为「找前驱 pred → 改 pred.next」**。虚拟头节点保证链表头也有前驱（dummy 即为头节点的前驱），彻底消除头部特殊情况。

### 各操作速查表

| 操作 | 前驱步数 | 合法边界 | size 变化 |
|------|---------|---------|---------|
| `get(index)` | index + 1 步（定位目标本身） | 0 ≤ index < size | 不变 |
| `addAtHead(val)` | 转发给 `addAtIndex(0, val)` | — | +1 |
| `addAtTail(val)` | 转发给 `addAtIndex(size, val)` | — | +1 |
| `addAtIndex(index, val)` | index 步（定位前驱） | index ≤ size；index < 0 → 按 0 处理 | +1 |
| `deleteAtIndex(index)` | index 步（定位前驱） | 0 ≤ index < size | -1 |

### 插入与删除的原子操作

```python
# ── 插入（两步，顺序不能颠倒）──
to_add.next = pred.next   # 第一步：新节点先接上后继，防止后继引用丢失
pred.next = to_add        # 第二步：前驱再改为指向新节点

# ── 删除（一步）──
pred.next = pred.next.next  # 跳过目标节点，直接连到其后继
```

## ⚠️ 高频错误（先看这里）

**1. get/delete 边界写成 `index > size - 1` 或 `index > size`**

```python
# ❌ 错误：index == size 时条件不触发，cur.next 走到 None，None.val 报 AttributeError
if index < 0 or index > self.size - 1:
    return -1

# ✅ 正确：链表下标从 0 开始，最大合法下标是 size-1，等价于 < size
if index < 0 or index >= self.size:
    return -1
```

**2. 插入时指针赋值顺序写反**

```python
# ❌ 错误：先改 pred.next，后继引用立刻丢失，to_add 与后续节点脱节
pred.next = to_add
to_add.next = pred.next   # 此时 pred.next 已经是 to_add，相当于 to_add.next = to_add（自环！）

# ✅ 正确：永远先保住还需要访问的引用，再做指针改动
to_add.next = pred.next   # 先接后继
pred.next = to_add        # 再接前驱
```

**3. addAtIndex 漏处理 `index < 0`**

```python
# ❌ 错误：只判断了上界，负数 index 直接进入 for 循环
def addAtIndex(self, index: int, val: int) -> None:
    if index > self.size:
        return
    # 漏了下面这行，range(负数) 不执行，表面正确但不符合题意
    pred = self.dummy
    for _ in range(index):
        pred = pred.next

# ✅ 正确：题意规定 index < 0 时等价于头插（按 index = 0 处理）
    if index < 0:
        index = 0
```

**4. 增删操作后忘记同步 size**

```python
# ❌ 错误：指针改了但 size 没变，后续所有边界判断连锁失效
pred.next = to_add

# ✅ 正确：每次成功增删立刻同步 size，缺一不可
pred.next = to_add
self.size += 1
```

## 详解

### 初始化

#### 核心思路

链表需要两个成员：

- `dummy`：虚拟头节点，永远不存真实数据，作为哨兵使得真实头节点也有前驱可寻
- `size`：链表当前长度，O(1) 获取，是全部边界判断的唯一依据

#### Python 代码

```python
def __init__(self):
    self.dummy = Node()  # 虚拟头节点，val=0 但无实际含义
    self.size = 0        # 空链表长度为 0
```

#### 图解

```text
初始状态：
dummy → None
size  = 0
```

虚拟头节点的意义：无论链表是否为空，始终有 `dummy.next = 真实头节点（或 None）`，所有操作的出发点统一为 `dummy`，不需要判断链表是否为空。

---

### get(index)

#### 核心思路

- 越界（`index < 0` 或 `index >= size`）直接返回 -1
- 否则从 `dummy` 出发走 `index + 1` 步，到达第 index 个节点，返回其值
- 走 `index + 1` 步而非 `index` 步，是因为出发点是 dummy（不算真实节点），需要多走一步越过它

#### Python 代码

```python
def get(self, index: int) -> int:
    if index < 0 or index >= self.size:
        return -1                   # 越界返回 -1
    cur = self.dummy
    for _ in range(index + 1):     # 从 dummy 走 index+1 步到达目标节点
        cur = cur.next
    return cur.val
```

#### 图解

```text
链表：[1] → [2] → [3]，get(1)

dummy → [1] → [2] → [3] → None
  ↑
 cur（出发点）

走 1 步：cur → [1]
走 2 步：cur → [2]  ← 返回 cur.val = 2
```

复杂度：时间 O(index)，空间 O(1)

---

### addAtHead(val)

#### 核心思路

头插 = 在第 0 个位置之前插入，等价于 `addAtIndex(0, val)`。直接转发，不单独实现，遵循 DRY 原则，同时避免额外维护一套边界逻辑。

#### Python 代码

```python
def addAtHead(self, val: int) -> None:
    self.addAtIndex(0, val)  # 头插转发给通用插入
```

---

### addAtTail(val)

#### 核心思路

尾插 = 在第 size 个位置之前插入（追加到末尾），等价于 `addAtIndex(size, val)`。同样转发给通用插入。

#### Python 代码

```python
def addAtTail(self, val: int) -> None:
    self.addAtIndex(self.size, val)  # 尾插转发给通用插入
```

---

### addAtIndex(index, val)

#### 核心思路

1. `index > size`：插入位置超出链表范围，直接返回，不做任何操作
2. `index < 0`：题意规定当作 0 处理（等价于头插）
3. 从 `dummy` 出发走 `index` 步，到达前驱节点 `pred`
4. 执行两步插入：先接后继（`to_add.next = pred.next`），再接前驱（`pred.next = to_add`）
5. `size += 1`

`addAtHead` 和 `addAtTail` 都通过此方法实现，三种插入场景完全统一。

#### Python 代码

```python
def addAtIndex(self, index: int, val: int) -> None:
    if index > self.size:
        return               # 超出范围，不插入
    if index < 0:
        index = 0            # 负数索引等价于头插

    pred = self.dummy
    for _ in range(index):   # 从 dummy 出发走 index 步找前驱
        pred = pred.next

    to_add = Node(val)
    to_add.next = pred.next  # 第一步：新节点先接上原来的后继（顺序关键）
    pred.next = to_add       # 第二步：前驱改为指向新节点
    self.size += 1
```

#### 图解

```text
链表：[1] → [3]，addAtIndex(1, 2)（在位置 1 插入值为 2 的节点）

找前驱（从 dummy 走 1 步）：
dummy → [1] → [3] → None
         ↑
        pred

to_add.next = pred.next（先接后继，保住 [3] 的引用）：
to_add(2) → [3]
dummy → [1] → [3] → None   ← [3] 引用已被 to_add 持有，不会丢失

pred.next = to_add（再让前驱指向新节点）：
dummy → [1] → [2] → [3] → None
```

复杂度：时间 O(index)，空间 O(1)

---

### deleteAtIndex(index)

#### 核心思路

1. 越界（`index < 0` 或 `index >= size`）直接返回，不操作
2. 从 `dummy` 出发走 `index` 步，到达前驱节点 `pred`
3. `pred.next = pred.next.next`：一步跳过目标节点，连到其后继
4. `size -= 1`

#### Python 代码

```python
def deleteAtIndex(self, index: int) -> None:
    if index < 0 or index >= self.size:
        return               # 越界不操作

    pred = self.dummy
    for _ in range(index):   # 从 dummy 出发走 index 步找前驱
        pred = pred.next

    pred.next = pred.next.next  # 跳过目标节点
    self.size -= 1
```

#### 图解

```text
链表：[1] → [2] → [3]，deleteAtIndex(1)（删除位置 1 的节点 2）

找前驱（从 dummy 走 1 步）：
dummy → [1] → [2] → [3] → None
         ↑
        pred

pred.next = pred.next.next（跳过 [2]，直连 [3]）：
dummy → [1] → [3] → None
              ↑
         [2] 被跳过（Python GC 自动回收）
```

复杂度：时间 O(index)，空间 O(1)

### 完整实现

```python
class Node:
    def __init__(self, val: int = 0):
        self.val = val
        self.next: 'Node | None' = None


class MyLinkedList:

    def __init__(self):
        self.dummy = Node()  # 虚拟头节点，不存真实数据
        self.size = 0

    def get(self, index: int) -> int:
        if index < 0 or index >= self.size:
            return -1
        cur = self.dummy
        for _ in range(index + 1):      # 走 index+1 步：越过 dummy 到目标
            cur = cur.next
        return cur.val

    def addAtHead(self, val: int) -> None:
        self.addAtIndex(0, val)         # 头插 = 在位置 0 前插入

    def addAtTail(self, val: int) -> None:
        self.addAtIndex(self.size, val) # 尾插 = 在位置 size 前插入

    def addAtIndex(self, index: int, val: int) -> None:
        if index > self.size:
            return                      # 超界不插入
        if index < 0:
            index = 0                   # 负数当 0 处理（题意）

        pred = self.dummy
        for _ in range(index):          # 找前驱：走 index 步
            pred = pred.next

        to_add = Node(val)
        to_add.next = pred.next         # 先接后继（顺序不能反）
        pred.next = to_add              # 再让前驱指向新节点
        self.size += 1

    def deleteAtIndex(self, index: int) -> None:
        if index < 0 or index >= self.size:
            return                      # 越界不删

        pred = self.dummy
        for _ in range(index):          # 找前驱：走 index 步
            pred = pred.next

        pred.next = pred.next.next      # 跳过目标节点
        self.size -= 1
```

## 口诀

```
虚拟头做哨兵，头插中插都一样
增删查都找前驱，dummy 出发走 index 步

插入两步顺序记：先接后继再接前（顺序反了后继丢）
删除一步记牢了：pred.next 直接跳一格

增的边界最宽松：index 可以等于 size（允许追加到尾）
查删边界严一格：index 必须小于 size（不能取等）
负数 index 别慌张：增操作当 0，查删直接返 -1

每次增删别忘了：size 同步改，边界全靠它
```

## 总结与扩展

`dummy head + size` 是链表操作的标准脚手架。虚拟头节点消除了「头部是否存在」的判断分支，让所有增删统一为「找前驱 → 改一条指针」；`size` 字段把长度变为 O(1) 信息，驱动全部边界守卫，无需遍历链表来感知长度。

写链表操作时始终遵守两个原则：

1. **指针改动前先保住还需要用的引用**：插入时必须先执行 `to_add.next = pred.next`，否则后继节点的引用永久丢失
2. **增删必须同步 size**：size 一旦与真实长度不一致，后续所有边界判断全部连锁失效

**拓展题目**：

- LeetCode 203（移除链表元素）：虚拟头节点统一删除头节点与中间节点，是本文 deleteAtIndex 思路的精简版
- LeetCode 206（反转链表）：同属「先保后继引用，再改指针方向」的原子操作，与插入操作同根同源
- LeetCode 21（合并两个有序链表）：虚拟头节点简化合并的起点处理，无需单独判断哪条链表的头更小
