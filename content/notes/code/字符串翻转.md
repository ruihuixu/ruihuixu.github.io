## 识别信号

> 看到什么特征，想到这篇笔记？

- 要求**反转单词顺序**，但每个单词内部字母顺序保持不变
- 要求将字符串**左旋/右旋** k 位，等价于把前 k 个字符移到末尾
- 题目强调 O(1) 额外空间，不能用切片拼接（`s[k:] + s[:k]`）直接解决
- 输入字符串含有**多余空格**（首尾或单词间连续空格），需要先清洗再操作
- 要求不使用 `split`、`join` 等内置库函数，模拟原地操作

## 核心模板（速查）

### 原子操作：通用翻转函数

所有翻转操作都复用这一个函数：

```python
def reverse(chars: list, left: int, right: int) -> None:
    while left < right:
        chars[left], chars[right] = chars[right], chars[left]
        left += 1
        right -= 1
```

### 两题框架对比

```python
# ── LCR 182：三步翻转，无需数据清洗 ────────────────────────
chars = list(password)
reverse(chars, 0, k - 1)       # 翻转 A 段（前 k 个字符）
reverse(chars, k, n - 1)       # 翻转 B 段（剩余字符）
reverse(chars, 0, n - 1)       # 翻转全体 → AB 变成 BA

# ── LeetCode 151：多一步数据清洗 ─────────────────────────────
chars = list(s)
chars = remove_extra_spaces(chars)      # 快慢指针去多余空格
reverse(chars, 0, len(chars) - 1)      # 整体翻转（单词顺序调对，字母是副作用倒的）
reverse_each_word(chars)               # 逐词翻转（消除副作用，字母恢复正序）
```

### 速查表

| 题号 | 操作顺序 | 额外步骤 | 分割依据 |
|------|---------|---------|---------|
| 151 | 整体翻转 → 逐词翻转 | 快慢指针去空格 | 空格字符 |
| LCR 182 | 翻 A → 翻 B → 翻整体 | 无 | 固定索引 k |

## ⚠️ 高频错误

**1. 去空格时直接过滤所有空格，单词间分隔消失**

```python
# ❌ 错误："hello   world" 变成 "helloworld"，丢失单词边界
chars = [c for c in chars if c != ' ']

# ✅ 正确：快慢指针，遇到新单词前先手动写入一个分隔空格
if slow != 0:           # 非首单词时才补空格
    chars[slow] = ' '
    slow += 1
```

**2. LCR 182 区间 off-by-one：第二段起点写成 k+1**

```python
# ❌ 错误：chars[k] 被遗漏，既不属于 A 段也不属于 B 段，永远不会被翻转
reverse(chars, 0, k - 1)
reverse(chars, k + 1, n - 1)   # 漏掉了 index k

# ✅ 正确：两段无缝衔接，[0, k-1] 和 [k, n-1]
reverse(chars, 0, k - 1)
reverse(chars, k, n - 1)
```

**3. 151 题翻转顺序颠倒：先逐词翻转再整体翻转**

```python
# ❌ 错误：先翻每个单词（字母正序 → 倒序），再整体翻，结果双重倒序一片乱
reverse_each_word(chars)
reverse(chars, 0, n - 1)

# ✅ 正确：先整体翻（单词顺序调对），再逐词翻（消除字母倒序的副作用）
reverse(chars, 0, n - 1)
reverse_each_word(chars)
```

**4. 遗漏最后一个单词的翻转**

```python
# ❌ 错误：循环到 n-1，最后一个单词后面没有空格，永远不触发翻转
for i in range(n):
    if chars[i] == ' ':
        reverse(chars, start, i - 1)
        start = i + 1
# 最后一个单词 [start, n-1] 从未被翻转

# ✅ 正确：循环到 n（含），i == n 时视为边界也触发一次
for i in range(n + 1):
    if i == n or chars[i] == ' ':
        reverse(chars, start, i - 1)
        start = i + 1
```

## 详解

### LeetCode 151：反转字符串中的单词

#### 核心思路

输入含多余空格的字符串，要求单词顺序倒置，单词内部字母不变。

三步流水线：

1. **去空格**：快慢指针清洗，去除首尾及连续空格，保留单词间单个空格
2. **整体翻转**：一次翻转让单词顺序倒置（字母此时倒序，是可接受的副作用）
3. **逐词翻转**：对每个单词单独翻转，精准消除步骤二的副作用

关键洞察：步骤二和步骤三"负负得正"——整体翻转引入字母倒序，逐词翻转恰好抵消。

#### Python 代码

```python
def reverseWords(s: str) -> str:
    chars = list(s)
    n = len(chars)

    # ── 步骤一：快慢指针去多余空格 ──────────────────────────────
    slow = 0   # 写指针，指向下一个有效写入位置
    fast = 0   # 读指针，扫描原始字符

    while fast < n:
        if chars[fast] != ' ':
            # 遇到新单词：非首单词前补一个分隔空格
            if slow != 0:
                chars[slow] = ' '
                slow += 1
            # 逐字母复制整个单词
            while fast < n and chars[fast] != ' ':
                chars[slow] = chars[fast]
                slow += 1
                fast += 1
        else:
            fast += 1   # 跳过空格，不写入

    chars = chars[:slow]  # 截断尾部无效数据
    n = len(chars)

    # ── 步骤二：整体翻转 ──────────────────────────────────────────
    def reverse(left: int, right: int) -> None:
        while left < right:
            chars[left], chars[right] = chars[right], chars[left]
            left += 1
            right -= 1

    reverse(0, n - 1)
    # "hello world" → "dlrow olleh"：单词顺序对了，字母倒序是副作用

    # ── 步骤三：逐词翻转，恢复字母正序 ──────────────────────────
    start = 0
    for i in range(n + 1):
        if i == n or chars[i] == ' ':
            reverse(start, i - 1)  # 翻转当前单词 [start, i-1]
            start = i + 1          # 更新下一个单词的起点

    return ''.join(chars)
```

#### 图解

```text
输入: "  hello   world  "

步骤一：快慢指针去空格
  fast 跳过前置空格，找到 'h'（slow=0，是首单词，不写分隔空格）
  复制 "hello"：slow→5，fast→7
  fast 遇到多余空格，继续移动到 10（'w'）
  slow=5，非首单词，先写 ' '：slow→6
  复制 "world"：slow→11，fast→15
  fast 遇到尾部空格，循环结束
  结果：chars[:11] = "hello world"

步骤二：整体翻转 [0, 10]
  h e l l o   w o r l d
  ↕                   ↕   （对撞交换到中间）
  d l r o w   o l l e h
  → "dlrow olleh"

步骤三：逐词翻转
  i=5 遇到空格：reverse [0,4]  "dlrow" → "world"  → "world olleh"
  i=11 越界：  reverse [6,10] "olleh" → "hello"  → "world hello"  ✓
```

**复杂度**：时间 O(n)，空间 O(n)（Python 需要 list 转换）

### LCR 182：左旋转字符串（动态口令）

#### 核心思路

将字符串前 `target` 个字符移到末尾，例如 `"abcdefg"` 左旋 2 位 → `"cdefgab"`。

数学建模：记 A = 前 k 个字符，B = 剩余字符，目标是 AB → BA。

利用转置公式推导：

```text
反转 A：A → A^T
反转 B：B → B^T
拼合后再反转整体：(A^T B^T)^T = BA   ✓
```

三步操作直接对应公式，无需辅助空间搬运字符。

注意：也可以先整体翻转再翻两段局部，顺序调换后切割点从 k 变为 n-k，结果等价，但本文统一用"先局部后整体"的顺序，与公式推导一致，便于记忆。

#### Python 代码

```python
def dynamicPassword(password: str, target: int) -> str:
    chars = list(password)
    n = len(chars)

    def reverse(left: int, right: int) -> None:
        # 原地翻转闭区间 [left, right]
        while left < right:
            chars[left], chars[right] = chars[right], chars[left]
            left += 1
            right -= 1

    # 三步翻转：先翻两个局部段，再整体翻转还原相对顺序
    reverse(0, target - 1)   # A^T：翻转前 target 个字符
    reverse(target, n - 1)   # B^T：翻转剩余字符
    reverse(0, n - 1)        # (A^T B^T)^T = BA

    return ''.join(chars)
```

#### 图解

```text
输入: password = "abcdefg", target = 2
目标: "cdefgab"

区间划分: A = "ab" [0,1]，B = "cdefg" [2,6]

步骤一：reverse(0, 1) → 翻转 A
  "ab" → "ba"
  字符串: b a | c d e f g
              ↑ 分隔点

步骤二：reverse(2, 6) → 翻转 B
  "cdefg" → "gfedc"
  字符串: b a | g f e d c

步骤三：reverse(0, 6) → 翻转整体
  "bagfedc" → "cdefgab"
  字符串: c d e f g a b  ✓

验证：(A^T B^T)^T = ("ba" + "gfedc")^T = "bagfedc"^T = "cdefgab" = BA  ✓
```

**复杂度**：时间 O(n)，空间 O(n)（list 转换；若输入为 char array 则 O(1)）

## 统一对比

| 维度 | LeetCode 151 | LCR 182 |
|------|-------------|---------|
| 问题类型 | 单词顺序倒置，字母不变 | 前 k 字符移到末尾 |
| 数据清洗 | 必须（快慢指针去空格） | 不需要 |
| 翻转总次数 | 1（整体）+ n/avg_word_len（逐词）≈ 2 趟 | 3 次固定翻转 |
| 分割依据 | 空格字符（动态） | 固定下标 k |
| 主要难点 | 空格清洗逻辑、最后一词边界 | 区间边界 [0,k-1] 与 [k,n-1] |
| 翻转顺序 | 先整体，后逐词（顺序不可颠倒） | 先两段局部，后整体（顺序不可颠倒） |
| 共用代码 | `reverse(left, right)` 完全相同 | 同左 |

**抽象本质**：两题均为"分块重排"问题，统一解法是：

- **分**：找到逻辑断点（空格 / 下标 k），确定块的边界
- **治**：各块内部分别翻转，消除内部顺序的干扰
- **合**：整体翻转一次，将块与块的相对位置调整到目标状态

## 口诀

```text
【字符串翻转口诀】

识别：
  单词换序 → 想 151，先清洗再两次翻转
  前 k 移尾 → 想 LCR 182，三步翻转无需清洗

151 三步：
  一洗：快慢指针去多余空格，首单词前不补空格
  二翻整体：单词顺序到位，字母倒序是副作用
  三翻单词：消除副作用，字母恢复正序

LCR 182 三步：
  翻前段 [0, k-1]
  翻后段 [k, n-1]   ← 起点是 k，不是 k+1
  翻全体 [0, n-1]

核心公式：(A^T B^T)^T = BA
  局部翻转消顺序，整体翻转定位置，负负得正回原样

边界两条：
  两段无缝：[0, k-1] 和 [k, n-1]，索引 k 不能丢
  末尾触发：循环写 i == n，末尾单词不遗漏
```

## 总结与扩展

两题共享"三次翻转"框架，本质都是分块重排。LCR 182 是数学模型的直接实现，对应转置公式 $(A^T B^T)^T = BA$，三步翻转与公式一一对应。151 在此基础上多了数据清洗环节，快慢指针去空格的逻辑与 LeetCode 27（移除元素）完全同源：slow 维护写入边界，fast 负责扫描筛选。掌握 `reverse(left, right)` 这一原子操作后，遇到"不能使用额外空间"的字符串重排题，优先考虑这套分块重排框架。
