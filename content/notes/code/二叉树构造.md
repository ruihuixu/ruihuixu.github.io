## 核心模板（速查）

所有构造题共享同一个三步骨架：

```
找根节点  →  在中序（或数组）中定位切割点  →  递归构造左右子树
```

```python
def build(序列参数) -> Optional[TreeNode]:
    if 终止条件（序列为空）:
        return None

    root_val = 从序列中确定根节点值   # 关键差异点
    root = TreeNode(root_val)

    idx = 在中序/数组中找 root_val 的位置
    left_len = idx 左侧的元素个数

    root.left  = build(左半序列参数)
    root.right = build(右半序列参数)

    return root
```

四题的差异仅在"如何找根"：

| 题号 | 根节点来源 | 切割依据 |
|------|-----------|---------|
| 106 | `postorder[-1]`（后序最后一个） | 在中序中找根的下标，按左子树长度切后序 |
| 105 | `preorder[0]`（前序第一个） | 在中序中找根的下标，按左子树长度切前序 |
| 654 | `max(nums)` | 最大值下标左侧为左子树，右侧为右子树 |
| 617 | 两节点值相加（无需切割） | 同步递归，一方为空即返回另一方 |

## ⚠️ 高频错误（先看这里）

### 错误 1：后序根节点取的是第一个而不是最后一个

```python
# ❌ 后序遍历顺序是「左→右→根」，根在末尾
root_val = postorder[0]

# ✅ 正确
root_val = postorder[-1]
```

### 错误 2：中序切割时下标偏移一位

```python
# ❌ idx 本身是根节点，右子树中序应从 idx+1 开始
root.right = build(inorder[idx:], ...)

# ✅ 跳过根节点
root.right = build(inorder[idx+1:], ...)
```

### 错误 3：后序切割长度没有对齐中序左子树长度

```python
# ❌ 后序的左子树段必须与中序左子树等长，不能随意切
left_len = idx           # 忘记减去 in_left 偏移，索引对不上

# ✅ 左子树节点数 = 中序根下标 - 中序左边界
left_len = idx - in_left
```

### 错误 4：654 未处理空数组导致 max() 报错

```python
# ❌ nums 为空时 max([]) 抛出 ValueError
def constructMaximumBinaryTree(self, nums):
    max_val = max(nums)   # 没有空数组保护

# ✅ 先判空（切片版）或用区间判断（下标版）
if not nums:
    return None
```

### 错误 5：617 合并时新建节点，未复用已有节点

```python
# ❌ 不必要地新建节点，浪费空间
root = TreeNode(root1.val + root2.val)

# ✅ 直接修改 root1，原地合并
root1.val += root2.val
return root1
```

## 详解

### 106. 从中序与后序遍历序列构造二叉树

#### 核心思路

后序遍历顺序是「左 → 右 → 根」，因此 `postorder[-1]` 必然是当前子树的根。拿到根之后，在中序数组中找到根的位置 `idx`，`idx` 左侧为左子树中序、右侧为右子树中序。根据左子树的节点数 `left_len = idx - in_left`，将后序数组同样切分，递归即可。

直接切片会产生大量临时数组（O(n²) 空间），优化做法是用哈希表预存中序下标，配合区间参数实现 O(n) 版本。

#### Python 代码

**切片版（思路清晰，适合理解）**：

```python
from typing import List, Optional

class Solution:
    def buildTree(self, inorder: List[int], postorder: List[int]) -> Optional[TreeNode]:
        if not postorder:
            return None

        root_val = postorder[-1]          # 后序最后一个是根
        root = TreeNode(root_val)

        idx = inorder.index(root_val)     # 在中序中找根的位置

        # 左子树：中序 [:idx]，后序 [:idx]
        root.left  = self.buildTree(inorder[:idx],    postorder[:idx])
        # 右子树：中序 [idx+1:]，后序 [idx:-1]（去掉末尾的根）
        root.right = self.buildTree(inorder[idx+1:], postorder[idx:-1])

        return root
```

**哈希优化版（O(n) 时间，面试推荐）**：

```python
class Solution:
    def buildTree(self, inorder: List[int], postorder: List[int]) -> Optional[TreeNode]:
        in_map = {val: i for i, val in enumerate(inorder)}

        def build(in_left: int, in_right: int,
                  post_left: int, post_right: int) -> Optional[TreeNode]:
            if in_left == in_right:        # 区间为空（左闭右开）
                return None

            root_val = postorder[post_right - 1]   # 后序末尾是根
            root = TreeNode(root_val)

            idx = in_map[root_val]                 # O(1) 查找
            left_len = idx - in_left               # 左子树节点个数

            root.left = build(
                in_left, idx,
                post_left, post_left + left_len
            )
            root.right = build(
                idx + 1, in_right,
                post_left + left_len, post_right - 1
            )
            return root

        return build(0, len(inorder), 0, len(postorder))
```

#### 图解

```
inorder   = [9, 3, 15, 20, 7]
postorder = [9, 15, 7, 20, 3]

步骤 1：postorder[-1] = 3 → 根节点

中序:  [ 9 | 3 | 15  20  7 ]
              ^
          根 3 的位置 idx=1
      左子树      右子树
       [9]       [15, 20, 7]

后序:  [ 9 | 15  7  20 | 3 ]
        ^              ^  ^
    左后序(len=1)  右后序   根

递归左子树: inorder=[9],       postorder=[9]       → 叶节点 9
递归右子树: inorder=[15,20,7], postorder=[15,7,20] → 继续...

结果:
        3
       / \
      9  20
        /  \
       15   7
```

### 105. 从前序与中序遍历序列构造二叉树

#### 核心思路

前序遍历顺序是「根 → 左 → 右」，因此 `preorder[0]` 是当前子树的根。在中序数组中找到根的位置 `idx` 后，左子树中序为 `inorder[:idx]`，长度 `left_len = idx`（若从零开始）。前序数组跳过第一个根节点，按 `left_len` 切分左右段。

105 与 106 完全对称，区别只有两点：根从末尾改为开头，切前序时起点要 `+1` 跳过根。

#### Python 代码

**切片版**：

```python
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        if not preorder:
            return None

        root_val = preorder[0]             # 前序第一个是根
        root = TreeNode(root_val)

        idx = inorder.index(root_val)      # 在中序中找根

        # 左子树：中序 [:idx]，前序 [1:1+idx]
        root.left  = self.buildTree(preorder[1:1+idx], inorder[:idx])
        # 右子树：中序 [idx+1:]，前序 [1+idx:]
        root.right = self.buildTree(preorder[1+idx:],  inorder[idx+1:])

        return root
```

**哈希优化版（O(n)）**：

```python
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        in_map = {val: i for i, val in enumerate(inorder)}

        def build(in_left: int, in_right: int,
                  pre_left: int, pre_right: int) -> Optional[TreeNode]:
            if in_left == in_right:
                return None

            root_val = preorder[pre_left]          # 前序首位是根
            root = TreeNode(root_val)

            idx = in_map[root_val]
            left_len = idx - in_left

            root.left = build(
                in_left, idx,
                pre_left + 1, pre_left + 1 + left_len
            )
            root.right = build(
                idx + 1, in_right,
                pre_left + 1 + left_len, pre_right
            )
            return root

        return build(0, len(inorder), 0, len(preorder))
```

#### 图解

```
preorder = [3, 9, 20, 15, 7]
inorder  = [9, 3, 15, 20,  7]

步骤 1：preorder[0] = 3 → 根节点

前序:  [ 3 | 9 | 20  15  7 ]
         ^
         根 3

中序:  [ 9 | 3 | 15  20  7 ]
              ^
          找到根 3，idx=1
      左 [9]     右 [15,20,7]

前序切分（跳过根）:
  左前序 = [9]          (长度 = left_len = 1)
  右前序 = [20, 15, 7]

结果:
        3
       / \
      9  20
        /  \
       15   7
```

### 654. 最大二叉树

#### 核心思路

逻辑与 106/105 完全相同，只是"根节点"的来源变了：当前数组的最大值就是根。找到最大值下标 `idx`，`nums[:idx]` 构建左子树，`nums[idx+1:]` 构建右子树，递归即可。

空数组是必须处理的边界：当切片后的子数组为空时，应返回 `None`。

#### Python 代码

```python
class Solution:
    def constructMaximumBinaryTree(self, nums: List[int]) -> Optional[TreeNode]:
        if not nums:                       # 空数组保护
            return None

        max_val = max(nums)
        idx = nums.index(max_val)          # 最大值下标

        root = TreeNode(max_val)
        root.left  = self.constructMaximumBinaryTree(nums[:idx])
        root.right = self.constructMaximumBinaryTree(nums[idx+1:])

        return root
```

#### 图解

```
nums = [3, 2, 1, 6, 0, 5]

第 1 层：max=6, idx=3
  左 [3,2,1]  右 [0,5]
       |            |
  max=3,idx=0   max=5,idx=1
  左[]  右[2,1]  左[0]  右[]

第 2 层展开：
  [2,1]: max=2,idx=0 → 左[] 右[1]
  [0]:   叶节点 0
  [1]:   叶节点 1

结果:
         6
        / \
       3   5
        \ /
        2 0
         \
          1
```

### 617. 合并二叉树

#### 核心思路

同步递归两棵树。每次进入一层时，先处理空节点的三种终止情况：

1. `root1` 为空 → 直接返回 `root2`（把 root2 整棵子树接过来）
2. `root2` 为空 → 直接返回 `root1`
3. 两者都不为空 → 值相加，递归处理左右子树

这道题不需要切分数组，是四题中最简单的一道。直接修改 `root1` 原地合并，避免新建节点。

#### Python 代码

```python
class Solution:
    def mergeTrees(self, root1: Optional[TreeNode], root2: Optional[TreeNode]) -> Optional[TreeNode]:
        if root1 is None:          # root1 为空，直接用 root2 填充
            return root2
        if root2 is None:          # root2 为空，保留 root1
            return root1

        root1.val += root2.val     # 两节点都存在，值相加（原地修改 root1）

        root1.left  = self.mergeTrees(root1.left,  root2.left)
        root1.right = self.mergeTrees(root1.right, root2.right)

        return root1
```

#### 图解

```
树 1:         树 2:
    1             2
   / \           / \
  3   2         1   3
 /               \   \
5                 4   7

同步递归过程（节点对）：
  (1,2) → val=3，继续递归 (3,1) 和 (2,3)
  (3,1) → val=4，继续递归 (5,None) 和 (None,4)
  (5,None) → root2=None，返回 root1(5)
  (None,4) → root1=None，返回 root2(4)
  (2,3) → val=5，继续递归 (None,None) 和 (None,7)
  (None,7) → root1=None，返回 root2(7)

结果:
        3
       / \
      4   5
     / \   \
    5   4   7
```

## 统一对比

| 题号 | 题目 | 根节点来源 | 切割方式 | 时间复杂度 | 核心难点 |
|------|------|-----------|---------|-----------|---------|
| 106 | 中序+后序构造 | `postorder[-1]`（后序末尾） | 中序找根下标，后序按左长度切割，去掉末尾根 | O(n) 含哈希 | 后序右区间末尾要减 1 |
| 105 | 前序+中序构造 | `preorder[0]`（前序首位） | 中序找根下标，前序跳过首位按左长度切割 | O(n) 含哈希 | 前序左区间起点要加 1 |
| 654 | 最大二叉树 | `max(nums)` | 最大值下标左右各一段，空数组返回 None | O(n²) 最坏 | 空数组边界，忘记 `if not nums` |
| 617 | 合并二叉树 | 两节点值之和 | 无需切割，同步递归 | O(min(m,n)) | 空节点判断必须在操作前 |

## 总结与扩展

三类构造模式覆盖了本文全部四道题：

**遍历序列重建（106 / 105）**：利用前/后序"根节点位置固定"的特性定位根，再借助中序数组切分左右子树。两题完全对称，核心是用哈希表将查找从 O(n) 降为 O(1)，整体达到 O(n)。区间参数统一用左闭右开，减少边界思维负担。

**值驱动构造（654）**：退化版的 106/105，不需要辅助数组，只需在当前范围内找最大值。空数组保护是唯一额外注意点。
**双树同步递归（617）**：不是"从零构造"，而是"在已有结构上合并"。空节点提前返回消除了后续 `.val` 访问的 AttributeError 风险，原地修改 `root1` 避免无谓的内存分配。

**递归写构造题的心法**：不要在脑中展开完整调用栈，只专注当前层做一件事——确定根、切分数据、把子段交给递归。相信递归，不要手动模拟。
