## 识别信号

> 看到什么特征，想到这篇笔记？

- 题目涉及**二叉搜索树（BST）**，或说明左子树所有节点小于根、右子树所有节点大于根
- 题目要求求 BST 中的**最值、差值、排名、众数**——这些在中序有序数组上都是相邻元素问题
- 题目要求对 BST 执行**插入、删除、修剪**——利用 BST 大小关系向左或向右递归，不需要遍历全树

## 核心模板（速查）

### 模板一：BST 中序遍历 + 前驱节点（98 / 530 / 501）

```python
class Solution:
    def inorderWithPrev(self, root):
        self.pre = None          # 记录中序遍历的前一个节点

        def inorder(node):
            if not node:
                return
            inorder(node.left)           # 左：先递归左子树

            # ← 中序时机：此处 pre 是有序序列中紧挨着 node 的前一个节点
            if self.pre is not None:
                # 对相邻两节点做操作，例如：
                diff = node.val - self.pre.val   # 530：计算差值
            self.pre = node              # 更新前驱

            inorder(node.right)          # 右：再递归右子树

        inorder(root)
```

**为什么用 `self.pre` 而不是局部变量**：Python 嵌套函数内对不可变类型（`None`、整数）直接赋值会创建局部变量而非修改外层变量，用 `self` 属性或 `nonlocal` 均可解决。

### 模板二：BST 利用有序性向左/右导航（700 / 235 / 701 / 450 / 669）

```python
def navigate(node, val):
    if not node:
        return None               # 边界：空节点
    if val < node.val:
        return navigate(node.left, val)    # val 在左子树
    elif val > node.val:
        return navigate(node.right, val)   # val 在右子树
    else:
        return node               # 找到目标
```

**3. 538 累加树：用正向中序（从小到大）而非反向中序（从大到小）**

```python
# ❌ 错误：正向中序累加的是「比当前小的值」，结果完全相反
def reverse_inorder_wrong(node):
    inorder(node.left)       # 先走小的一侧
    self.total += node.val
    node.val = self.total
    inorder(node.right)

# ✅ 正确：反向中序（右 → 根 → 左），从大到小累加「所有比当前大的节点之和」
def reverse_inorder(node):
    reverse_inorder(node.right)   # 先走大的一侧
    self.total += node.val
    node.val = self.total
    reverse_inorder(node.left)
```

```python

# ✅ 正确：找右子树的最左节点（真正的后继），再把 root.left 挂到它的左侧
```

### 查找与验证（700 / 98）

#### 700. 二叉搜索树中的搜索

**核心思路**：利用有序性，每次只走一侧，O(log n)。

```text
BST 示例（查找 val=3）：
      5
     / \
    3   7      val < 5 → 走左子树
   / \   \     val == 3 → 找到，返回该节点
  2   4   9
```

```python
from typing import Optional

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val; self.left = left; self.right = right

class Solution:
    def searchBST(self, root: Optional[TreeNode], val: int) -> Optional[TreeNode]:
        if not root:
            return None                          # 没找到
        if val == root.val:
            return root                          # 找到
        elif val < root.val:
            return self.searchBST(root.left, val)   # val 在左子树
        else:
            return self.searchBST(root.right, val)  # val 在右子树
```

> 口诀：**小走左，大走右，等于返回。**

#### 98. 验证二叉搜索树

**核心思路**：中序遍历产生有序序列，用 `self.pre` 记录前一个节点，要求 `pre.val < node.val` 严格成立。

```python
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        self.pre = None          # 中序前驱节点

        def inorder(node):
            if not node:
                return True
            if not inorder(node.left):          # 左
                return False
            # 中：与前驱比较，必须严格递增
            if self.pre is not None and self.pre.val >= node.val:
                return False                    # 违反严格递增，非法
            self.pre = node                     # 更新前驱
            return inorder(node.right)          # 右

        return inorder(root)
```

> 注意：`>=` 而非 `>`，因为 BST 要求**严格**递增（不允许相等值）。

### 统计属性（530 / 501）

#### 530. 二叉搜索树的最小绝对差

**核心思路**：BST 中序 = 有序数组，最小绝对差一定出现在相邻节点之间，边遍历边用 `pre` 计算差值。

```python
class Solution:
    def getMinimumDifference(self, root: Optional[TreeNode]) -> int:
        self.pre = None
        self.min_diff = float('inf')

        def inorder(node):
            if not node:
                return
            inorder(node.left)                                    # 左
            if self.pre is not None:                              # 中
                self.min_diff = min(self.min_diff, node.val - self.pre.val)
            self.pre = node                                       # 更新前驱
            inorder(node.right)                                   # 右

        inorder(root)
        return self.min_diff
```

> 530 与 98 代码骨架几乎一样，都是「中序 + pre 指针」。

#### 501. 二叉搜索树中的众数

**核心思路**：中序遍历中相同值必定连续出现，用 `pre`、`count`、`max_count` 三变量一次遍历完成；`count > max_count` 时**清空**结果集再记录，`count == max_count` 时**追加**。

```python
from typing import List

class Solution:
    def findMode(self, root: Optional[TreeNode]) -> List[int]:
        self.pre = None
        self.count = 0
        self.max_count = 0
        self.result = []

        def inorder(node):
            if not node:
                return
            inorder(node.left)

            # 计算当前值的连续出现次数
            if self.pre is None or node.val != self.pre.val:
                self.count = 1               # 新值，重置计数
            else:
                self.count += 1              # 相同值，累加

            # 更新众数集合
            if self.count > self.max_count:
                self.max_count = self.count
                self.result = [node.val]     # 清空，重新记录
            elif self.count == self.max_count:
                self.result.append(node.val) # 并列，追加

            self.pre = node
            inorder(node.right)

        inorder(root)
        return self.result
```

> 一次遍历即可，无需二次遍历求众数——利用「count > max_count 时清空结果集」的技巧。

### 最近公共祖先（236 / 235）

#### 236. 二叉树的最近公共祖先（普通树）

**核心思路**：后序遍历（左 → 右 → 根），自底向上收集信息。左右子树都找到目标时，当前节点是 LCA；只有一侧找到，则返回那侧的结果。

```text
后序收集示意（p=5, q=1）：
      3            ← 左返回5，右返回1，两侧都有 → 3 是 LCA
     / \
    5   1
   / \ / \
  6  2 0  8
    / \
   7   4
```

```python
class Solution:
    def lowestCommonAncestor(self, root: TreeNode, p: TreeNode, q: TreeNode) -> TreeNode:
        if not root:
            return None
        if root == p or root == q:          # 找到 p 或 q，直接返回
            return root

        left  = self.lowestCommonAncestor(root.left,  p, q)   # 左子树找
        right = self.lowestCommonAncestor(root.right, p, q)   # 右子树找

        if left and right:
            return root                     # 两侧都找到 → 当前节点是 LCA
        return left if left else right      # 只有一侧找到 → 返回那侧
```

#### 235. 二叉搜索树的最近公共祖先

**核心思路**：利用 BST 有序性，自顶向下直接导航。若 p、q 都在左侧则走左，都在右侧则走右，否则当前节点就是 LCA（p 和 q 分处两侧，或当前节点等于其中一个）。

```python
class Solution:
    def lowestCommonAncestor(self, root: TreeNode, p: TreeNode, q: TreeNode) -> TreeNode:
        cur = root
        while cur:
            if cur.val > p.val and cur.val > q.val:
                cur = cur.left              # p、q 都在左子树
            elif cur.val < p.val and cur.val < q.val:
                cur = cur.right             # p、q 都在右子树
            else:
                return cur                  # cur 在 [p,q] 区间内，即是 LCA
        return None
```

| 对比维度 | 236（普通树） | 235（BST） |
|---------|-------------|-----------|
| 策略 | 后序递归，自底向上 | 自顶向下，有序性剪枝 |
| 时间复杂度 | O(n) | O(log n) |
| 代码形式 | 递归 | 迭代 |

### 增删改（701 / 450 / 669）

#### 701. 二叉搜索树中的插入操作

**核心思路**：利用有序性导航到空位，直接创建新节点。递归返回值「接回」新节点，无需额外处理父指针。

```python
class Solution:
    def insertIntoBST(self, root: Optional[TreeNode], val: int) -> Optional[TreeNode]:
        if not root:
            return TreeNode(val)             # 找到空位，创建并返回

        if val < root.val:
            root.left  = self.insertIntoBST(root.left,  val)   # 往左走
        else:
            root.right = self.insertIntoBST(root.right, val)   # 往右走

        return root                          # 返回当前节点让父节点重连
```

#### 450. 删除二叉搜索树中的节点

**核心思路**：找到目标节点后分五种情况处理（可合并为三类）：

```text
情况一：目标是叶节点 → 直接删除（返回 None）
情况二：目标只有右子树 → 右子树顶替
情况三：目标只有左子树 → 左子树顶替
情况四/五：目标有左右两棵子树 →
  找右子树的最左节点（中序后继 successor）
  将目标的左子树挂到 successor 的左侧
  返回目标的右子树作为新根
```

```text
图解（删除节点 5）：
      5                   6
     / \       →         / \
    2   6               2   9
         \             / \
          9           1   3
    （把 5 的左子树 2 挂到后继 6 的左侧）
```

```python
class Solution:
    def deleteNode(self, root: Optional[TreeNode], key: int) -> Optional[TreeNode]:
        if not root:
            return None

        if key < root.val:
            root.left  = self.deleteNode(root.left,  key)   # 向左找
        elif key > root.val:
            root.right = self.deleteNode(root.right, key)   # 向右找
        else:
            # 找到目标节点
            if not root.left:
                return root.right        # 情况一/二：无左孩子
            if not root.right:
                return root.left         # 情况一/三：无右孩子
            # 情况四/五：两个子树都有
            successor = root.right
            while successor.left:
                successor = successor.left      # 找右子树最左节点（后继）
            successor.left = root.left          # 左子树挂到后继节点的左侧
            return root.right                   # 右子树成为新根

        return root
```

#### 669. 修剪二叉搜索树

**核心思路**：利用有序性可以「跳过」整棵子树。`node.val < low` 时左子树全不合法，但**右子树可能有合法节点**；`node.val > high` 时右子树全不合法，但**左子树可能有合法节点**。

```python
class Solution:
    def trimBST(self, root: Optional[TreeNode], low: int, high: int) -> Optional[TreeNode]:
        if not root:
            return None

        if root.val < low:
            # 当前节点及左子树全部不合法，递归修剪右子树后直接返回
            return self.trimBST(root.right, low, high)

        if root.val > high:
            # 当前节点及右子树全部不合法，递归修剪左子树后直接返回
            return self.trimBST(root.left, low, high)

        # 当前节点合法，左右子树仍需递归修剪
        root.left  = self.trimBST(root.left,  low, high)
        root.right = self.trimBST(root.right, low, high)
        return root
```

### 转换（108 / 538）

#### 108. 将有序数组转换为二叉搜索树

**核心思路**：每次取数组中间元素为根，左半段建左子树，右半段建右子树，递归构造。中间元素为根保证高度平衡。

```text
nums = [-10, -3, 0, 5, 9]，mid = len//2 = 2

                0          ← nums[2]
               / \
            -3     5       ← nums[1]、nums[3]
            /         \
         -10             9 ← nums[0]、nums[4]
```

```python
class Solution:
    def sortedArrayToBST(self, nums: List[int]) -> Optional[TreeNode]:
        if not nums:
            return None

        mid  = len(nums) // 2                        # 取中间位置（偶数长度取右中）
        root = TreeNode(nums[mid])                   # 中间元素为根
        root.left  = self.sortedArrayToBST(nums[:mid])       # 左半建左子树
        root.right = self.sortedArrayToBST(nums[mid + 1:])   # 右半建右子树
        return root
```

> `mid = len(nums) // 2` 对偶数长度数组取右中位数，LeetCode 接受左中位数 `(len(nums) - 1) // 2`，两者均合法。

#### 538. 把二叉搜索树转换为累加树

**核心思路**：累加树要求每个节点值 = 原值 + 所有比它**大**的节点之和。BST 中比某节点大的全在「右边和根」，因此用**反向中序遍历（右 → 根 → 左）**，维护一个从大到小的累加和逐节点更新。

```text
反向中序（右→根→左）访问顺序与累加：
      5
     / \
    3   7    访问顺序: 7 → 5 → 3

访问 7: total = 0+7 = 7,  node.val = 7
访问 5: total = 7+5 = 12, node.val = 12
访问 3: total = 12+3 = 15, node.val = 15

结果:     12
          /  \
        15    7
```

```python
class Solution:
    def convertBST(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        self.total = 0       # 从大到小的累加和

        def reverse_inorder(node):
            if not node:
                return
            reverse_inorder(node.right)     # 先右（大值）
            self.total += node.val          # 累加
            node.val = self.total           # 更新当前节点
            reverse_inorder(node.left)      # 再左（小值）

        reverse_inorder(root)
        return root
```

| **669** | 修剪 BST | 整块跳过不合法部分 | 前序 | 返回值重连 | 小了跳右，大了跳左 |
| **108** | 有序数组转 BST | 中间元素为根 | 前序 | `mid` 索引 | 二分递归建平衡树 |
| **538** | 累加树 | 右侧均比当前大 | 反向中序 | `total` | 右→根→左累加 |

**两条主线总结**：

```text
主线一（中序 = 有序数组）：98 / 530 / 501 / 538
  → 把树的遍历等价为有序数组上的双指针，用 pre 指针追踪前驱

主线二（有序性剪枝）：700 / 235 / 701 / 450 / 669 / 108
  → 每次决策"向左还是向右"，跳过无关的半棵树

236 是普通树后序 LCA，作为 235 BST-LCA 的对比基础
```

## 总结与扩展

BST 的十一道题，本质上都在利用**「中序遍历 = 严格递增有序数组」**这一核心性质：

- **统计类**（98 / 530 / 501 / 538）：把树展开为有序数组来思考，用 `self.pre` 在遍历中模拟有序数组上的双指针或前缀和，一次遍历解决问题。538 是反向中序的特殊变体，从大到小累加。

- **操作类**（700 / 235 / 701 / 669 / 108）：利用有序性在每步做出「向左还是向右」的决策，将 O(n) 暴力变为 O(log n) 剪枝。701 和 669 都通过返回值「重连」子树，无需记录父节点。

- **最难的结构操作** 是 450（删除）：需要正确处理有两个子节点的情况——找到中序后继节点（右子树最左节点），将左子树挂接到后继节点的左侧，再返回右子树作为新根。

- **236 vs 235** 是普通树与 BST 的经典对比：前者必须后序遍历 O(n) 全树；后者利用 BST 有序性自顶向下 O(log n) 直接定位。

掌握这两条主线后，BST 的题目可以用统一的思维框架来解决，而不是每道题单独记忆：看到「统计/属性」先想中序 + pre；看到「导航/操作」先想利用大小关系向单侧递归。
