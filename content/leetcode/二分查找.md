## 识别信号

> 看到什么特征，想到这篇笔记？

- 数组**有序**，要求 O(log n) 时间查找某个值或位置
- 题目要求找"第一个满足条件的位置"或"最后一个满足条件的位置"（左/右边界）
- 没有显式数组，但答案的**取值范围具有单调性**（如平方根、判断完全平方数）
- 题目返回"插入位置"，或要求找某个临界点而非精确值

## 核心模板（速查）

### 两套区间写法

**写法一：左闭右闭 `[l, r]`（推荐）**

```python
def binary_search(nums: list[int], target: int) -> int:
    l, r = 0, len(nums) - 1          # 右端是合法下标，区间闭合
    while l <= r:                      # l==r 时仍有一个候选，不能跳过
        mid = l + (r - l) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] > target:
            r = mid - 1               # mid 已排除，右端收到 mid-1
        else:
            l = mid + 1
    return -1
```

**写法二：左闭右开 `[l, r)`**

```python
def binary_search(nums: list[int], target: int) -> int:
    l, r = 0, len(nums)               # 右端是哨兵，不参与比较
    while l < r:                       # l==r 时区间为空，退出
        mid = l + (r - l) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] > target:
            r = mid                   # 右端开，直接赋 mid（不需要 -1）
        else:
            l = mid + 1
    return -1
```

### 三类查找对比表

| 查找目标 | `while` 条件 | 命中时操作 | 未命中返回 | 代表题 |
|---------|-------------|-----------|----------|--------|
| 精确查找 | `l <= r` | `return mid` | `-1` | 704 |
| 左边界（第一个 ≥ target） | `l <= r` | `r = mid - 1` 继续收缩 | 校验 `l` | 34 左端、35 |
| 右边界（最后一个 ≤ target） | `l <= r` | `l = mid + 1` 继续收缩 | 校验 `r` | 34 右端 |

## ⚠️ 高频错误

**1. `while` 条件与区间定义不匹配**

```python
# ❌ 左闭右闭却用 l < r，l==r 时漏掉最后一个候选
l, r = 0, len(nums) - 1
while l < r:       # 错误：应为 l <= r
    ...

# ✅ 左闭右闭对应 l <= r
while l <= r:
    ...
```

**2. 左闭右开时右边界缩到 `mid - 1`**

```python
# ❌ 左闭右开，右端本就不取，缩到 mid-1 会跳过 mid 这个有效右端
r = mid - 1    # 错误

# ✅ 左闭右开的右端赋 mid 即可，无需减一
r = mid        # 正确
```

**3. 找左边界时命中后立即 `return`**

```python
# ❌ 遇到 nums[mid]==target 立即返回，找到的可能不是最左的
if nums[mid] == target:
    return mid    # 可能存在更左的相同元素

# ✅ 命中时继续向左收缩，退出后 l 才是最左位置
if nums[mid] == target:
    r = mid - 1   # 继续往左压
```

**4. 34 题找到边界后未做合法性校验**

```python
# ❌ 直接使用 l，可能越界或指向不等于 target 的元素
return l

# ✅ 必须先校验
if l >= len(nums) or nums[l] != target:
    return -1
return l
```

**5. 值域二分忘记处理 `x == 0` 的边界（69 题）**

```python
# ❌ l=1, r=0 时 while 不进入，ans 未初始化
ans = 0
l, r = 1, x      # x=0 时 r=0 < l=1，直接跳出，ans=0 —— 其实对，但边界不清晰

# ✅ 显式处理，意图明确
if x == 0:
    return 0
```

## 详解

### 704. 二分查找（精确查找）

#### 核心思路

有序数组找目标值——二分最基础形式。每轮将候选区间缩半，`mid` 命中直接返回，否则排除 `mid` 后继续搜索。左闭右闭写法，区间不变量：`target` 若存在，必然在 `[l, r]` 内。

#### Python 代码

```python
class Solution:
    def search(self, nums: list[int], target: int) -> int:
        l, r = 0, len(nums) - 1      # 左闭右闭：右端是合法下标
        while l <= r:                  # l==r 仍有一个候选
            mid = l + (r - l) // 2    # 防整数溢出写法（Python 无溢出，但习惯统一）
            if nums[mid] == target:
                return mid
            elif nums[mid] > target:
                r = mid - 1            # mid 已排除，闭区间向左收
            else:
                l = mid + 1            # mid 已排除，向右收
        return -1
```

**时间复杂度**：O(log n)　**空间复杂度**：O(1)

#### 图解

```text
nums = [-1, 0, 3, 5, 9, 12],  target = 9

初始:  l=0                    r=5
       [-1,  0,  3,  5,  9, 12]
                   ↑ mid=2, nums[2]=3 < 9 → l=3

第2轮: l=3           r=5
       [-1,  0,  3,  5,  9, 12]
                        ↑ mid=4, nums[4]=9 == 9 → return 4 ✓
```

### 35. 搜索插入位置

#### 核心思路

本质是找"第一个 ≥ target 的位置"（左边界）。精确命中时直接返回；未命中时，循环结束后 `l` 恰好落在第一个大于 `target` 的下标，即插入点。**不需要额外处理**，返回 `l` 通吃两种情况。

#### Python 代码

```python
class Solution:
    def searchInsert(self, nums: list[int], target: int) -> int:
        l, r = 0, len(nums) - 1
        while l <= r:
            mid = l + (r - l) // 2
            if nums[mid] == target:
                return mid             # 精确命中，直接返回
            elif nums[mid] > target:
                r = mid - 1
            else:
                l = mid + 1
        # 未命中时 l 落到"第一个严格大于 target 的下标"，即插入位置
        return l
```

**时间复杂度**：O(log n)　**空间复杂度**：O(1)

#### 图解

```text
nums = [1, 3, 5, 6],  target = 2

初始:  l=0  r=3,  mid=1, nums[1]=3 > 2 → r=0
第2轮: l=0  r=0,  mid=0, nums[0]=1 < 2 → l=1
退出:  l=1 > r=0，返回 l=1

验证：在下标 1 处插入 2 → [1, 2, 3, 5, 6] ✓
```

### 34. 在排序数组中查找元素的第一个和最后一个位置

#### 核心思路

拆成两次独立二分：

1. **找左边界**：命中时不返回，令 `r = mid - 1` 继续向左压，退出后 `l` 是最左位置。
2. **找右边界**：命中时令 `l = mid + 1` 继续向右压，退出后 `r` 是最右位置。

两次均用左闭右闭，退出后必须校验边界合法性。

#### Python 代码

```python
class Solution:
    def searchRange(self, nums: list[int], target: int) -> list[int]:

        def find_left(nums: list[int], target: int) -> int:
            """找最小的 i 使 nums[i] == target，不存在返回 -1"""
            l, r = 0, len(nums) - 1
            while l <= r:
                mid = l + (r - l) // 2
                if nums[mid] < target:
                    l = mid + 1
                else:
                    r = mid - 1    # nums[mid] >= target：命中或偏大，继续向左
            # 退出时 l 是第一个 >= target 的位置
            if l >= len(nums) or nums[l] != target:
                return -1
            return l

        def find_right(nums: list[int], target: int) -> int:
            """找最大的 i 使 nums[i] == target，不存在返回 -1"""
            l, r = 0, len(nums) - 1
            while l <= r:
                mid = l + (r - l) // 2
                if nums[mid] > target:
                    r = mid - 1
                else:
                    l = mid + 1    # nums[mid] <= target：命中或偏小，继续向右
            # 退出时 r 是最后一个 <= target 的位置
            if r < 0 or nums[r] != target:
                return -1
            return r

        return [find_left(nums, target), find_right(nums, target)]
```

**时间复杂度**：O(log n)　**空间复杂度**：O(1)

#### 图解

```text
nums = [5, 7, 7, 8, 8, 10],  target = 8

--- find_left ---
l=0, r=5  mid=2, nums[2]=7 < 8  → l=3
l=3, r=5  mid=4, nums[4]=8 ≥ 8  → r=3
l=3, r=3  mid=3, nums[3]=8 ≥ 8  → r=2
退出: l=3，nums[3]=8 ✓  → 左边界 = 3

--- find_right ---
l=0, r=5  mid=2, nums[2]=7 ≤ 8  → l=3
l=3, r=5  mid=4, nums[4]=8 ≤ 8  → l=5
l=5, r=5  mid=5, nums[5]=10 > 8 → r=4
退出: r=4，nums[4]=8 ✓  → 右边界 = 4

结果: [3, 4] ✓
```

### 69. x 的平方根

#### 核心思路

无数组可用——在**值域**上二分。答案 `k` 满足 `k² ≤ x < (k+1)²`，等价于"最大的 k 使 k² ≤ x"，属于**右边界二分**。命中条件改为 `mid * mid <= x`，命中时用 `ans = mid` 暂存再向右继续搜索。

#### Python 代码

```python
class Solution:
    def mySqrt(self, x: int) -> int:
        if x == 0:
            return 0
        l, r = 1, x
        ans = 0
        while l <= r:
            mid = l + (r - l) // 2
            if mid * mid <= x:
                ans = mid          # 暂存可行解，继续往右找更大的
                l = mid + 1
            else:
                r = mid - 1        # mid 太大，向左压
        return ans
```

**时间复杂度**：O(log x)　**空间复杂度**：O(1)

#### 图解

```text
x = 8,  期望输出 2（因为 2²=4≤8，3²=9>8）

l=1, r=8  mid=4, 16 > 8  → r=3
l=1, r=3  mid=2, 4  ≤ 8  → ans=2, l=3
l=3, r=3  mid=3, 9  > 8  → r=2
退出: l=3 > r=2，返回 ans=2 ✓
```

### 367. 有效的完全平方数

#### 核心思路

判断 `num` 是否为完全平方数——即是否存在整数 `k` 使 `k² == num`。同 69 题在值域上二分，但这次是**精确查找**变体：命中 `mid*mid == num` 直接返回 `True`，循环结束未命中返回 `False`。

#### Python 代码

```python
class Solution:
    def isPerfectSquare(self, num: int) -> bool:
        l, r = 1, num
        while l <= r:
            mid = l + (r - l) // 2
            sq = mid * mid
            if sq == num:
                return True          # 精确命中
            elif sq > num:
                r = mid - 1
            else:
                l = mid + 1
        return False
```

**时间复杂度**：O(log num)　**空间复杂度**：O(1)

#### 图解

```text
num = 16,  期望 True
l=1, r=16  mid=8,  64 > 16 → r=7
l=1, r=7   mid=4,  16 == 16 → return True ✓

num = 14,  期望 False
l=1, r=14  mid=7,  49 > 14 → r=6
l=1, r=6   mid=3,  9  < 14 → l=4
l=4, r=6   mid=5,  25 > 14 → r=4
l=4, r=4   mid=4,  16 > 14 → r=3
退出: l=4 > r=3,  return False ✓
```

## 统一对比

| 题号 | 查找类型 | 区间初始化 | `while` 条件 | 命中时操作 | 未命中返回 |
|-----|---------|-----------|-------------|-----------|----------|
| 704 | 精确查找（索引域） | `r = n-1` | `l <= r` | `return mid` | `-1` |
| 35 | 左边界 / 插入点 | `r = n-1` | `l <= r` | `return mid` 或退出用 `l` | `l` |
| 34 左 | 左边界 | `r = n-1` | `l <= r` | `r = mid-1` 继续收缩 | 校验后返回 `l` |
| 34 右 | 右边界 | `r = n-1` | `l <= r` | `l = mid+1` 继续收缩 | 校验后返回 `r` |
| 69 | 右边界（值域） | `l=1, r=x` | `l <= r` | `ans=mid; l=mid+1` | `ans` |
| 367 | 精确查找（值域） | `l=1, r=num` | `l <= r` | `return True` | `False` |

**核心规律**：

- 精确查找：命中即返回，`l` / `r` 严格各退一步
- 左边界：命中时 `r = mid - 1`，退出后 `l` 是答案
- 右边界：命中时 `l = mid + 1`，退出后 `r` 是答案
- 索引域 vs 值域：初始化范围不同，其余逻辑完全一致

## 总结与扩展

二分查找的精髓是**维护区间不变量**：每轮迭代结束后，目标必然仍在当前区间内。选定左闭右闭或左闭右开之后，`while` 条件与边界收缩必须与之严格一致，任何"摇摆"（如左闭右闭却写 `r = mid` 或 `while l < r`）都会造成死循环或漏解。

五道题可归纳为两大模式：
- **索引域二分**（704 / 35 / 34）：在数组下标范围搜索，精确查找 vs 左/右边界查找的差异只在"命中后是否继续收缩"
- **值域二分**（69 / 367）：没有数组，搜索答案的取值范围，`mid` 表示候选答案而非下标，原理完全相同

**扩展练习**：

- [162. 寻找峰值](https://leetcode.cn/problems/find-peak-element/)：无序数组也能二分，靠相邻比较维护不变量
- [33. 搜索旋转排序数组](https://leetcode.cn/problems/search-in-rotated-sorted-array/)：分段有序，先判断 `mid` 落在哪一段再收缩
- [410. 分割数组的最大值](https://leetcode.cn/problems/split-array-largest-sum/)：值域二分进阶，二分答案 + 贪心验证
