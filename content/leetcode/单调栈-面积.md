## 识别信号

> 看到什么特征，想到这篇笔记？

- 给定高度数组，求能**积蓄的水量**，出现"接雨水""蓄水"等词 → 42
- 给定高度数组，柱子宽度为 1，求能勾勒的**最大矩形面积** → 84
- 需要为每个元素找"左右第一个更高/更矮的元素"，要求 O(n) 解法
- 弹出时呈现三元结构：**左边界 ← 弹出元素 → 当前 i（右边界）**，宽度 = `i - left - 1`

## 核心模板（速查）

两题共用同一套框架：遍历数组，维护单调栈（存索引），弹出时以"左边界 ← 弹出 → 右边界"三元组计算面积。
区别只在两处：栈的单调方向（递减 vs 递增）和弹出时高度的取法。

宽度公式两题完全相同：`width = i - stack[-1] - 1`（弹出后的新栈顶是左边界）。

### 42. 接雨水 —— 单调递减栈

```python
def trap(height: list[int]) -> int:
    stack: list[int] = []   # 单调递减栈，存索引；栈底到栈顶高度依次降低
    result = 0
    for i in range(len(height)):
        while stack and height[i] > height[stack[-1]]:
            mid = stack.pop()          # 凹槽底部索引
            if stack:                  # 必须有左边界才能形成有效凹槽
                left = stack[-1]
                width = i - left - 1                              # 凹槽宽度
                h = min(height[i], height[left]) - height[mid]   # 这层水高
                result += width * h
        stack.append(i)
    return result
```

### 84. 柱状图最大矩形 —— 单调递增栈 + 哨兵

```python
def largestRectangleArea(heights: list[int]) -> int:
    heights = [0] + heights + [0]   # 首尾哨兵，消除所有边界特判
    stack: list[int] = []           # 单调递增栈，存索引；栈底到栈顶高度依次升高
    result = 0
    for i in range(len(heights)):
        while stack and heights[i] < heights[stack[-1]]:
            mid = stack.pop()              # 矩形高度来源
            left = stack[-1]               # 哨兵保证此处栈必然非空
            width = i - left - 1           # 矩形可延伸的宽度
            result = max(result, heights[mid] * width)
        stack.append(i)
    return result
```

### 速查对比表

| 项目 | 42 接雨水 | 84 最大矩形 |
|------|----------|-----------|
| 栈的单调性 | 递减（遇高弹出） | 递增（遇矮弹出） |
| 弹出触发 | `height[i] > height[stack[-1]]` | `heights[i] < heights[stack[-1]]` |
| 弹出时高度 | `min(两侧高度) - 底部高度` | `heights[mid]`（弹出柱自身） |
| 弹出时宽度 | `i - left - 1` | `i - left - 1`（**完全相同**） |
| 哨兵 | 不需要（空栈时直接 `break`） | 首尾各加 `0` |

## ⚠️ 高频错误

**1. 哨兵节点：84 题首尾缺一不可**

两种极端输入各自需要一个哨兵，少一个就会漏算：

```python
# ❌ 只加尾部 0：降序数组 [8,6,4,2] 中弹出 8 时栈空，左边界不存在
heights = heights + [0]

# ❌ 只加首部 0：升序数组 [2,4,6,8] 遍历全程只压栈不弹出，结果为 0
heights = [0] + heights

# ✅ 必须首尾都加：首部充当永久左边界，尾部触发所有元素弹出
heights = [0] + heights + [0]
```

**2. 积水计算方向：高度公式"减数"与"被减数"写反**

积水高度 = 水位线 - 底部，水位线由两侧短板决定，不是底部减短板：

```python
# ❌ 错误：底部高度减两侧，结果为负数
h = height[mid] - min(height[i], height[left])

# ✅ 正确：两侧短板决定水位，减去底部高度才是实际水深
h = min(height[i], height[left]) - height[mid]
```

**3. 宽度公式 off-by-one：漏掉 `-1` 会多算右边界一列**

宽度只统计左右边界**之间**的中间列，不含边界本身：

```python
# ❌ 错误：i - left 把右边界 i 本身也算进去了
width = i - left

# ✅ 正确：减 1 排除右边界列
width = i - left - 1
# 直观验证：left=2, i=5 → 中间索引 3、4，共 2 列 = 5 - 2 - 1 ✓
```

**4. 42 题等高柱：用 `>` 触发弹出而非 `>=`，等高直接压栈**

等高柱应成为"更右侧的左边界"，用 `>=` 会在等高时多余弹出并立刻压回，导致后续弹出时左边界偏左、宽度偏小：

```python
# ❌ 错误：height = [5,5,1,3]，i=2 弹出 idx=1 后压入 idx=2；
#          下一次弹出 idx=2 时 left 用了 idx=0，宽度应是 1 却算成 2
while stack and height[i] >= height[stack[-1]]:

# ✅ 正确：等高时直接压栈，等高柱自动成为新的（更右侧的）左边界
while stack and height[i] > height[stack[-1]]:
```

## 详解

### 42. 接雨水：逐层计算图解

#### 核心思路

单调栈解法**横向按水位层**计算积水，与逐列计算的双指针不同。

每次弹出时形成三元结构：
- `mid`：凹槽**底部**（被弹出的索引）
- `left = stack[-1]`：凹槽**左壁**（弹出后的新栈顶）
- `i`：凹槽**右壁**（触发弹出的当前元素）

积水量 = `宽度 × 高度`，高度 = 短板（两侧最小值）减去底部。
弹出后若栈空，说明无左壁，本次凹槽无法积水，直接跳过。

等高柱的处理：使用严格大于（`>`）触发弹出，等高时直接压栈，等高柱自动成为新的左边界，宽度计算自然修正，无需额外逻辑。

#### Python 代码

```python
from typing import List

class Solution:
    def trap(self, height: List[int]) -> int:
        stack: list[int] = []   # 单调递减栈，存索引
        result = 0
        for i in range(len(height)):
            # 遇到比栈顶更高的柱子，凹槽成立，计算每一层积水
            while stack and height[i] > height[stack[-1]]:
                mid = stack.pop()
                if not stack:
                    break               # 弹出后栈空：无左壁，跳过本次计算
                left = stack[-1]
                width = i - left - 1
                h = min(height[i], height[left]) - height[mid]
                result += width * h
            stack.append(i)
        return result
```

时间复杂度 O(n)，空间复杂度 O(n)。每个索引最多入栈一次、出栈一次。

42 题还有双指针 O(1) 空间解法（面试最常见最优解）：两端夹逼，小侧先动，实时维护 `left_max` / `right_max`。单调栈解法的价值在于思路与 84 题统一，掌握一套框架即可覆盖两题。

### 84. 柱状图最大矩形：左右扩展图解

#### 核心思路

以某根柱子的高度为矩形高度时，矩形能向两侧延伸到**左右第一个比它矮**的柱子为止。

单调递增栈在遇到更矮的柱子时弹出，弹出时三元结构中：
- `mid`：矩形**高度**来源（弹出柱自身高度）
- `left = stack[-1]`：矩形**左边界**（弹出后的新栈顶，即左侧第一个更矮的柱）
- `i`：矩形**右边界**（当前触发弹出的更矮柱）

矩形面积 = `heights[mid] × (i - left - 1)`。
哨兵消除了所有 `if` 判空：首部 `0` 是永久左边界，尾部 `0` 触发栈内剩余所有元素弹出。

#### Python 代码

```python
from typing import List

class Solution:
    def largestRectangleArea(self, heights: List[int]) -> int:
        heights = [0] + heights + [0]   # 哨兵：首保左边界，尾触发全部弹出
        stack: list[int] = []           # 单调递增栈，存索引
        result = 0
        for i in range(len(heights)):
            # 遇到比栈顶更矮的柱子，栈顶柱子的左右边界均已确定
            while stack and heights[i] < heights[stack[-1]]:
                mid = stack.pop()
                left = stack[-1]            # 哨兵在，左边界始终存在，无需判空
                width = i - left - 1
                result = max(result, heights[mid] * width)
            stack.append(i)
        return result
```

时间复杂度 O(n)，空间复杂度 O(n)。

## 统一对比

| 维度 | 42 接雨水 | 84 最大矩形 |
|------|----------|-----------|
| 几何含义 | 凹槽积水（低处存水） | 矩形向外扩展（高柱撑起） |
| 寻找目标 | 左右第一个**更高**的柱子 | 左右第一个**更矮**的柱子 |
| 栈的单调性 | 递减（底→顶：高→低） | 递增（底→顶：低→高） |
| 弹出触发 | 遇到**更高**柱弹出 | 遇到**更矮**柱弹出 |
| 弹出时高度 | `min(两侧) - mid 自身` | `heights[mid]` 自身 |
| 弹出时宽度 | `i - left - 1` | `i - left - 1` |
| 哨兵策略 | 无需哨兵，空栈时 `break` | 首尾各加 `0` |
| 计算方向 | 横向（按水位层扫描） | 纵向（以柱高向外延伸） |

两题宽度公式**完全相同**，区别仅在三处：栈的维护方向、弹出触发条件、弹出时高度的取法。

```
弹出时两题共用结构：
  left（新栈顶） ← mid（被弹出） → i（当前，右边界）

  width = i - left - 1          ← 两题完全相同

  42: h = min(height[i], height[left]) - height[mid]
  84: h = heights[mid]
```

**时间与空间复杂度**：

| 题号 | 推荐解法 | 时间复杂度 | 空间复杂度 |
|------|---------|-----------|-----------|
| 42 | 单调栈 | O(n) | O(n) |
| 42 | 双指针（空间最优） | O(n) | O(1) |
| 84 | 单调栈 + 哨兵 | O(n) | O(n) |

42 的双指针解法空间最优，面试中最常考；单调栈解法便于与 84 统一记忆。84 没有 O(1) 空间解法，单调栈是唯一 O(n) 方案。

## 口诀

```
【识别】
  凹槽积水 → 找更高 → 递减栈 → 42
  矩形面积 → 找更矮 → 递增栈 → 84

【弹出时三元结构（两题通用）】
  left（新栈顶） ← mid（被弹出） → i（当前，右边界）
  宽度：i - left - 1
  42 高度：min(height[i], height[left]) - height[mid]
  84 高度：heights[mid]

【哨兵原则】
  42 不加哨兵，空栈则 break 跳过
  84 必须首尾各加 0：
    首部 0 → 永久左边界（防弹出后栈空）
    尾部 0 → 触发全部弹出（防升序数组漏算）

【方向口诀】
  凹槽蓄水找更高，递减栈横向按层扫
  矩形面积找更矮，递增栈哨兵来保驾
  宽度公式两题同，高度取法是关键
```

**42 实现步骤**（4 步）：

```
1. 初始化单调递减栈 stack = []
2. 遍历每根柱子 i：
   a. while 栈非空 且 height[i] > height[stack[-1]]：
      - 弹出 mid；若栈空则 break（无左壁）
      - 计算 width = i - left - 1，h = min(两侧) - mid高，result += width × h
   b. push(i)
3. 返回 result
```

**84 实现步骤**（5 步）：

```
1. heights = [0] + heights + [0]（加哨兵）
2. 初始化单调递增栈 stack = []
3. 遍历每根柱子 i：
   a. while 栈非空 且 heights[i] < heights[stack[-1]]：
      - 弹出 mid；left = stack[-1]（哨兵保证非空）
      - 计算 width = i - left - 1，result = max(result, heights[mid] × width)
   b. push(i)
4. 返回 result
```

## 总结与扩展

42 和 84 是单调栈的两道核心题，形成完美对称：42 用递减栈横向按层计算积水，84 用递增栈以弹出时机确定矩形边界。宽度公式 `i - left - 1` 两题完全相同，唯一区别是高度的取法——42 取短板减底部，84 取弹出柱自身高度。

掌握对称关系后，两道 Hard 只需一套框架推导，区别点仅有三处：栈的方向、弹出触发条件、高度计算公式。哨兵是 84 独有的技巧，首尾各一个 `0` 彻底消除边界特判，代码更干净。
