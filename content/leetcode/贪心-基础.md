## 识别信号

> 看到什么特征，想到这篇笔记？

- 求「最多满足 / 最大收益 / 最长序列」，且每步决策之间相互独立
- 排序后能用一次线性扫描得到最优解，无需回溯或记忆化搜索
- 直觉上每步取最优不会妨碍其他步骤，反证也找不出反例
- 376 / 53：贪心与 DP 均可解——贪心更简洁，DP 思路更通用，二者等价

### 贪心 vs 动态规划

| 特征 | 倾向贪心 | 倾向 DP |
|------|---------|--------|
| 每步决策独立、无后效性 | ✓ | — |
| 需要记录并复用历史状态 | — | ✓ |
| 局部最优 = 全局最优（反证可验证） | ✓ | — |
| 子问题重叠、需要记忆化 | — | ✓ |
| 代表题 | 455 / 122 / 1005 | 背包 / LCS |

## ⚠️ 高频错误

**1. 376：`pre_diff` 每轮都更新，平坡被误计为摆动**

```python
# ❌ 错误：[1, 2, 2, 2, 3] 中，平坡将 pre_diff 重置为 0，
#          下一步同向上升再次触发条件，结果偏大
for i in range(len(nums) - 1):
    cur_diff = nums[i + 1] - nums[i]
    if (pre_diff <= 0 and cur_diff > 0) or (pre_diff >= 0 and cur_diff < 0):
        result += 1
    pre_diff = cur_diff   # 每轮都更新 → 平坡段 pre_diff=0，上升再次触发

# ✅ 正确：只在摆动发生时更新 pre_diff，平坡不打断方向记录
    if (pre_diff <= 0 and cur_diff > 0) or (pre_diff >= 0 and cur_diff < 0):
        result += 1
        pre_diff = cur_diff   # 仅此处更新，平坡不影响 pre_diff
```

**2. 53：遇到负数元素就重置（应是连续和变负才重置）**

```python
# ✅ 正确：先累加元素，连续和本身变为负数时才抛弃
for num in nums:

    result = max(result, count)
    if count < 0:
        count = 0   # 当前前缀已成负担，从下一元素重新开始
```

```python
# ❌ 错误：外层遍历孩子，内层扫描所有饼干
# 每块饼干没有被"消费"，不同孩子可能匹配到同一块饼干

    for cookie in s:

# ✅ 正确：外层遍历饼干（天然保证每块只用一次），孩子指针单向前进

        child += 1
```

## 详解

### 455. 分发饼干

**题意**：孩子胃口数组 `g`，饼干尺寸数组 `s`，`s[j] >= g[i]` 才能满足孩子 `i`，每个孩子最多给一块，求最多能满足几个孩子。

#### 策略证明

贪心选择：**用刚好满足该孩子胃口的最小饼干分配**。

反证：若用更大的饼干给小胃口孩子，大饼干的多余尺寸被浪费，而它本可满足更大胃口的孩子。用更小的饼干同样能满足当前孩子，同时为更大胃口保留了更大的饼干。因此小饼干先满足小胃口是全局最优策略。

#### 图解

```
g（升序）: [1, 2, 3]
s（升序）: [1, 1, 2, 4]

cookie=1 >= g[0]=1  →  child=1（孩子0被满足）
cookie=1  < g[1]=2  →  跳过，这块小饼干无法满足孩子1
cookie=2 >= g[1]=2  →  child=2（孩子1被满足）
cookie=4 >= g[2]=3  →  child=3（孩子2被满足）

结果：3
```

#### Python 代码

```python
class Solution:
    def findContentChildren(self, g: list[int], s: list[int]) -> int:
        g.sort()      # 孩子胃口升序：先满足小胃口，不浪费大饼干
        s.sort()      # 饼干尺寸升序：先用小饼干，大饼干留给大胃口
        child = 0     # 当前待满足孩子的下标

        for cookie in s:                           # 每块饼干只用一次
            if child < len(g) and cookie >= g[child]:
                child += 1                         # 满足当前孩子，移向下一个

        return child  # 被满足的孩子总数
```

时间 O(n log n)，空间 O(1)。

### 376. 摆动序列

**题意**：差值序列正负交替称为摆动序列。可删去若干元素，求最长摆动子序列长度。

#### 策略证明

贪心选择：**不真正删元素，只统计方向改变的峰 / 谷数量**。

单调坡上的中间节点对摆动子序列没有贡献，只有峰顶和谷底有贡献。每次检测到方向从上升转下降（峰）或从下降转上升（谷），计数加一。

`pre_diff` 延迟更新的原因：平坡（相邻相等）不改变坡的方向，不应重置方向记录。若实时更新，平坡后 `pre_diff=0` 会让紧随其后的同向延续误触发计数，导致结果偏大。

#### 图解

```
数组:    1    7    4    9    2    5
差值:       +6   -3   +5   -7   +3
方向:        ↑    ↓    ↑    ↓    ↑
峰/谷:       峰   谷    峰   谷   峰
result:  1 → 2  → 3  → 4  → 5  → 6

平坡情况 [1, 2, 2, 2, 3]（pre_diff 延迟更新的意义）:
差值:       +1    0    0   +1
pre_diff:    1    1    1    1   （平坡不触发更新，pre_diff 保持 +1）
最后 +1 不满足 pre_diff≤0，不触发 → result=2（正确：最长摆动子序列 [1,3]）
```

#### Python 代码

```python
class Solution:
    def wiggleMaxLength(self, nums: list[int]) -> int:
        if len(nums) <= 1:
            return len(nums)

        pre_diff = 0   # 等效于序列首端补一个与 nums[0] 相同的元素
        cur_diff = 0
        result = 1     # 序列末端默认算一个峰 / 谷（至少长度 1）

        for i in range(len(nums) - 1):
            cur_diff = nums[i + 1] - nums[i]

            # 方向从下降/持平转上升（谷），或从上升/持平转下降（峰）
            if (pre_diff <= 0 and cur_diff > 0) or \
               (pre_diff >= 0 and cur_diff < 0):
                result += 1
                pre_diff = cur_diff   # 关键：只在摆动发生时才更新

        return result
```

时间 O(n)，空间 O(1)。

> DP 解法：`dp[i][0]` 表示以第 i 个元素为峰结尾的最长摆动长度，`dp[i][1]` 为以谷结尾，O(n²)；贪心更优。

### 53. 最大子数组和

**题意**：找一个连续子数组使元素和最大，返回最大和（数组非空，可包含负数）。

#### 策略证明

贪心选择：**当前连续和变为负数时立刻抛弃，从下一个元素重新开始累积**。

若当前前缀和为负，将其拼接到后续任何子数组上只会让总和减小。因此一旦连续和 < 0，后续子数组从下一元素重新起算比带着负前缀更优。局部最优（不保留负前缀）叠加得到全局最优（最大连续和）。

#### 图解

```
index:  0    1    2    3    4    5    6    7    8
nums:  -2    1   -3    4   -1    2    1   -5    4
count: -2    1   -2    4    3    5    6    1    5
        ↓         ↓
       重置       重置
      (count<0) (count<0)
result:-2    1    1    4    4    5    6    6    6

最大子数组：[4, -1, 2, 1]，和 = 6
重置发生在 index=0（count=-2<0）和 index=2（count=-2<0）。
```

#### Python 代码

```python
class Solution:
    def maxSubArray(self, nums: list[int]) -> int:
        result = float('-inf')   # 负无穷：处理全负数组（任意单元素都是有效答案）
        count = 0                # 当前连续和

        for num in nums:
            count += num
            result = max(result, count)   # 实时记录历史最大值
            if count < 0:
                count = 0        # 负前缀是累赘，从 0 重新开始累积

        return result
```

时间 O(n)，空间 O(1)。

> 等价于 Kadane 算法，对应 DP 转移 `dp[i] = max(dp[i-1] + nums[i], nums[i])`，贪心写法更简洁。

### 122. 买卖股票的最佳时机 II

**题意**：每天可以买卖股票，同一时刻只能持有一只，可无限次买卖，求最大利润。

#### 策略证明

贪心选择：**只要明天涨，今天就买——收集所有正涨幅区间的利润**。

关键推导：任意一段完整收益都可以被拆解为逐日差值之和：

```
prices[4] - prices[1]
= (prices[4]-prices[3]) + (prices[3]-prices[2]) + (prices[2]-prices[1])
```

只累加正数项，等价于找到了所有最优买卖区间。不遗漏任何上涨段 = 全局利润最大。

#### 图解

```
prices = [7, 1, 5, 3, 6, 4]

逐日差值: 1-7=-6   5-1=+4   3-5=-2   6-3=+3   4-6=-2
只收正值:            +4                +3
总利润 = 4 + 3 = 7

对应操作：第2天买入，第3天卖出（+4）；第4天买入，第5天卖出（+3）
```

#### Python 代码

```python
class Solution:
    def maxProfit(self, prices: list[int]) -> int:
        profit = 0
        for i in range(len(prices) - 1):
            # 只累加正涨幅：今天到明天上涨才收利润，下跌直接跳过
            profit += max(prices[i + 1] - prices[i], 0)
        return profit
```

时间 O(n)，空间 O(1)。无需显式记录买入卖出点，利润拆解后逐日累加即可。

### 1005. K 次取反后最大化的数组和

**题意**：可将数组某元素取反，操作恰好 K 次（可重复选同一位置），求操作后数组的最大和。

#### 策略证明（两轮贪心）

**第一轮贪心**：按绝对值降序，优先翻转绝对值最大的负数。

反证：跳过绝对值大的负数去翻小的，总和增益更小，而大负数留着持续拖累总和。绝对值大的负数翻正后对总和增益最大，应优先处理。

**第二轮贪心**：K 剩余为奇数次时，翻转绝对值最小的数。

奇数次翻同一个元素的净效果 = 翻一次；偶数次 = 不变。翻绝对值最小的数，对总和的负面影响最小。绝对值降序排序后，`nums[-1]` 始终是绝对值最小的元素。

#### 图解

```
nums = [3, -1, 0, 2], k = 3
绝对值降序排列: [3, 2, -1, 0]

第一轮遍历（翻负数）:
  nums[0]=3  ≥0 → 跳过 (k 仍为 3)
  nums[1]=2  ≥0 → 跳过 (k 仍为 3)
  nums[2]=-1 <0  → 翻为 +1 (k=2)
  nums[3]=0  ≥0 → 跳过 (k 仍为 2)

第二轮：k=2 为偶数 → 不翻
结果数组: [3, 2, 1, 0]，sum = 6
```

#### Python 代码

```python
class Solution:
    def largestSumAfterKNegations(self, nums: list[int], k: int) -> int:
        # 按绝对值从大到小排序：优先处理大负数，末尾保留绝对值最小的元素
        nums.sort(key=lambda x: abs(x), reverse=True)

        # 第一轮：从前往后依次翻转绝对值大的负数
        for i in range(len(nums)):
            if nums[i] < 0 and k > 0:
                nums[i] = -nums[i]
                k -= 1

        # 第二轮：k 还有剩余且为奇数，翻转绝对值最小的数（排序后的末尾元素）
        # 偶数次翻同一个数净效果为零，无需操作
        if k % 2 == 1:
            nums[-1] = -nums[-1]

        return sum(nums)
```

时间 O(n log n)，空间 O(1)。

## 总结与扩展

贪心没有固定模板，核心只有一件事：**每道题都要说清楚贪心选择是什么，并给出局部最优推全局最优的直觉或反证**。

这五道题覆盖了贪心的基本形态：

- **455**：排序后双指针匹配，贪心策略几乎等同常识，是贪心入门题
- **376**：不实际操作数组，用状态变量 `pre_diff` 模拟删除效果，稍有难度
- **53**：Kadane 算法，贪心等价于 DP，理解"抛弃负前缀"的直觉比记公式更重要
- **122**：利润分解是关键，把复杂的买卖点问题转化为逐日正差值累加，极度简洁
- **1005**：两轮贪心嵌套，同一道题内分阶段贪心，体现贪心策略可以组合使用

376 和 53 还可以用 DP 解——贪心更简洁，DP 思路更通用。两种方法都掌握，遇到新题时会更有把握。
