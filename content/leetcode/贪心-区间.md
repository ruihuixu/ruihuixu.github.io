## 识别信号

> 看到什么特征，想到这篇笔记？

- 给出一组区间 `[l, r]`，要求**最少操作数**（最少箭、最少移除）或**合并后的结果**
- 题目问"最多能保留几个不重叠区间"或"最少要去掉几个"——两种问法本质相同
- 字符串题要求**同一字符只出现在一个片段**，切割成尽可能多的部分
- 关键判断：是要"分开"（找不重叠组数）还是"合并"（粘成一块）——决定排序方向

## 核心模板（速查）

### 按右端点排序模板（452 / 435）

```python
# 适用：找最多不重叠组数，或统计需要移除的区间数
def template_right_sort(intervals: list[list[int]]) -> int:
    intervals.sort(key=lambda x: x[1])   # 按右端点升序
    count = 1
    boundary = intervals[0][1]           # 当前组的右端点（箭位 / 保留区间右端）

    for i in range(1, len(intervals)):
        if intervals[i][0] > boundary:   # 452：严格 >（相邻算一组）
        # if intervals[i][0] >= boundary: # 435：>= 表示相邻不重叠，可保留
            count += 1
            boundary = intervals[i][1]
        # 否则重叠，跳过（452=不加箭；435=不保留此区间）

    return count  # 452 直接返回；435 返回 len(intervals) - count
```

各题在右端点模板里的唯一区别：

| 题号 | 判断符号 | 返回值 | 含义 |
|------|---------|--------|------|
| 452  | `> boundary` | `count` | 相邻气球共用一支箭，严格超出才加箭 |
| 435  | `>= boundary` | `len - count` | 相邻不算重叠，可各自保留 |

### 按左端点排序模板（56）

```python
# 适用：合并所有重叠区间
def template_left_sort(intervals: list[list[int]]) -> list[list[int]]:
    intervals.sort(key=lambda x: x[0])   # 按左端点升序
    result = [intervals[0]]

    for cur_l, cur_r in intervals[1:]:
        if cur_l <= result[-1][1]:        # 相邻也算重叠（<=），合并
            result[-1][1] = max(result[-1][1], cur_r)   # 右端点取 max！
        else:
            result.append([cur_l, cur_r])

    return result
```

### 763 最远出现位置模板（无需排序）

```python
# 适用：划分字母区间，维护「最远出现下标」
def template_last_pos(s: str) -> list[int]:
    last = {c: i for i, c in enumerate(s)}   # 每个字母最后出现的下标
    result = []
    left = right = 0

    for i, c in enumerate(s):
        right = max(right, last[c])           # right 只增不减
        if i == right:                        # 遍历到右边界，切割
            result.append(right - left + 1)
            left = i + 1

    return result
```

## ⚠️ 高频错误

**1. 452 把 `>` 写成 `>=`（相邻气球被多算一支箭）**

```python
# ❌ 错误：[1,2] 和 [2,3] 相邻，本可被一支箭穿过，却多加了一支
if points[i][0] >= arrow_pos:
    arrows += 1

# ✅ 正确：只有严格超出箭位才需要新箭
if points[i][0] > arrow_pos:
    arrows += 1
```

**2. 435 把 `>=` 写成 `>`（相邻区间被误判为重叠，少保留一个）**

```python
# ❌ 错误：[1,2] 和 [2,3] 边界相触不算重叠，两个都可保留
if intervals[i][0] > end:
    count += 1

# ✅ 正确：左端点等于 end 时也不重叠，可以保留
if intervals[i][0] >= end:
    count += 1
```

**3. 435 直接返回 `count` 而忘了做减法**

```python
# ❌ 错误：count 是「最多保留数」，不是「最少移除数」
return count

# ✅ 正确：题目问移除数 = 总数 - 最多保留数
return len(intervals) - count
```

**4. 56 合并时忘记取 `max`，右端点反而缩水**

```python
# ❌ 错误：如 [1,10] 后接 [2,5]，合并后变成 [1,5]，把右端点压缩了
result[-1][1] = cur_right

# ✅ 正确：取两者中较大的右端点，防止当前区间被完全包含时缩水
result[-1][1] = max(result[-1][1], cur_right)
```

**5. 763 扩展右边界时直接赋值，right 倒退导致提前错误切割**

```python
# ❌ 错误：right 可能倒退，提前触发切割
right = last[c]

# ✅ 正确：right 只增不减
right = max(right, last[c])
```

## 详解

### 452. 用最少数量的箭引爆气球

#### 核心思路

**局部最优**：重叠的气球用一支箭一起射，箭数最少。
**全局最优**：总箭数最少。

按右端点升序排序后，把箭打在当前重叠组的最小右端点处——向左覆盖所有重叠气球，同时为后续气球"让出"最大空间。

#### 图解

```
输入：[[10,16],[2,8],[1,6],[7,12]]
按右端点排序：[[1,6],[2,8],[7,12],[10,16]]

arrow=6（射第一个气球右端点）
[2,8]  : left=2 <=6，同一支箭穿过，继续
[7,12] : left=7 > 6，射不到！新箭，arrow=12
[10,16]: left=10<=12，同一支箭穿过，继续

结果：2 支箭
```

#### Python 代码

```python
from typing import List


class Solution:
    def findMinArrowShots(self, points: List[List[int]]) -> int:
        if not points:
            return 0
        # 按右端点升序：优先处理结束最早的气球，给后面留最大空间
        points.sort(key=lambda x: x[1])
        arrows = 1
        arrow_pos = points[0][1]   # 第一支箭射在第一个气球的右端点

        for i in range(1, len(points)):
            # 左端点严格超过箭位，当前箭射不到，必须新增一支
            if points[i][0] > arrow_pos:
                arrows += 1
                arrow_pos = points[i][1]   # 新箭落在当前气球右端点
            # 否则左端点 <= arrow_pos，当前箭可穿过，无需操作

        return arrows
        # 时间 O(n log n)，空间 O(1)
```

### 435. 无重叠区间

#### 核心思路

换个角度：**最少移除数 = 总数 − 最多保留的互不重叠区间数**。

求最多不重叠区间数与 452 本质相同——都是按右端点贪心。唯一区别：
- 452 中相邻气球（`[1,2]` 和 `[2,3]`）共用一支箭（判断用 `>`）
- 435 中相邻区间不算重叠，可以各自保留（判断用 `>=`）

#### 图解

```
输入：[[1,2],[2,3],[3,4],[1,3]]
按右端点排序：[[1,2],[2,3],[1,3],[3,4]]

end=2，count=1（保留[1,2]）
[2,3]：left=2>=end=2，不重叠，保留，end=3，count=2
[1,3]：left=1< end=3，重叠，跳过（移除）
[3,4]：left=3>=end=3，不重叠，保留，end=4，count=3

最多保留 3 个，最少移除 4-3 = 1 个
```

#### Python 代码

```python
class Solution:
    def eraseOverlapIntervals(self, intervals: List[List[int]]) -> int:
        if not intervals:
            return 0
        # 按右端点升序：每次保留结束最早的，给后面留最大空间
        intervals.sort(key=lambda x: x[1])
        count = 1                      # 最多可保留的不重叠区间数
        end = intervals[0][1]          # 当前已保留的最后一个区间的右端点

        for i in range(1, len(intervals)):
            if intervals[i][0] >= end:
                # 左端点 >= end：边界相触也不算重叠，可以保留
                count += 1
                end = intervals[i][1]
            # 否则重叠，跳过（相当于移除这个区间）

        # 总数减去最多保留数 = 最少移除数
        return len(intervals) - count
        # 时间 O(n log n)，空间 O(1)
```

### 763. 划分字母区间

#### 核心思路

一个片段的右边界，由该片段内**所有字母最后出现位置的最大值**决定。维护一个只增不减的右边界 `right`，当遍历位置 `i` 恰好等于 `right` 时，意味着片段内所有字母已"闭合"，此处切割。

两步走：
1. 预处理：记录每个字母最后出现的下标 `last[c]`
2. 贪心扫描：遍历时扩展 `right`，`i == right` 时切割

#### 图解

```
s = "ababcbacadefegdehijhklij"
last: a→8, b→5, c→7, d→14, e→15, f→11, ...

i=0('a'): right=max(0,8)=8
i=1('b'): right=max(8,5)=8
...
i=8('a'): right=8，i==right → 切割！长度=8-0+1=9，left=9

i=9('d'): right=max(9,14)=14
...
i=15('e'): right=15，i==right → 切割！长度=15-9+1=7，left=16

...最后一段长度=8

结果：[9, 7, 8]
```

#### Python 代码

```python
class Solution:
    def partitionLabels(self, s: str) -> List[int]:
        # 第一步：记录每个字母最后出现的下标（后面的覆盖前面的）
        last = {c: i for i, c in enumerate(s)}

        result = []
        left = 0    # 当前片段的左边界
        right = 0   # 当前片段的右边界（只增不减）

        for i, c in enumerate(s):
            # 当前字母的最后位置可能更远，扩展右边界
            right = max(right, last[c])
            if i == right:
                # 遍历到右边界：片段内所有字母均已收录，在此切割
                result.append(right - left + 1)
                left = i + 1   # 下一片段从 i+1 开始

        return result
        # 时间 O(n)，空间 O(1)（字母表固定 26 个）
```

### 56. 合并区间

#### 核心思路

与 452/435 的"分开"思维相反，56 是"合并"思维：把所有重叠的区间粘成一个大区间。

按**左端点**排序（而非右端点）——保证相邻区间在空间上连续排列，一旦当前区间的左端点超过前一个区间的右端点，后续所有区间也必然超过（已排序），可以安全切断。

注意：本题边界相触（`[1,4]` 和 `[4,5]`）也算重叠，需要合并，判断用 `<=`。

#### 图解

```
输入：[[1,3],[2,6],[8,10],[15,18]]
按左端点排序：[[1,3],[2,6],[8,10],[15,18]]

result=[[1,3]]
[2,6] : left=2<=3，重叠，合并，right=max(3,6)=6 → result=[[1,6]]
[8,10]: left=8> 6，不重叠，加入  → result=[[1,6],[8,10]]
[15,18]:left=15>10，不重叠，加入 → result=[[1,6],[8,10],[15,18]]
```

#### Python 代码

```python
class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        if not intervals:
            return []
        # 按左端点升序：保证重叠区间相邻排列，一次扫描即可完成合并
        intervals.sort(key=lambda x: x[0])
        result = [intervals[0]]

        for i in range(1, len(intervals)):
            cur_left, cur_right = intervals[i]
            if cur_left <= result[-1][1]:
                # 左端点 <= 当前末尾右端点：重叠（含边界相触），合并
                # 必须取 max：防止当前区间被完全包含时右端点缩水
                result[-1][1] = max(result[-1][1], cur_right)
            else:
                # 不重叠，直接加入结果
                result.append([cur_left, cur_right])

        return result
        # 时间 O(n log n)，空间 O(n)（结果数组）
```

## 统一对比（各题排序策略）

```
区间贪心家族
├── 找「最多不重叠」→ 按右端点排序
│   ├── 452（扎气球）  箭数 = 重叠组数，直接返回
│   └── 435（移除区间）答案 = 总数 − 最多保留数
├── 合并重叠区间 → 按左端点排序
│   └── 56（合并区间） 重叠粘连，返回合并后列表
└── 划分字符串片段（区间思想的变体）
    └── 763（划分字母）last 数组代替区间端点，无需排序
```

| 题号 | 排序方式 | 核心变量 | 重叠/切割判断 | 返回值 |
|------|---------|---------|-------------|-------|
| 452  | 右端点升序 | `arrow_pos` | `left > arrow_pos`（严格 `>`） | 箭数 |
| 435  | 右端点升序 | `end` | `left >= end`（`>=` 不重叠则保留） | `总数 - 保留数` |
| 763  | 无（用 `last`） | `right` | `i == right`（抵达右边界切割） | 各段长度列表 |
| 56   | 左端点升序 | `result[-1][1]` | `left <= result[-1][1]`（`<=` 含边界） | 合并后区间列表 |

**三个边界符号的记忆口诀**：
- 452 用 `>`：相邻气球**共用一箭**，不超出不加箭
- 435 用 `>=`：相邻区间**各自独立**，可以都保留
- 56 用 `<=`：相邻区间**视为重叠**，必须合并

## 口诀

```
区间贪心两件事：先排序，再扫描。

按右端点排 → 留结束早的，给后面腾空间（452 / 435）
按左端点排 → 从左到右扫，判断是否能合并（56）
不排序用 last → 维护右边界，i == right 就切割（763）

三个符号要分清：
  452 用 >   ——相邻气球共一箭，严格超出才加箭
  435 用 >=  ——相邻不重叠，各自保留，答案要做减法
  56  用 <=  ——相邻也要合并，右端点必须取 max

435 答案 = 总数 − 最多保留数，别忘做减法。
56 合并时右端点取 max，防止被大区间吞掉后缩水。
763 right 只增不减，直接赋值会倒退。
```

## 总结与扩展

四道区间贪心题统一在「排序 + 贪心扫描」框架下，核心差异只有**排序方向**和**边界符号**：

- **452（扎气球）** 和 **435（移除区间）** 底层算法完全相同，都是按右端点排序求最多不重叠组数；差异仅在判断符号（`>` 与 `>=`）和返回值形式——452 直接返回箭数，435 用总数相减。
- **56（合并区间）** 与前两题「镜像对立」：前两题找不重叠，56 合并重叠；排序方向也从右端点变为左端点。
- **763（划分字母）** 是 452/435 框架在字符串上的变体，用字母最后出现位置替代区间端点，时间复杂度降至 O(n)；它也可以完整转化为 56 题（构造字母区间后按左端点合并）。
