## 项目概述

**一句话定位**：基于统一表征学习的序列推荐系统，采用因果 Transformer 主干 (HSTU) + 向量检索 (ANN) 的技术路线，定位为高质量召回 + 轻粗排一体化层。

### 背景与挑战

传统双塔召回系统存在三个核心问题：

1. **兴趣表达静态化**：用户向量更新频率低（小时/天级），无法捕捉实时兴趣漂移
2. **序列信息丢失**：历史行为通常 pooling 为单一向量，时序结构被压缩
3. **训练 - 推理割裂**：召回和排序目标不一致，特征体系不统一

本项目的目标是在"效率 - 效果"曲线上找到更好的甜点区——比双塔召回更准，比精排成本更低。

---

## 核心工作

### 1. 数据底座建设

**问题**：高维多模态特征（82+ 号特征，维度 1024-4096）全量加载导致内存压力，训练 - 推理特征口径不一致。

**解决方案**：

- **索引化惰性读取**：只在需要时加载对应样本的特征，内存占用从 TB 级降至 GB 级
- **统一补齐函数**：`process_cold_start_feat` 同时服务于训练和推理，确保特征处理口径一致
- **泛化交叉特征**：支持动态配置和自动发现，便于新特征快速接入

**技术价值**：数据链路从"能跑"升级到"可持续"，为后续模型迭代打下可扩展的基础。

---

### 2. 模型架构升级

**主干网络**：采用 8 层 HSTU（Hierarchical Sequential Transfer Unit）作为序列编码器。

**核心设计**：

| 模块 | 作用 | 技术细节 |
|------|------|----------|
| **三重时间编码** | 捕捉多维时间语义 | RoPE（相对位置）+ RelativeTimeBias（可学习间隔）+ FourierTimeEncoding（绝对时间） |
| **多路径注意力** | 融合不同序列模式 | 标准自注意力 + RoPE 路径 + 时间偏置路径 |
| **RMSNorm + DeepNorm** | 稳定深层训练 | 替代 LayerNorm，残差缩放系数优化 |

**训练目标**：InfoNCE + 全量难负样本 + in-batch ranking 联合优化

- InfoNCE：拉近用户向量与正样本，推远负样本
- Ranking：候选集合内相对排序，提升首屏命中质量
- -logQ 修正：校正 popular 采样偏置

---

### 3. 训练优化

**混合精度训练**：
- AMP（Automatic Mixed Precision）支持 BF16/FP16
- 显存占用减半，训练速度提升 2-3x

**学习率调度**：
- Warmup + Cosine 衰减
- 梯度裁剪防止梯度爆炸

**开关化实验设计**：
- 每个优化模块支持独立开关
- 便于灰度发布、快速回滚和消融分析

---

### 4. 工程治理

**向量检索**：
- PyTorch 原生分块 TopK 实现
- 支持 FP16 加速，双分块显存管理
- 无 FAISS 依赖，灵活定制

**配置治理**：
- 固定随机种子确保可复现
- 保存完整训练配置与指标快照
- manifest 文件管理交叉特征

**分层评估体系**：
- 整体指标（HR@10, NDCG@10, score）
- 分群报表（识别收益是否集中在少数人群）
- 时间切分验证（避免数据泄露）

## 技术亮点

### HSTU 主干网络

```
用户行为序列 → 特征映射 → 三重时间编码 → 8 层 HSTU → 归一化 → query 向量
                                            ↓
                                    多路径注意力融合
                                    (RoPE + 时间 + 标准)
```

相比标准 Transformer，HSTU 的优势：
- 三路注意力并行，表达力更强
- RelativeTimeBias 针对推荐场景定制
- RMSNorm 更稳定，适合深层网络

---

### RelativeTimeBias（当前项目独有）

可学习相对时间偏置，针对时间间隔建模：

```python
# 伪代码示意
def forward(self, seq, ts):
    # ts: 时间戳序列
    delta_ts = ts[:, :, None] - ts[:, None, :]  # 时间间隔矩阵
    rel_bias = self.learnable_bins(delta_ts)    # 离散化 + 可学习嵌入
    attn_scores += rel_bias                      # 加到注意力分数
```

**设计动机**：RoPE 捕捉相对位置，但对时间间隔不敏感。RelativeTimeBias 补充"两次行为相隔多久"的语义。

---

### 检索 - 排序一致性

传统链路：
```
召回（双塔）→ 粗排（简化精排）→ 精排（完整模型）
   ↓            ↓                   ↓
 目标 1       目标 2              目标 3  (可能不一致)
```

当前项目：
```
统一表征空间
   ↓
训练：InfoNCE(q, v_pos, v_neg)
检索：topK by q · v_i
粗排：score = q · v_i
```

**价值**：训练、检索、粗排用同一套相似度计算，没有"训练学一套，检索用另一套"的问题。

## 总结与反思

### 核心价值

这个项目最大的价值不是"离线指标涨了多少"，而是形成了**可持续迭代的推荐工程能力**：

1. 数据底座：索引化惰性读取 + 统一补齐函数
2. 模型设计：HSTU + RelativeTimeBias + 三重时间编码
3. 训练治理：AMP + 调度器 + 开关化实验
4. 评估体系：整体指标 + 分群报表 + 时间切分

### 待改进方向

1. **线上验证**：当前处于离线验证完成阶段，需时间切分和线上 AB 持续验证
2. **HSTU vs Transformer**：待补直接消融实验
3. **规模上限**：PyTorch TopK 在亿级候选库下不如 FAISS 成熟
